Lookups with metadata

Metadata is data about data and provides even more flexibility for lookup use, enabling data associated directly with a particular lookup selection. When adding a lookup group, you can select a dataform to act as the metadata form for that group (see 1. Add a Lookup Group in Configuring lookups). These are ordinarily custom forms created to store information related to the lookup values used with the lookup group. When a metadata form is associated with a lookup group, the lookups for that group indicate metadata, as shown.

Examples

The Contract Type lookup in ClientSpace PEO has associated metadata that can determine:

  • What Template is associated with the contract type?
  • What Legal Entity / SUTA defaults are available when that contract type is selected?
  • Default Workers' Comp Configuration for a workspace.

 

The Pay Frequency in ClientSpace PEO has metadata associated that can determine:

  • How many pay cycles per year for that frequency, such as Weekly = 52, SemiMonthly = 24, Bi-Weekly = 26.
  • Minimum per check fee to ensure a certain amount of administrative costs are submitted for the year.

 

Before you configure lookup metadata, please log an Extranet case.
