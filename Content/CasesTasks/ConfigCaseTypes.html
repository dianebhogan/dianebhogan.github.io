Configuring case types

You can add case types with default values so that when your users add cases, some of the fields are all set! To add client service case types, you must be a Global Admin and assigned a role with Edit rights to the gen_AdminCaseTypes form. To view a list of your case types, see Finding your case types.

To add client service case types:

Go to Admin Workspace.

Search for Client Service Case Type or select Client Service > Client Service Case Type.

The Client Service Case Type displays.

Click Add.

The Client Service Case Type Add form opens.

Complete the form as required:

Default Subject

The Default Subject field can be used to set the Subject line for the Case Type in question on create. After this field is filled, the subject line no longer defaults. This is by design. We want to preserve any additional information that may have been added.

Default Issue

The Default Issue field can be used to set the Issue field for the Case Type in question on create. After this field is filled, the Issue field no longer defaults. This is by design as we want to preserve any additional information that may have been added.

Default Assignment

When you assign a case, you can specify the Default Assignment as Creator, Single User, or Client Role.

  • Creator: Defaults to the Assigned To user on the case, the person that creates the case
  • Single User: Defaults to the user selected in Assigned To User (fields change depending on the selection in Default Assignment).
  • Client Role: Enables Assigned To Field and Backup Assigned To.

 

When you select Client Role, Assign To Field and Backup Assigned To are required.

  • Backup Assigned To ensures that if the selected client role is not completed on the Client Team dataform, the case type defaults to a meaningful value.
  • If the Client Team field is empty and the Backup Assigned To has not been set, the Default Assignment reverts to the Manager of the Responsible Department field on the Client Service Case Category metadata associated with the case type.

The system dynamically selects the targeted user based on the Client workspace settings.

Click Save.
