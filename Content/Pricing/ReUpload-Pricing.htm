Re-uploading pricing data

After you have generated the initial pricing, you may need to generate pricing again from time-to-time because numbers change. For example, employee numbers can fluctuate, which then changes the payroll numbers. Re-uploading the pricing data file makes this process very easy.

To re-upload the data:

  1. From the Pricing Console, in the States and Codes area, click Pre-load.
    The Pre-load State and Code List dialog box opens.

  1. Click Choose File.
    File Explorer opens.

  2. Locate the file and click Open.
    The file appears in the Choose File box.

  3. Click Pre-load.
    For data that already exists, the pre-load process displays an informative message that requires action when returning to the grid.

  1. Click Ok to close the message.
    You are returned to the Pricing Console. The newly uploaded row appears in a different grid, States and Codes Pre-load Worksheet, for reviewing the data, potentially correcting and re-uploading, and merging when ready. In our example, we have an entry with red text and yellow background, which indicates the data already exists, and we need to determine if a merge is a correct action.

  1. To merge all data when you have multiple entries in the grid, click Merge All.

  2. To merge only the selected row, click Merge Selected.
