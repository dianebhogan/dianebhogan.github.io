Configuring Slack

A Slack administrator will have the ability within Slack to Browse and Add new apps to the Slack install. This normally appears at the bottom of the Slack user list.

To configure Slack:

Open Slack.

Select Apps.

The Browse Apps page opens.

If not installed, install the Slash Commands app.

If Slash Commands is already installed, click View to add a new configuration.
The Slash Commands page opens.

Click Add Configuration.

The Integration Settings page opens.

To configure the integration settings for your installation:

Command: The precursor command you will use in Slack to begin requesting data from . In our example we are using /clientspace.

URL: The install URL of your installation with /next/peo/slack appended to the end. For example, https://yourinstall.clientspace.net/next/peo/slack

Method: Select POST. This is the request-response method when connecting the third- party application.

Token: Simply adding the configuration will cause the Slack system to generate a token. This token identifies the Slack application for third-party integrations and will be used when configuring the application. Copy this token to notepad because you will use this later.

Customize Name: The name that will identify the slash command responses. The Slack username that your ClientSpace data will come from.

Customize Icon: The icon that will appear in Slack next to the response.

Preview Message: This field will give you an advance preview of what you can expect the slack interaction to look like.

All other configurations should be left at their defaults.

Click Save to save your Slash Command Config.
