Securing attachments

You secure attachments (files) the same way for non-admins across all entities in the system.

All entities work like this:

First, you must have Edit rights for the entity (dataform or otherwise) that you are on to do anything but view files.

If you have Edit rights, you can optionally lock files down using an _$Attachment security entity.

Entity Security entity

Dataforms

Use {TableName}_$Attachment: Set by selecting the Attachments Secured option on Dataform properties.

Orgs, Contacts, Activities

Use CRM_$Attachment: Must be added by a representative. If you would like to utilize CRM attachment security, log an Extranet case.

Tasks

Use Incident_$Attachment: Must be added by a representative. If you would like to utilize task attachment security, log an Extranet case.

Attachments panel in Client Master

To use the Attachments panel on the Client Master, you need to be part of a role that has the gen_ClientMaster_$Attachment security entity.

To add a security entity to a role:

Go to System Admin > Security > Roles.

The Security Roles dashboard opens.

Filter the roles by Role Title, Users that are associated with the role, or Entity Name of the associated Security Entities.

If a user needs access to a particular security entity, do one of the following:

Add the user to a role that already contains the required entity by filtering roles by desired entity.

Add the security entity to a role that user is already associated with by filtering roles.

Add the appropriate View, Add, Edit, Delete, and Admin rights for the role.

To see if a user is already in a role with a specific entity, filter the Roles by user and Entity.
