Administering role security

In this procedure, you first select a role, and then you edit the rights associated with the role.

To administer role security:

Go to the System Admin > Security > Roles.

The Security Roles dashboard opens.

Use More to filter by Role Title, Users that are associated with the role, or Entity Name of the associated Security Entities.

You can also use a wildcard when searching for roles. When setting search parameters, place a % symbol before your search string to make it a wildcard keyword search. Example: %Service Case

In the Security Roles list, open an entry.

The Role form opens showing Title, Description, and Administrative fields. In the Action Center, you see Entities and Users with a counter. The number 0 (zero) indicates there are no records.

If a user needs access to a particular security entity, you can do one of two actions:

Add the user to a role that already contains the required entity by filtering roles by the targeted entity.

Add the security entity to a role that the user is already associated with by filtering roles.

To view a list of users that are assigned to the selected role, in the Action Center, click Users.

The Security Role Users dashboard opens.

To add users to the select role, click Add.

To view the rights for the role, in the Action Center, click Entities.

The Security Role Entities list is displayed with the columns Entity Name, View, Add, Edit, Delete, and Admin.

To add or remove rights for the role, select or clear the options for View, Add, Edit, Delete, and Admin.

To see if a user is already in a role with a specific entity:

Filter the Roles by User and Entity.

When setting search parameters, place a % symbol before your search string to make it a wildcard keyword search. Example: %Service Case
