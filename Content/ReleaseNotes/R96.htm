Enhancements

Enhancements are grouped as Change in Functionality, ClientSpace Premium (ClientSpace with additional modules), Advanced Administration, and General.

Change in Functionality

Case

Enhancement
46928

Email Integration: Password field

When editing the password for an email integration credential, the Password field is now hidden. A new option for changing the password is available in the Action Center as (Change Password). The Password field is only available on adds. See Creating, modifying, or deleting Email Integration credentials.
47111

PrismHR API: Log exceptions

When the queue record fails to import into ClientSpace, we now stamp the error message onto the Queue record for Contact, User, and PayrollBatch. See Import Queue management.

44623

Importing Lookup Values

You can now configure lookup value imports to accept invalid values, which then logs the error for review. A new option for lookup fields, Allow Invalid Lookup Values is now on the Import Map Detail. See Configuring the Import Map Detail and Managing import data.

43740

Client Info Widget: Last Visit Date filter

On the Workspace landing page, the Client Info Widget now shows the Last Visit Date based on Completed visits.
47062

ClientSpace Calendar: External calendar

From the ClientSpace Calendar, for external events created by the logged-in user, you can now view the location, attendees, and any meeting notes. This enhancement provides added convenience – you no longer need to switch to the external calendar application to see these details. See Viewing the ClientSpace Calendar.

ClientSpace Premium

Case

Enhancement
46081

OSHA Dataform: Location OSHA Totals

A new multi-dataform, Location OSHA Totals, is now available. It contains the Total Hours Worked and Average EE Count per year, per location, and is populated by the PrismHR API. This enhancement eliminates the need to add this data from PrismHR manually. See OSHA 300A Report Details.
46085

OSHA 300A Report: Calculate headcount and hours by location

The OSHA 300A report now uses the new dataform Location OSHA Totals to obtain data for headcount and hours by location. A new scheduled process PrismHR Import OSHA Location Data pulls data for the previous year and populates the dataform Location OSHA Totals. See OSHA 300A Report Details.

46036

OSHA: NAICS Code by Location

The Client Location dataform has a new field, Override NAICS. When the Client Location is configured with an Override NAICS code, and the 300A report is run by location, the Override is shown on the report, replacing the Organization default NAICS code. When the Client Location has no Override NAICS code, the code is retrieved from the Org / Client Master. See OSHA 300A Report Details.

42719

Risk Management: Pricing State

The Pricing Policy field now filters based on the Comp Model. This enhancement filters out irrelevant policies to aid in reducing errors when selecting a policy.

47041

Benefit Plan Manager: Main Benefit Group

When creating a benefits batch, if a benefit group of Main does not exist, the system creates a new benefit group named Main and sets it as the default Benefit Group. This enhancement makes pricing benefits easier.

46590

Benefit Plan Manager: Rate Group

The name Plan Tier was changed to Rate Group to match the PrismHR terminology. The name change appears in dataforms, links, methods, and rules.

46439

Benefit Plan Manager: Annual Period

The Offered Benefits Plan dataform was updated to remove the field requirement for the Annual Period field on Add or Edit. The field is still required to export the Offered plan. This enhancement allows you to edit the Offered Benefits Plan without completing the Annual Period field because it may not yet be determined.

46527

Client Profitability Report: Task information

The merge proc PEO_merge_client_profitability now includes Task information. This enhancement enables accurately defining the time spent on client work. For information about the Client Profitability Report, see [%=MyVariables.ProductName%] Profitability Report.

46903

CCMSI Connection

ClientSpace now provides a connection to the CCMSI API (CCMSI assists clients with self-insured property and casualty claims management). Workers' Comp Claim data can now be entered into ClientSpace and exported to CCMSI, saving users time. You must be a CCMSI customer to use the API. To configure the connection, see CCMSI.

Advanced Administration

Case

Enhancement
46508

Dataforms: Field display conditions for tabs

Dynamic field display conditions now support tabs along with fields. This enhancement enables you to display tabs when needed, based on a field value. See Configuring dynamic field display.

44044

Search: Field Display Rule Search

The Field Display Rule Search dashboard now provides Rule Number as a column, displaying the rule ID number. You can search by Rule Number in Quick Search and More search. For details on using Search, see Searching in [%=MyVariables.ProductName%].

44173

Set Field Value On Form

A new rule _SetFieldValueOnForm allows administrators to set a field value on another form to a specific value. The rule is configurable in all forms; however, the TargetForm cannot be a multiform. The FieldValue parameter validates the configuration against the FieldName datatype. This enhancement does not support the time datatype.
46596

Scheduled process: Seed Client Team Role

The scheduled process Seed Client Team Role now deletes all Client Team Role rows before recreating the Client Team Role forms. This enhancement eliminates the previous requirement to delete all records manually before running the scheduled process. See Scheduled processes.

General Enhancements

Case

Enhancement

47175

Attachments: Drag files

You can now drag files to the Attachments pane without the need to use the Upload File dialog box. This enhancement provides a more seamless user experience when adding attachments. See Attachments.

45813

Organizations: Primary Workspace

You can now specify the default workspace for an organization with multiple workspaces. A new field Primary Workspace is available on the Home tab for System Admins. See Adding and editing Organizations.

43622

Uploaded Files Queue: Workspace column on dashboard

When viewing the Uploaded Files Queue, a column for the associated Workspace is now available, which provides clarity when viewing the DocuSign entries. See Viewing document status in the DocuSign process.

46278

Email Admin: Email Template column

The Email Admin dashboard has a new optional column, Email Template. This enhancement aids in troubleshooting issues with the notification by listing which email template sent the notifications. The Email Template column includes a link to the related template, allowing the admin to jump directly to the template from Email Admin. See Email Admin.

44333

Employment dataform: Last Hire Date

The Employment dataform field Hire Date is now named Last Hire Date.

46816

Workflow Channels: Cloneable message for triggered form

If a Workflow Channel is set to unpublished and then the triggered dataform used by the Workflow Channel is changed to not cloneable, a new warning message displays when publishing the channel: Workflow channel cloning is no longer enabled for this triggered form. If you still wish to use this as your triggered form, go to Dataform admin and check the 'Enable WFC Cloning' checkbox. This warning displays each time the Triggered Form is not Workflow Cloneable, and the Workflow Channel changes to Published (Active). See Configuring your workflow, Step 1: Channel.

46867

Lookups

Both the Lookup (core) and Lookup Type Ahead (core) datatypes support basic and advanced lookup filtering in the same way, using either the Filter Field (basic) or the Filter Source (advanced). Additionally, Lookup - Multiple (core) datatypes now support basic filtering using the Filter Field. See the follow-on basic and advanced filtering topics included with Configuring lookups.

47781

Lookups: Info bubbles

The fields Filter Value and Filter Source now have info bubbles for lookup filtering at the field level.

47747

Client Service Case: Notifications

The recipient procedures for Client Service Case Email Templates now include Client Team users, to ensure that notifications are sent to the intended recipients.

46275

Search: Search users by Date Created

In More search, you can now search users by Date Created. For details on using Search, see Searching in [%=MyVariables.ProductName%].

43529

Modules: Category column

In the Modules dashboard, the Category column now shows the decoded name, such as Business Intelligence rather than BI.

45924

Widgets: Trigger conditions

For Widget Layouts secured to a specific security entity, users (including Global Admins) who do not have View rights to that entity are no longer able to view the layout.

44366

Client Master: Pricing Renewal Date

On the Client Master, the field Renewal Date was changed to Pricing Renewal Date. This enhancement provides clarity to what the renewal date references.

44043

PrismHR API: Employee mailing address mapping

Data mapping for the Employee Mailing Address through the PrismHR API is now available. This enhancement provides a correct mailing address for employees. To view the data mapping spreadsheet, see ClientSpace data mapping for the PrismHR API.

 

The data is mapped on the HRPAPI_employee import maps:

  • AltAddress1
  • AltAddress2
  • AltCity
  • AltCounty
  • AltState
  • AltZipCode
  • AltZipSuffix
