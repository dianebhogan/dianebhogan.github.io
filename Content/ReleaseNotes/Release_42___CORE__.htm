Set field from Pattern Business Rule   

Resolution:                                                                                                    

A new Core business rule _SetFieldValueFromPattern has been created that has the ability to extract the first instance of a value from within a text field using a .Net RegEx pattern and set the value of another text field on the same page to this extracted value.    This business rule operates based on the following:

  • The SourceField value will always be treated as text
  • TargetField must be a Text data type
  • TargetField will ALWAYS be overwritten with the results of the pattern match
  • SourceField and TargetField must be on the current dataform and cannot be used on tabbed forms
  • If the pattern match returns no value, the TargetField will be cleared

For help configuring this rule, contact your NetWise account manager

Import Users    - Phase one

Resolution:                                                                                                    

The nightly import process has been enhanced to allow the import of User records into the system either by inserting new records wholesale, or by cloning an existing user record based on the presence of a "CloneAUser" flag on the import file.  Phase one of this process by necessity contains the following restrictions/caveats:

  • Cannot import global admin
  • Department must be an integer (no foreign key lookup at this time)
  • AdHoc Profile, integer (no foreign key lookup at this time)
  • Time Zone, integer (no foreign key lookup at this time)
  • Domain, integer (no foreign key lookup at this time)

User Cloning

  • Cloning a User is only possible during Add
  • 'CloneAUser' and 'CloneFromUser' are fields that manage cloning  
  • 'CloneAUser' indicates a User should be cloned  
  • 'CloneFromUser' is the ImportID of the User to be cloned
  • If 'CloneAUser' is empty (or "0"), no User will be cloned
  • If 'CloneAUser' is set (equal to "1"),  
    • If the import process was unable to locate user by CloneFromUser = User.ImportID, a Hard Error is returned in the Import log file  
    • If 'CloneFromUser' is a valid User, that User will be cloned.  Workspaces, Roles and Modules will be cloned if set in the Import file

Password  

  • If provided, will be encrypted and set, Force Password Change flag will be cleared even if set in the Import Row  
  • If not provided, Force Password Change flag will not cleared, will be set to the value in the Import Row

For more information about how this new functionality works, or for help configuring, contact your NetWise Account Manager

Email Templates   

Resolution:                                                                                                    

The new Email Template architecture has been enhanced to allow notification templates to be configured that run on any dataform save like a template task.  Triggering for this functionality mirrors Link Configuration, including condition expressions.

  • Trigger conditions can be configured based on the associated dataform fields, or from data returned via Server Data Procs.
  • An email template merge has been added to the dataform save architecture.  This allows configuration for custom notifications utilizing field replacement data from a custom stored procedure.
  • Mail recipients of the notification are currently also retrieved from a custom stored procedure
  • Dataform field delete has been enhanced to check if there are template conditions that use the field being deleted.

Form more detailed information, read Configuring Email Template Triggering in ClientSpace.
