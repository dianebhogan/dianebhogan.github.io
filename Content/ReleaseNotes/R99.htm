48561

Client Team Role: Subscriber list

The Client Team Role column displays all roles as a comma-separated list for users with multiple client team roles. Now users are listed once rather than as multiple entries. See Subscribing client team members to cases.
46874

PrismHR API: Payroll Batch - Remote Cutoff Date

The PrismHR Import Ongoing scheduled process now includes Remote Cutoff Date. Remote Cutoff Date is the deadline for reporting payroll and helps the payroll processor monitor the client payroll reporting status. To import this data, configure the API Config parameter GetPayrollBatchCutoffDate. See Payroll Batch dataform, Payroll Batch Import Mappings, and Configuring the API for Import Map Details.

Advanced Administration

Case

Enhancement
48930

DocuSign: Email Templates

Previously the default for aspnet:MaxJsonDeserializerMembers was 1000. Now the default value is 5000. This enhancement allows large numbers of addresses in DocuSign Email Templates.
49046

PrismHR API: User Imports

When retrieving User Details for a User from an Import Queue record, if PrismHR returns an error "User xxxx does not exist," the existing ClientSpace User record is set to Inactive (if it exists). The queue record is set to Completed with the message “User 'userId' cannot be found in Prism, inactivated in ClientSpace (if exists), queue record automatically Completed." See PrismHR API user-related processes for initial and ongoing imports.

48935

PrismHR API: Employee Termination Date

When extracting the Termination Date, if the Employee Status (PrismHR value) begins with T, then set the TerminationDate value in the import file to the Employee/client/statusDate value. If the status is multiple characters (such as T1, T2), ClientSpace only uses the first character, looking for a T to determine the Employee was terminated. See Importing to [%=MyVariables.ProductName%].
