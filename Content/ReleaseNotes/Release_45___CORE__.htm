Release 45 ( CORE )

Product Version: CORE – Release 45 Notes - The following Tasks have been completed/fixed in this release.

Task 36884 - ENHANCEMENT                                                                                                                                   

Standard dataform search pages - text field searches   

Resolution:                                                                                                    


Text fields in the "More" dropdown on the built in multiform search page in NEXT have been enhanced to perform a "starts-with" LIKE search of the field.  Previously these fields required an exact match of the field data for filtered results. 

These changes have been built into the dataform SQL generator, so they will be inherent across all multiform dataforms.

Task 36903                                                                                                                                      

Remove Izenda from Classic Core   

Resolution:                                                                                                    

The Izenda older Ad-Hoc reporting module has been deprecated and replaced with the Business Intelligence module - any module links to the older Izenda Ad-Hoc reporting app have been repointed to open Business Intelligence.  This provides more dynamic Dashboards and a cleaner interface. 

Task 36975 - ENHANCEMENT                                                                                                                                      

Strip out 'image' files from email when attaching and creating case or task   

Resolution:                                                                                                    

The ClientSpace Outlook add-in has been enhanced to remove common signature images.  These images, most commonly social media badges cause clutter within the system and will often be stored many, many times in the database.  The ClientSpace Outlook add-in now strips away any image whose name begins with "Image0" - the most common naming convention for these types of images, leaving other email images intact.

Bypass Workspace Landing Page   

Resolution:                                                                                                    

NEXT has been enhanced to allow customization of workspace behavior when accessing workspaces from a list such as the Workspace Search. 

Any template which contains a workspace creation dataform can be defaulted by setting the UsePrimaryDataform flag to open to this primary dataform when accessed from a list - as Classic currently does, instead of taking you to the workspace landing page.  

The "Go to workspace" jump icon on the right nav menu can then be used to access the workspace landing page.  This setting is not exposed through the User Interface, and will affect all child workspaces of the template to which this setting is applied. 

For more information, or to request this customization for one of your templates, create an Extranet case for your NetWise account manager. 


Task 36904                                                                                                                                      

Large Number of Activities Times out on Activity Search   

Resolution:                                                                                                    

The Activities code has been optimized to resolve performance issues in the Activities right Nav widget when there are large numbers (over 100,000) activities attached to a single organization.

Task 36934

Resolution:                                                                                                    

Resolved an issue where custom links configured as "Link 2" were not displaying properly. 


Task 36976                                                                                                                                     

Badges with inactive records   

Resolution:                                                                                                    

Resolved an issue in NEXT where the new dataform badge functionality would not allow you to access Inactive records unless there was more than one active record. 

Badges now work as follows:

  • If there are no linked dataforms, clicking link will place you in "ADD" mode
  • If there is only one linked dataform, clicking the link will open that dataform - clicking + will put you in "ADD" mode.
  • More than one dataform, active linked dataforms the badge presents a count of the Active dataforms ie: 3
  • More than one Active with Inactive records as well - the badge presents a count of the active with an asterisk denoting the presence of inactive records as well ie: 3*
  • One or more inactive with No Active records - Badge presents zero with asterisk ie: 0*
