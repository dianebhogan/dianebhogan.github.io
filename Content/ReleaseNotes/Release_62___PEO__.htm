Release 62 ( PEO )

Sprint Began: 11/6/2017     Sprint Ended: 12/5/2017


The following items have been completed in this release:


Convert Case Category fields to be Metadata - ENHANCEMENT

The Case Category dataform has been converted to act as metadata on the Case Type lookup. The Client Service Case Category Metadata form is still available for configuration through the admin workspace.


Date Range filters not working for active reports in Next

Resolved an issue where when running an active report, that has a Date Range field on the filter page, the Start/End parameters weren't being sent to the report stored procedure.


Policy type ahead does not display inactive policies

The wc policy typeahead data type has been enhanced to respect a ShowInactive:true database setting.  Setting this in field properties in the database will allow inactive policies to display in the WC Policy data type fields. 

Because this is set at the database level this is not a configurable option. 

If you have a policy field that you would like to display Inactive policies, please contact your Account manager.


Benefit Plan DataType Enhancement - ENHANCEMENT

Plan Code is now part of the Benefit Plan Decode in the Benefit Plan DataType. It now displays as: Plan Title - Plan Category - Effective Date (PlanCode) Ex. Aetna HMO - Vision - 01/01/2012 (PC1). This allows you to use the % wildcard character to filter the drop down by Plan Code to more quickly find what you are looking for.


Convert Headers to Widget Layouts - ENHANCEMENT

All existing dataform headers in ClientSpace PEO have been converted to Widget layouts, and the old, hardcoded headers have been deprecated. This allows for greater flexibility and customizability of form headers.


PrismHR API - User Import ENHANCEMENT

New functionality has been added to allow ClientSpace user creation using the users from PrismHR.  The "PrismHR Import Users" scheduled process can be used to generate an HRPAPI_initial_users CSV file which will be read by the associated import configuration.  When the file is imported by the system it will:

  • Automatically create CS User.  This user will be cloned from a user that can be specified either in the import file, or as a default in the import configuration
  • Automatically create a Third party login record for each new  user at the time of creation


PrismHR API - Contact Import  - ENHANCEMENT

ClientSpace has been enhanced with an ongoing contact import via the PrismHR API. Changes and additions made to contacts in PrismHR will synchronize to ClientSpace.


SSO from PrismHR to ClientSpace - ENHANCEMENT

ClientSpace has been updated to use the new TSSO standard for single sign on authentication from PrismHR to ClientSpace.  This uses a central connection point (Extranet) to determine the appropriate install, making configuration easier.  For detailed documentation on how to configure this integration refer to Implementing SSO from PrismHR to ClientSpace.


SSO from ClientSpace to PHR - links -  ENHANCEMENT

Links have been added to the Workspace Landing Page and the Client Service case in ClientSpace which log you directly into PrismHR if the ClientNumber of the workspace from which you click the link exists in PrismHR and if you have a CS-PrismHR  third party user record configured.


Overall Breakdown not working 

Resolved an issue where the Modal dialog version of the Overall Breakdown would not work properly. Modal dialog now matches the Overall Breakdown data recorded in the Pricing Widget on the Workspace Landing Page and ties to data in the Pricing Matrix on the pricing console


Secure Pricing Console Add State Code button - ENHANCEMENT

Pricing Console security has been enhanced in NEXT to respect the biz_pricing_submitted_batch entity when securing the "Add States and Codes" button on the pricing console. 

Users without any rights to this entity will be able to add States and Codes to a pricing batch until the RFP batch in the workspace has been submitted.  After submission Only users with at least View rights to the biz_pricing_submitted_batch entity will be able to add States and Codes to pricing.


PrismHR API Client dates ENHANCEMENT

The export client function of the PrismHR API has been enhanced to set the PrismHR Contract Date to the Client Master Contract Effective Date and the PrismHR Status date to the Client Master Status date. If no Client Master Contract Effective date is available, the PrismHR Contract Date is set to the Client Master Status Date


Client Alerts widget does not collapse - leaves whitespace when empty

In a previous release the Client Alert banner was "widgetized" removing it from the dataform page header but an issue was found if the workspace had no client alerts.  The Client Alerts banner will now always display if it is included in the widget layout - if there are no Client Alerts, the banner has a green background and displays the text "There are no client alerts at this time."  When there are client alerts on the client master the banner is red and displays the text of the Client Master Client Alert field.
