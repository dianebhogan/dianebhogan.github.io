Release 48 ( PEO )

Product Version: PEO – Release 48 Notes - The following Tasks have been completed/fixed in this release.


Task 37259 - ENHANCEMENT                                                                                                                                     

HE/SE Rules by WC Policy Type   

Resolution:                                                                                                    

Two new methods have been created that allow you to create soft errors based on contract type and Dataform table and field requirements.   

  • "SE_RequiredTablePerContractTypeEngine" allows you to configure a soft error based on the Contract Type on the Client Master which requires a specific data form to be present in the workspace.
  • "SE_RequiredFieldPerContractTypeEngine"  allows you to configure a soft error based on the Contract Type on the Client Master which requires a specific field on a specified data form to be completed in the workspace. (accepts a comma delimited list of fields on the form)  

 If you would like to request help in configuring rules using these new methods, please create an extranet case for your NetWise account manager.

Resolution:

Task 37253 - ENHANCEMENT                                                                                                                                      

Admin Profit Based on Min Admin not Priced   

Resolution:                                                                                                    

The pricing system has been enhanced to take Admin profit minimums into account during the pricing process - these enhancements include:

PEO DEFAULT INFO:

A new checkbox has been added on the PEO Default form called "Include Minimum Admin Supplement in Pricing".  If checked then anywhere that Admin is being used to determine Gross Profit, Commissions, and so forth the calculations have been changed to to take in account the Minimum Admin Supplement. If checked the application will also show the Minimum Admin Supplement on the Profit Grid. If not checked the system would work as it had before and does not show on the Profit Grid.

PRICING BATCH:

A new field has been added on Pricing Batch called "Minimum Admin Per Account" which defaults to the Minimum Admin Per Account from the Default Admin table. This field is read-only and if not filled in on the Default Admin table then set to $0.00 on the Pricing Batch.

MINIMUM ADMIN LOGIC:

The Pricing Batch Minimum Admin Per Head previously defaulted from the Default Admin table on the greater of the Min Admin Per Account or the Min Admin Per Head. This defaulting logic has been altered to default directly from the Min Admin Per Head field on the Default Admin. If not filled in on the Default Admin table then set to $0.00 on the Pricing Batch.

A new rule has been created named "HE_ValidateSubmit_PerHeadMinAdmin". This rule is triggered on submit to ensure the Pricing Batch.Minimum Admin Per Head field is greater or equal to the priced admin. This rule respects the Override Validations checkbox on the batch which will circumvent this hard error check.

The existing "HE_ValidateSubmit_PerAccountAdmin" rule that is triggered on submit has been modified to ensure the Pricing Batch Minimum Admin Per Account field is greater than or equal to the priced admin. This rule has been renamed "HE_ValidateSubmit_PerAcctMinAdmin".

DISCOUNT THRESHOLD LOGIC:

Previously, the "HE_ValidateSubmit_PerHeadAdmin" rule worked solely with the Discount Threshold records (role based discounting for current user) on Submit to ensure the User who is submitting the RFP is within acceptable discounting guidelines when it comes to Admin, WC, and SUTA. This rule has been renamed to "HE_ValidateSubmit_DiscountThresholds" to more clearly identify its purpose and avoid confusion with the other above rules. The logic on this rule has not changed.

PROFIT CALCULATION LOGIC:

The two minimum admin fields on the Pricing Batch field are used for calculating the Admin Profit shown on the bottom of the Pricing Console and update the Admin portion of the profit calculation to take the greater of the priced Admin or the greater of the Minimum Admin per Account (new field on Pricing Batch) or Minimum Admin per Head (existing field on Pricing Batch). Previously only the Classic application would display an additional line item below the Admin Profit named "Additional Profit from Minimum" when clicking the Gross Profit links.  This has been added in ClientSpace NEXT, but only if the ”Include Minimum Admin Supplement in Pricing” on the PEO Default is checked. This value has been relabed as “Minimum Admin Supplement”.

Previously, the Gross Profit (value shown in the links) did not include the "Additional Profit from Minimum". This has been fixed to include the "Additional Profit from Minimum", but only if the ”Include Minimum Admin Supplement in Pricing” on the PEO Default is checked.

A new field has been added to the Pricing Batch Profit Area called "Minimum Admin Supplement". The Miniumum Admin Supplement will store the equivalent value from the Profit Grid", but only if the ”Include Minimum Admin Supplement in Pricing” on the PEO Default is checked. If not including minimum Admin Supplement in Pricing this amount will be $0.00.

Commission calculations that are dependent on Admin have been updated to take into account these new changes.
