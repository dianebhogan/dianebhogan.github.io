Release 74 Notes

R74 Sprint: 11/1/2018 to 11/30/2018. To learn about product features that you may not recognize, contact your Customer Success Manager.

Enhancements

Enhancement

Modify document search to use the new TotalRecords design

The document search has been modified to use a more efficient method of generating total record counts for returned search data.

Ability to move or insert a task at another level

Workflow channel configuration has been enhanced with the ability to move dependent workflow items through a simple drag and drop interface. The intelligent drag and drop functionality is aware of the parent item type for the child workflow object that you are attempting to move and will only allow this item to be moved to a workflow parent of the same type. Because of this, triggering conditions of the parent will always match that of the child which is being moved.

Multi-tenant email template administration

Email template administration has been enhanced. A Workspace Templates field was added to email template configuration that allows you to configure a workflow channel to only be triggered when the executing dataform was created from one of the selected Workspace Templates. This enhancement allows for greater control when triggering a specific email template. The Workspace Templates field is a multi-select, which allows you to apply email templates across multiple workspace templates. If no specific template(s) are selected, the Workspace Templates field will default to All Templates.

Email templates pipelines

Email templates have been enhanced to support multiple pipelines. The Pipeline selector has been changed to a multi-select, allowing you to create a single email template that can apply to multiple execution pipelines, eliminating the need to create separate email templates for dataform saves and imports for example. 

Multi-tenant Workflow Channel administration

Workflow channel administration has been enhanced. A Workspace Templates field was added to workflow channels configuration that allows you to configure a workflow channel to only be triggered when the executing dataform was created from one of the selected workspace templates. This enhancement provides greater control when triggering a specific workflow channel. The Workspace Templates field is a multi-select, which allows you to apply a workflow channel across multiple workspace templates. If no specific template(s) are selected, the Workspace Templates field will default to All Templates.

Multi-tenant business rule administration

The business rules engine has been enhanced to support multi-tenancy. A Workspace Templates field was added to rules configuration that allows you to configure rules to only be triggered when the executing dataform was created from one of the selected workspace templates. If no specific template(s) are selected, the Workspace Templates field will default to All Templates.

Forgotten password message for expired user accounts

A previous update, which stopped emails from being sent to expired user accounts caused an issue for users with accounts that automatically expired through inactivity. The user would receive a success message that an email had been sent, but never actually received the email. The system has been updated, so that expired or inactive users will receive a message that there is an issue with their account and they should contact the system administrator.

Convert dataform administration - Finalized

Dataform properties administration conversion, which allows this function to work with our cross-browser NEXT application, has been completed. This development does not include field administration, which will be addressed in an upcoming release. For more information about configuring dataform properties, read the dataform properties section of Dataforms - Workspace Building Blocks.

Third party users - Finalized

Updated the third party application administration to resolve some search issues and make the user interface more functional.

tblEmailRecipient deadlock errors

The code that generates email recipients was streamlined to minimize the potential for deadlock errors when there are simultaneous attempts to access a record.

35351

 Hourly rate workflow channel case creating too often Resolved an issue where masked numeric fields such as money or decimal fields would interpret an imported value of the same amount, but a different number of decimal places as having been changed, when in fact no change had taken place. For example, the dataform stored the value 123.45 (2 decimal place mask), and the import contained the value 123.4511. This issue would cause events such as notifications to trigger every time the data was imported as if it was being changed each time. The system has been updated to apply the mask during import, thereby avoiding the false changed flag for these types of records.
