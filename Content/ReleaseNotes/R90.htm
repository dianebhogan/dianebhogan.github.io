Enhancements

Hover on the icons to discover the level of product enhancements:

Case

Level of change

Enhancement

20347

Hard Error and Soft Error rules to prevent duplicate records

Hard and Soft errors were added for methods _HE_PreventDuplicateRecords and _SE_PreventDuplicateRecords to accept a field parameter and validate that it is unique. See Business Rule Methods.

31131

 

System level merges

The merge system now prevents users from deleting System merges and presents the message “System merges may not be deleted.”

32109

 

View list and User menu

On the topmost banner bar, the View list now automatically closes when you click outside the list. The User menu now automatically closes when you click outside the menu.
40598  

Loss History dataform: Gross Payroll field

A Gross Payroll field has been added to the Loss History dataform.

40837

 

Operator in Custom Links

Custom link configuration now includes a full set of datatype specific operators when creating link display conditions. Links previously only supported an equal (=) condition and now function as condition expressions elsewhere in ClientSpace such as Email Templates and Workflow Channels. See Configuring link display conditions.
40846

VAO dates and Claim Financials

The Claim Financials system now updates the most recently created record when imported financial records have the same VAO date as an existing record. The Current Record option is selected, as the logic automatically enables the option as read-only. See Claim financial form and calculation rules.

41089

Create modules for single dataforms

The list column data is now available in Dataform Admin for single and multi dataforms, making it easier to create modules for single dataforms. See Defining dataform multiform list column headers and Making a Dataform open as a Module.

41669

Number of Dataform tabs

The number of dataform tabs is now limited to 5 for all new tab configurations. This change helps to avoid performance issues for dataforms with a high number of tabs that contain many fields. For dataforms previously configured with more than 5 tabs, the tab limit is set to the current number of tabs so that existing configurations are not affected. An error message displays to notify users that the tab limit cannot be exceeded. See Adding dataform tabs.

41773

Suppress Location Name on OSHA 300 and 300A

Enhanced the OSHA 300 and 300A reports by adding a Hide Location Name option to the reports filter pages. When selected, the reports are produced with the /LocationName removed from the Establishment Name field on the report where it would normally display after Client Name.

41815

 

Search by First Name for Calendar event attendees

The Calendar now allows you to search for calendar events by both the first and last names of a calendar attendee. See Using the [%=MyVariables.ProductName%] Calendar.

42176

 

Processing delete events for I-9 data

The PrismHR API was enhanced to include delete events for I-9 records from PrismHR. When a delete event is processed, the associated ClientSpace record is set to a status of Deleted, and the record is inactivated.

42263

 

Time Tracking categories

The default Time Tracking Category is now set on all Case Category Metadata records where they were not previously set. The following time tracker categories were created if they did not already exist: Accounting, Admin, Benefits, Client Services, Employee Portal, Human Resources, Implementation, IT, Legal, Payroll, Risk, Sales, Underwriting. For each time tracking category, the default category group is set to Internal. This will help ensure new time records have category filled in automatically.

42283

Style and State type field display rules

Style and State type field display rules can now be configured on the same field with the same triggering conditions. The enhancement is based on the following rules:

  • As a global admin, both Style and State rules can be added with the same triggers to a field in ClientSpace simultaneously.
  • State rules validate against State rules.
  • Style rules validate against other Style rules.

42463

 

Workflow Channels: Cloning single dataforms

The Workflow Channel dataform cloning system was enhanced to allow single dataforms to be cloned once per Workspace. This allows single forms in a Workspace that are not automatically created as part of Workspace creation to be cloned as part of a workflow.

  • When a single dataform is selected as the Triggering Form for a Workflow Channel item, the Only trigger this workflow once option on the Conditions step is selected and marked read-only.
  • When a single dataform is selected as the Triggering Form for a Workflow Channel item, and the form already exists in the Workspace through business logic (for example, Client Master, Client Team), that dataform is not duplicated. See Step 4: Conditions (optional).

42516

Dataform Admin: Show Create and Created By

Dataform and Field Admin now provide audit information. The following fields are available:

  • Date Created
  • Date Updated
  • Created By
  • Updated By

See Adding a dataform.

42632

 

Removed Enterprise Library: SqlDatabaseExtensions and EnterpriseLibraryRepository

Core systems were refactored to use a newer data access function. These systems include:

  • SqlDatabaseExtensions and EnterpriseLibraryRepository
  • ClientSpace.Repository.SqlClient
  • ClientSpace.Services.DocumentService
  • ClientSpace.Services.CalendarSync

42666

 

DocuSign Email Template Replace Fields

DocuSign Email Templates now support field replacement functionality. See Step 2: Content.

42715

Offered Benefits Plan dataform

The Offered Benefits Plan dataform was updated. The Category field was moved to the top of the form immediately above the Benefit Plan field, which it filters.

43030

 

PrismHR API Version 1.20

All PrismHR API data access methods were converted to use the REST API structure from the previous SOAP API structure, which required a refresh of the WSDL code whenever a change was made to the API. This update is a required change to prepare for the PrismHR API version 1.20 update coming soon.

43077

 

Report/Merge filter pages

Reports and merges that use a filter page now support the Workspace landing page.

43138

Benefit Plan Groups functionality

The Apply Plan Group link on the Benefits Batch dataform now includes the following items:

  • In the Apply Plan Group dialog, an option for Classification was added with a list of the existing Active Classification records in the workspace, as well as an option for All Classes.
  • If All Classes is selected, the Classification list disables and clears.
  • Offered Benefit Plans from the selected Benefit Plan Group are created with the indicated Class populated or for all classes if selected.
  • If All Classes is cleared and Classification is blank, the offered plans are created without a classification.
  • The logic that creates the Offered Benefit Plans was updated to follow the lookup value defaults. (Example: Contribution Method field is populated with the default lookup value.)

43164

 

Workflow Channel and Email Templates

Consolidated the underlying table structure that stores Email Template and Workflow Channel trigger conditions. This enhancement standardizes data collection and reduces database table duplication.

43323

 

Removed Classic Code

In our continued effort to sunset our classic code, classic CRM functionality related to Sales activity management was removed from the application.

43425

 

Employee Portal How can we help widget on self-hosted sites

Self-hosted PrismHR sites can now access the Employee Portal How can we help? widget.

43429

 

Pricing Code Pay Frequency field

The Pricing Code Pay Frequency (decode) was added to the merge stored procedure peo_merge_excel_companydata.

43515

 

PrismHR API v1.20 Conversion: Authentication method change

Updated the PrismHRAPI login methodology to include both the method used in the previous version 1.18 as well as the new version 1.20.

43659

 

PrismHR Imports

During the Import process, a client number check is now performed. If there is no matching client number in ClientSpace, an error is logged in the import log, and a queue record is not created. See Importing to [%=MyVariables.ProductName%].

43155

 

Set Business Type rule deleting Business Type data on IndustryCode Imports

The Set Business Type rule on the Client Master has been updated. The update resolves an issue where data was removed from the Business Type field when the Alt Industry Code (SIC Code on Client Master. On some sites, this could be the NAICS Code) was updated through the import, and no metadata was configured on the related Industry Code.

Defensive code was added to:

  • Validate whether there is a related Metadata record to the Industry Code and, if not, prevent the rule from updating the Business Type.
  • Validate that the related Industry Code Metadata record Business Type is not blank, and prevent the rule from updating the Business Type if blank.
  • Prevent the rule from running if the Business Type is marked Dirty when the rule is triggered.
