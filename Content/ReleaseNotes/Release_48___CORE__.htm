Outgoing Email Template   

Resolution:                                                                                                    

Notification of email activities has been enhanced to utlilize Email Templates if one is properly configured. Template configuration should be as follows:

  • Code: CRMNOTE
  • Template Type: Not Triggered
  • Subject: Configurable text - allows the use of one replace field - 
    • {CRMNoteSubject} the text from the CRM Note Subject field
  • Body: Configurable text - allows the use of three replace fields   
    • {CRMNoteBody} the text from the CRM Note Body field   
    • {CRMNoteEmailSignature}   
    • {CRMNoteSentTo}
  • From Email Override: The email will normally be sent from the current user.  If that User does not have an email address (unlikely), the From Email Override will be used
  • If the template contains only the above replace fields (and no other Html or text), the outgoing email will be identical to the email sent prior to this change, with the exception that it will be an Html email, not plain text (primary difference being the line breaks)
  • When saving a CRM Note of type Email:  
    • The note will be saved to the system (no changes)  
    • If email is to be sent, a test for a published "CRMNOTE" Email Template is made  
    • If the Email Template cannot be located, normal Email Notification will be issued, as it did before this change  
    • If the Email Template is located, the Subject and Body replace fields will be populated from the CRM Note form  
    • Email Notification will be issued, with attachments if applicable


Task 37295    - ENHANCEMENT                                                                                                          

Docusign Scheduled Process   

Resolution:                                                                                                    

Add an 'IsSystem' field to the scheduled process table.  If this value is true, the system will disable all key scheduled process detail page fields to prevent unauthorized modifications to those fields.  Changes to those field values will have to be made by NetWise.  If you find you have a scheduled process that is marked as a system process but requires configuration - create an extranet case to request the changes.

Next Enhancements   

Resolution:                                                                                                    

ClientSpace NEXT has been enhanced so that the dataform header (black bar at top) will display Dataform Name : Decode - this decode can either be a custom decode function, such as is used on the employee dataform to display employee name, or if no decode is configured the system will default to  Dataform Name : Default Attribute (for the dataform you are on). Task 37091  - ENHANCEMENT                                                                                                                                        

Marketo Setup - Phase 1  

Resolution:                                                                                                    

ClientSpace has been enhanced with a new API integration to Marketo. 

In Phase one of this integration we will be creating Marketo Lead records by pushing contact information from ClientSpace into the Marketo system. 

These ClientSpace contact records are then stamped with the Marketo ID allowing information synchronization from ClientSpace to Marketo via a scheduled process. 

For more information on Marketo and implementing this in your ClientSpace installation, create an Extranet Case for your NetWise Account Manager.
