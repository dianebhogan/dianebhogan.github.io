Release 52 ( PEO )

Sprint Began: 4/17/2017     Sprint Ended: 5/5/2017


The following items have been completed in this release:


Secured Case Types   ENHANCEMENT

Case Type security in NEXT has been enhanced to be more discrete - if the "Secure" checkbox on the Case Type dataform is checked the system will automatically generate a security entity with the format of CaseType_TypeTitle with TypeTitle being the title of the Case Type in question. Unchecking the Secure checkbox will delete the security entity, removing it from all roles in which it has been applied.

View | Add | Edit rights on the case type entity exhibit the same behavior as other role security -

  • If a user does not have View rights to the secured case type then cases of this type will not be visible in the list and the case type will not be an available option when creating cases.
  • Giving View rights will allow the user to view cases of this type if standard case type security would allow them to view these cases. View rights are necessary if Add and Edit rights are to be added to the role.
  • Giving Add rights will allow the user to add new cases of this type, but not allow them to edit existing cases of this type.
  • Giving Edit rights will allow the user to edit existing cases of this type, but not allow them to Add new cases of this type.

Enhance Client Team Widget on WSLP (NEXT)

The Workspace Landing Page in NEXT has been enhanced with a new Configurable Widget for the Client Team.

This Widget supports up to 10 Client team member fields, displaying the Sales Rep, Payroll Specialist, Benefits specialist and Human Resource consultant by default. Other team members (up to the limit of 10) may be configured to display when the panel is expanded. For configuration instructions see "Configuring the Client Team Widget". 

Display Time on Discussion Messages (NEXT)

The Case discussion time stamps have been enhanced to display based on time stamp which is stored in the threads, displaying time in the time zone selected under App Settings | Miscellaneous | Time Zone Name.

Add Button on Ext Client Service Case Dashboard (NEXT)

Resolved an issue where external users would have access to an Add button without Add Rights on the External Client Service Case dashboard. Previously clicking "Add" for these users would cause a spinning load meter and produce an error. Users without add rights now do not see the add button.

Widgetize the Client Information section of the Workspace Landing Page (NEXT)

The Client Info panel on the Client Workspace Landing Page (Client or Terminated status) has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

The Reports | Case Detail button in the case form results in an error (NEXT)

Fixed an issue in NEXT where the Case Detail report from the from the Client Service Case would throw an error when run from NEXT stating "This link type is not supported at this time." Link now produces the case detail report.

Enhance Employee Info Widget on WSLP (NEXT)

The Employee Info panel on the Client Workspace Landing Page (Client or Terminated status) has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

Enhance Payroll Info Widget on WSLP (NEXT)

The Payroll Info panel on the Client Workspace Landing Page (Client or Terminated status) has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

Enhance Case Search on WSLP to Widget (NEXT)

The workspace landing page in NEXT has been enhanced for workspaces in Client, Terminated or Implementation status.

The case panel has been re-engineered as a widget to allow for future customization. Drill down functionality on the case widget from a Workspace in Client or Terminated status will take you to the Client Service Case multiform search for that workspace.

Drill down functionality on the case widget from a workspace in Implementation status will take you to the Case Search dashboard

Pricing Console locking

The pricing console has been enhanced to set the PricingBatch.Reprice flag during Quick Create to "true", essentially locking pricing while the Quick Create save is happening. This is to eliminate unexpected behavior that can occur during a long running Quick Create where the user attempts to refresh the page or change pricing while the save is occurring,

After Quick Create save is complete the Reprice flag is cleared.

Client Team Role Table not Updating

Resolved an issue where changing Client Team members from certain areas in the system was not appropriately updating the Client Team Roles dataform. System will now appropriately Add/Update team roles as Client Team Members change.
