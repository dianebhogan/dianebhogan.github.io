Task 33504   - Enhancement                                                                                                                                     

COI Holder Scope Logic (Core)

Resolution:                                                                                                    

The system has been modified to enhance the transition when moving dataforms within the application.  When a dataform is 'moved' to a new workspace, it will still be visible to the user in the case of an 'Apply' or if edited from a multi-form list.  The record can still be opened, edited, saved, etc.  If the list is refreshed, however the moved record will no longer appear.

If however, when the record is moved, the current user does not have permission to access the new workspace, they will receive a pwDataformMoveNoPermissionException message, indicating that the requested data is no longer in this workspace and that they do not have permission to access the new workspace.

This change was implemented for COI Certificate holders, but applies to any dataform by changing the 'Workspace' field in the admin fieldset.

File attachments are also moved with the record.

Task 33455   - Enhancement                                                                                                                          Click here for the Video

Auto Close Tasks rules

Resolution:                                                                                                    

A new Core business rule has been created:  _TaskAutoComplete.

This can be configured to trigger on HdrActions and dataform field values to close tasks that have been triggered from specified template tasks.  The "Entire Workspace" parameter enables users to limit the closing of the tasks to only tasks attached to the triggering record or any tasks in the workspace.  

The "Notification" parameter determines whether the completion will trigger notifications or not.

You are also able to enter a comma delimited string of template task ID's and any tasks that have been triggered from those template tasks will be completed and Archived, as if you used the Complete Link on the task itself.

Task 33246                                                                                                                                       

Scheduled Process Exception

Resolution:                                                                                                    

The system has been changed it so that if the 'Generate Scheduled Tasks' scheduled process fails at one task, it keeps going.  The exception log has also been enhanced to provide more detail, logging the task that failed.  This change lessens the impact of a single failed process and provides insight into which process requires attention.

Task 33454                                                                                                                                       

Modify Merge Error

Resolution:                                                                                                    

Modified the error thrown when there is not data returned while running a merge to provide more useful information for troubleshooting.

Task 33599                                                                                                                                      

Izenda db schema is stale

Resolution:                                                                                                    

Fixed a bug in the Izenda Reporting module that would retain "stale" dataform data so that when dataform fields were added or removed they would in some instances not display properly in the Izenda reporting module, leading to errors.

Task 33647,33650                                                                                                                                       

Data access objects not compatible with new transaction implementation

Resolution:                                                                                                    

The system has been modified to utilize a more efficient transaction handling model, resulting in improved performance of some modules.

Task 33548                                                                                                                                      

tblCustomLinkConfiguration bug - DisplayActionData = NULL   

Resolution:                                                                                                    

Resolved an issue that allowed for blank values to get passed to a scripted Header Link, causing the error "Conversion from type 'DBNull' to type 'String' is not valid" to occur in the exception logs each time a header was opened with the artificially misconfigured link DisplayActionData.
