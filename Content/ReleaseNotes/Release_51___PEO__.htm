peo_pricingcode_reprice bug   

Resolution:                                                                                                    

A mask was added to the CompAdjustmentOverride field to allow decimal(17, 6).  This prevents Next from attempting to round the value to (18,4) which could cause the field to appear dirty.

HE Business Rule for Location Codes   

Resolution:                                                                                                    

A new Hard Error business rule has been created on the Client Master which, on save checks the locations associated with that workspace to insure the location code(s) are appropriately set.  If configured with "CheckPrimaryOnly" checked this rule will only check the Primary location for that workspace.  If unchecked, all of the locations are required to have a location code set.  It should be noted that this is an optional rule, which by default in inactive, and may be activated through Dataform Admin | Manage Rules.

Resolution:                                                                                                    

The Payroll Info section of the Workspace landing page in next has been re-factored.  Annualized payroll has been re-calculated to account for situations where there is less than 90 days of recorded payroll in order to increase the factor used as a multiplier when generating the number for annualized gross payroll.

Batch Repricing   

Resolution:                                                                                                    

Client Pricing has been enhanced with a "Batch In Reprice" function which locks the pricing batch at the beginning of the re-price process so if multiple users attempt to re-price the same deal at the same time the system will present an error "This deal is in the process of being repriced by another user.  Changes are not allowed at this time.  Please try again later."  Once re-pricing calculations are complete and the current pricing saved, this lock is released, allowing the deal to again be re-priced.

Beef Up Case Type Ahead

Resolution:                                                                                                    

In ClientSpace NEXT the decode SQL which returns attached case information has been updated to return "Case # #### : Subject of Case" (with #### being replaced with the actual case number) to make it easier to determine which case is being selected when linking to a case from another dataform or record.  Previously the system would merely return "Case# ####".

LOA Widget   

Resolution:                                                                                                    

The Employee Leave of Absence form has been enhanced with a widget in the header of the form containing information for the related employee. This information consists of:

Set Policy Comp Code ImportID on Policy Clone   

Resolution:                                                                                                    

The Policy renewal process has been enhanced to set the ImportID of the cloned policy comp code records on creation.

The ImportID is set to State-Code-pkPolicyID. where State is the two character Pricing Comp Code State, Code is the Admin Comp Code Master CompCode, and pkPolicyID is the primary key of the new parent policy.
This ImportID will then need to be used on any Carrier provided policy rates that are imported to insure the policy comp code records are updated and duplicate records not generated.
