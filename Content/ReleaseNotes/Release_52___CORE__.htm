Release 52 ( CORE )

Sprint Began: 4/17/2017     Sprint Ended: 5/5/2017


The following items have been completed in this release:


CRM Activity Summary Feed (NEXT) ENHANCEMENT - Video coming soon...

A client activity summary may has been added to the Organization and Contact search dashboards in NEXT - this allows you to see a summarized list of activities from either the dashboard list, or the Activities Panel.

You can also filter activities for specific workspaces or assigned users.

Clicking the individual summary items will display more information in a preview pane to the right.

ClientSpace Branding (NEXT) ENHANCEMENT- Video coming soon...

The app settings in ClientSpace next have been enhanced to allow configuration of custom graphics.  An App Setting checkbox has been added to "Enable Custom Branding"

When this is checked, the system will display four File Upload fields. The description for these fields provides the recommended sizes:

  • Logo
  • Logo_Small
  • Login_Logo
  • Login_Banner

When "Enable Custom Branding" is checked:

  • The ClientSpace logo (csLogo) above the left nav changes to the client's uploaded image.
  • The ClientSpace logo (csLogoSmall) above the left nav changes to the client's uploaded image.
  • The ClientSpace login logo changes to the client's uploaded image.
  • The ClientSpace login logo_banner changes to the client's login_banner that is uploaded.

!Note:  Graphics are loaded on application pool start of the ClientSpace site - after uploading and saving your custom graphics you will need to create a case for you NetWise Account manager to restart the application pool on your site before the new graphics will be visible.

DocuSign - Void Rule   ENHANCEMENT

The DocuSign process has been enhanced to better handle void documents, allowing the user to void a document that has already begun processing through the DocuSign system.  This enhancement includes:

  • A new biz rule (core) _VoidDocuSignDocument which updates the Uploaded Files Queue record status to 'voided'. The scheduled process "DocuSign Request Signature" will void the envelope(s) in DocuSign.  The Scheduled process "DocuSign Get Docs Status" will retrieve any voided documents and attach to the dataform if configured to attach (otherwise the voided document will be attached only to the Uploaded Files Queue record)
  • A new soft error (core) _SE_VoidDocuSignDocument, which warns user that documents (envelopes) will be voided.  This rule will also identity the quantity of 'envelopes' that are affected.

Both rules are configurable with the following parameters:

  • DataformOnly (checkbox).  When checked, will only void envelopes (and thereby documents) associated with the Dataform on which the rule is configured.  When unchecked, will void all envelopes in the Workspace associated with the Dataform on which the rule is configured.
  • AllowVoidStatus (comma delimited string).  Provide the statuses of Uploaded Files Queue records that can be voided.  See the 'Signature Status' lookup group for valid statuses and use the CODE in this parameter, separated by comma, with no spaces.

Email Template Configuration Errors

We have enhanced the Email Template configuration so that on save of the configuration or when retrieving the merge field information from the custom stored procedure,  the system will perform basic validation of the stored procedure to insure it can be run from the configured triggering dataform. If the merge procedure cannot be triggered properly from the dataform the user will receive an error, stating the issue.

Setup Table Relationships

 A scheduled process was created to seed the foreign key relationships between dataforms linked with a parent form field, This will help when creating business intelligence reports by automatically recognizing and populating the relationships between tables when choosing a data source.

Filter Fields by Field Label A-Z when adding to an Import Mapping   ENHANCEMENT

Import configuration has been updated to sort dataform fields alphabetically when adding new fields to the import, making field selection easier.

Update Manage Rules Admin to search by Datafrom Name

Resolved an issue in Rules configuration where you had to use Table Name (gen_ClientMaster) instead of dataform name (Client Master) as in other configuration dialogs. Rules management now works like other configuration modules.

Activity Summary (Next)   ENHANCEMENT

The Activity summary has been enhanced, allowing it to be toggled on and off by the user.  When hidden, the Preview Panel can be used to scroll through the list of summary activities in the same order as they would appear in the summary list.
