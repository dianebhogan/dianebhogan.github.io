Release 42 ( NEXT )

  • Product Version: NEXT – Release 42 Notes - The following Tasks have been completed/fixed in this release.

Task 36034   - ENHANCEMENT                                                                                                                                    

Calendar Multi-User Selector Search Enhancements    

Resolution:                                                                                                    

The "Select Users" page presented after clicking the magnifying glass on the Calendar Multi-User Selector has been enhanced with the following:

  • A drop down has been added for department to filter the list of users.  (we also do this on Tasks/Index)
  • The email search criteria has been added to the QuickSearch as a % search User.
  • Company  has been added to the QuickSearch  as a "starts with" search
  • Company, email address, and last name has been added to the More drop down.  

Task 36125                                                                                                                                      

Calendar Detail Invitee selector includes all contacts for associated Org/Workspace    

Resolution:                                                                                                    

Resolved an issue where The Calendar Invitee search was including any contact associated with the Calendars Workspace/Org regardless of whether the contact matches the other search criteria.  Contact inclusion now respects all filter criteria.

Task 36131                                                                                                                                     

Integer datatype bug   

Resolution:                                                                                                    

Resolved and issue with the integer datatype where it would not allow zeros or negative number. Zeros would result in the field displaying a capital D, and negative numbers get stripped of the - sign.  Integer datatype now works properly in NEXT. 

Task 36148                                                                                                                                      

Browser Crash in Internet Explorer   

Resolution:                                                                                                    

Resolved an issue in NEXT where when making a change to a dataform, if Apply is hit, then one of the top navigation buttons was used, it would crash Internet explorer giving a generic error - "Internet Explorer has stopped working."  Issue no longer occurs.

Task 34354   - ENHANCEMENT                                                                                                                                     

Auto scroll on activity   

Resolution:                                                                                                    

The right nav menu in NEXT has been enhanced.  If the screen resolution is such that the right menu is pinned at the bottom of the screen, clicking the down arrow to open a right nav item will auto scroll the right menu down to expose the newly opened menu item.

Task 35851                                                                                                                                      

Surcharge Security Bug in NEXT   

Resolution:                                                                                                    

Resolved a NEXT issue where if a Surcharge Type is secured a related surcharge cannot be edited by a user, even if they have the security entity or are a global admin.  The system now respects the security rights just like any other secured part of the system. 

Task 35975

Resolution:                                                                                                    

The Mass Update but has been secured in NEXT Allowing for Role security using the CRM_OrgMassUpdate and CRM_ContactMassUpdate security entities.

Work Items duplicate prompt   

Resolution:                                                                                                    

The NEXT system has been enhanced to recognize when you already have a module open in your left NAV.  For example, if you've already clicked the Workspaces module, on the top nav, to open it up in the Work Items (left nav), and you click on it again, ClientSpace will recognize that you already have it in your Work Items and will prompt "I see that you already have Workspaces open, would you like to open a new version of Workspaces?", clicking "Create New" will open the new version and clicking "Navigate to Existing" will take you to the existing one.

Task 36110        - ENHANCEMENT                                                                                                                                                                                                                          

Attendee Availability "Force Resync" button    

Resolution:                                                                                                    

Enhanced External Sync of calendar with the ability to "Force Resynch".  This pulls any newly scheduled or changed calendar events for attendees into the Attendee availability view.  This functionality depends on the calendar events being available at the url location of the ICS url on the user profile.  If the event hasn't synced to this in Outlook it is not available for ClientSpace to include.  In testing we found as much as a 2 minute delay at times depending on Network and server traffic around the Outlook server.

Task 34143

Resolution:                                                                                                    

Resolved an issue where moving a single instance of a recurring event on an External calendar would generate two copies of the moved event, the original and one for the re-scheduled time.  Calendar now appropriately moves the selected recurrence and does not retain the original time for that occurrence.

Task 36624                                                                                                                                    

Left Nav Dirty State On Load   

Resolution:                                                                                                    

Resolved an issue where the Left Nav bar in Next was not recognizing any changes made to the page state by code that fired on load of the page.  This happens for example when a Lookup on the page is set to default, or through header script logic.  The system now recognizes the dirty state of these fields,  even if set by coded events on page load.

Task 36039                                                                                                                                    

Attachments Right Menu - View Full List

A "View Full Search" link has been added at the bottom of the list, and a magnifying glass button at the top, similar to the Tasks and Time Tracker right menus.

Task 33518       - ENHANCEMENT                                                                                           Coming soon...                                                                                                                                 

File Versioning User Interface

Resolution:                                                                                                    

The file versioning functionality has been implemented in NEXT, allowing you to store different versions of a related file.  Functionality mirrors the Classic behavior, where the default to either Pin a new file or overwrite the existing file is set in Admin Settings  | App Settings | Miscellaneous | File Versioning - Checking this box will default the system to pin a new version of the file, retaining the previous version(s).  Unchecking the box will default to overwrite the existing file with the new version.  The rest of the functionality is consistent with Classic, providing a "Manage Versions" dialogue to provide information about attached files, determine which is the currently pinned version and delete extraneous files.
