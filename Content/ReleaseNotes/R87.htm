33948 and 39106

  

Workflow Channel: Trigger workflow from other task items

Workflow configuration now includes triggering based on the status of Task workflow items other than the immediate parent of the triggered item. To take advantage of the enhancement, administrators must change the immediate parent item to fire the workflow item trigger, which then allows the admin to build complex workflow triggers based on the status of multiple workflow tasks. For more information, see the topic Triggering a workflow item from other task items.

 

Workflow Conditions now provide the ability to specify additional task triggering conditions that are not related to the directly preceding workflow task. The status from any task in a related workflow chain can be used as an additional triggering condition by selecting the option for Workflow Item from the Source list and then selecting the related workflow task when prompted to select a workflow item. The only field available for triggering configuration, in this case, is Status, and the only operator that can be selected is Equals. See Triggering a workflow item from other task items.
38983

Import Map Detail configuration: Update field if value provided

The Default Value field on the Import Map Detail configuration was enhanced. Previously, if a no value was provided in this field and the import Source file column was empty, the blank value was written to the dataform field regardless of whether the dataform field contained information. The system now allows you to configure this in several ways on a field-by-field basis. You can configure the default value field to handle blank values using the following options.

Standard Imports

None: If the Import file column is empty, it clears the dataform field.

Current Value: If the Import file column is empty, it leaves the current value in the dataform field.

Special Value: 

If the Import file column is empty, sets the value in the dataform field to the value provided (for example, "UseThisValue").

If there a problem with the configured Default Value, the Current Value is the default.

Salesforce Imports

None: If the Import file column is empty, it clears the dataform field.

Current Value: If the Salesforce field value is empty OR API Path is set to UseDefault, it leaves the current value in the dataform field.

Specific Value: 

If the Salesforce field value is empty OR API Path is set to UseDefault, it sets the value in the dataform field to the value provided (for example, "UseThisValue").

For User, Organization, or Contact fields, the provided value MUST be the PK of the form to use (no decoding is performed, the raw value is used).

Salesforce Exports

None: If the Salesforce field value is empty OR API Path is set to UseDefault, it clears the Salesforce Object field.

Current Value: If the Salesforce field value is empty OR API Path is set to UseDefault, it leaves the current value in the Salesforce Object field.

Specific Value:

If the Salesforce field value is empty OR API Path is set to UseDefault, it sets the value in the Salesforce Object field to the value provided (for example, "UseThisValue").

For Account, Opportunity, or Contact fields, the provided value MUST be the SalesforceID of the record to use (no decoding is performed, the raw value is used).

See Configuring the Import Map Detail.

Import Map Header

Import configuration has a new field Workspace Source Field in the Import Map Header. The administrator can configure an import to select the appropriate workspace when inserting a row of data based on the following criteria:

ID (Record Primary Key): This option inserts/updates the record in the workspace where the dataform of that type and the RecordID match the Workspace Source Column from the source file.
