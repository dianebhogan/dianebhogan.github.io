Enhancements

Hover on the icons to discover the level of product enhancements: .

Produce Task Notifications through Email Templates (Case 30409)

The task notification system was enhanced to use a System Email Template if available, making task notifications customizable. These enhancements include:

  • New System Email Template with the code value INCIDENT.
  • If the INCIDENT template does not exist, ClientSpace uses the original notification style.
  • For emails generated from the INCIDENT template, the Email Admin dashboard displays TaskSystemEmailTemplate in the Source column (otherwise displays Task).

Score Card Merge proc (Case 34710)

A new stored procedure for the Client Scorecard merge, peo_merge_client_profitability, has been added and includes scorecard fields and the following additions. Note: To implement this enhancement, you need a merge template. For assistance, please log a case in the Extranet. 

Payroll tab: The merge date range now applies to all results and returns the following:

  • Count of payroll batches in date range (integer)
  • Count of paychecks (employees paid) in date range 
  • Bill Amount, Cost Amount, Bill Code, and metadata value on Bill Code (Billing Voucher)
  • Total count of payroll batch dataforms by batch type in date range (integer)  R, S, M, Other
  • TotalWSEPaid by: TotalR, TotalS, TotalM, TotalOther
  • Total Checks Processed equal to the sum of Employee Count field on all batches in the date range
  • Average EE Count: Total Employee Count divided by count of batches in the date range

Contract Info: Surcharge list data associated with current batch returns:

  • fkSurchargeTypeID (decoded)
  • Surcharge
  • SurchargeDescription

Commission list data associated with current batch returns:

  • fkSalesEntityID (decoded) 
  • luCommissionMethod (decoded)
  • SalesCreditPct
  • PercentProfit
  • PercentPayroll
  • FlatCommission
  • PerHeadCommission
  • SurchargePercent
  • PercentPremium
  • PercentAdmin
  • EffectiveDate
  • ExpirationDate

Client Service: A new column shows HourlyCost by Department (Client Visit Type Metadata).

Unemployment: Two new columns:

  • Now returns the Latest Value as of value.
  • Returns a sum of all of the active benefit charges. **No changes to the Summary tab.

Email Templates Address Recipients (Case 39875)

The Address Recipient Proc list now reflects options compatible with the Table Name from Step 1 of Email Templates.

Redirect of Extranet module (Case 39927)

The module that allows registered users to connect to the Extranet and log cases for ClientSpace now redirects users to the Help Center. Note: In a future sprint, we are deprecating the Extranet Module. 

Benefits Plan Tier Validate ImportID (Case 40105)

The Benefits system now has the business rule Validate Benefits Plan Tier Title to validate the Plan Tier title. This enhancement prevents generating duplicate ImportIDs.

Org Primary Contact (Case 40338)

The peo_employee_emailtemplate_recipient_v2_proc now includes the Primary Contact of the Organization related to the Employee Workspace.

Workflow Channels: Notify triggering user (Case 40406)

Workflow Channel configuration has been enhanced to add the current user to the Notifications tab so that when the workflow triggers, the current user is aware that something has been created. This enhancement includes:

  • A new option Notify Current User is available when adding a notification user.
  • The system notifies the Created By user when creating a task or dataform if Notify Current User is enabled.
  • The notification uses the Enable Email setting on the user profile.
  • When the current user is added to the notification list, the user receives email notifications of all updates.  

For more information, see the Help topic Step 3: Notifications.

Help document links (Case 40423)

Links for non-video Help documents were re-directed to the new version of these articles on our newly integrated Help site. 

Email Templates user select (Case 40413)

Email Templates now allow the System Admin to select users across tabbed dataforms in the From Address area of Email Templates.

  • When the Triggering Form for the Email Template is the primary tab in a tab set, the modal pop-up on the Address step of Email Templates for From Address now contains child tab forms on the From a Field on a Form option. 
  • This interface works the same as the Content step of the Workflow Channel (Triggering form, Child tabs with a dash, everything else).

Workflow Channel notifications (Case 40855)

Workflow Channel notification now includes child tab form fields when selecting notification users From a Field on a Form. The field selector includes child tab form fields where the parent tab form is the triggering dataform for the workflow channel.

Trigger Conditioning for Parent Form fields: Email Templates and Workflow Channels (Case 40414)

Workflow Channels and Email Templates were enhanced. Parent form fields are now available for primary triggering and additional triggering conditions. Available Operators for parent form fields are IsDirty/Empty/Not Empty. Admin workspace Parent form fields also include equals and not equals operators with values decoded in the Values list.

Project Milestone dataform (Case 40594)

The Project Milestone dataform now has an Owner field (fkOwnerUserID).

Organization Category updates from Client Master (Case 40599)

The Org Category now uses the Client Master status metadata when using the Activate Future workflow and scheduled process.

Product name changed to ClientSpace (Case 40857 and 40858)

External mailbox references to Pulse: References to the name Pulse are now removed from the External Mailbox in the application and replaced with ClientSpace. The name change is on the page for posting to ClientSpace using the Email Add-In (Post to ClientSpace).

Application name in App Settings: The name Pulse was removed from the following areas within the application and replaced with ClientSpace:

  • The header of the home page
  • Help Center text
  • The header of the application help page
  • The default company name in the application

Enterprise Library: ZIPCodeRepository (Case 40992) 

Removed enterprise libraries and replaced them with more efficient code to optimize system performance.

I-9: Ongoing Imports and Subscription (Case 41250) 

The PrismHR API now includes an ongoing I-9 import configuration. The HRPAPI_I9 import includes:

  • Identification Document: Identification documents for List A, B, and C documents
  • Document Expiration Date for each of these documents
  • Non-Resident Alien: checkbox value 
  • Alien Registration Number
  • Alien Registration Expiration Date

PrismHR API v1.18 (Case 41483) 

The ClientSpace PrismHR API service was updated to version 1.18 of the API.  

Pricing Console: Quick Create (Case 41569) 

The pricing console Quick Create function was enhanced. When using the Save and New functionality, the State field now clears, ensuring that users select a state.

Issue summary

Resolution
40263   Outlook Add-In record search not functioning for selecting inactive records

Resolved issues with the External Mailbox module when attaching inactive records to a dataform. The action would fail. Inactive records can now be attached to dataforms, just as active records. 
40620   Child Workflow Dataform items - Add Untriggerable issue

Resolved issues with the dependent ancestor in a child workflow item. When you Add Untriggerable from the first workflow dataform item that is not the channel trigger, the child workflow item does not update the Triggerable flag on the dataform field record.


Example:
  • Workflow triggers from an Employee form.
  • This generates a task based on an employee field that was added as Add Untriggerable. When selected, this field is marked as triggerable in dataform admin.
  • That task creates a case (this resets the closest dataform ancestor to the case form).
  • The Case creates a Task that is triggered by a Case field that is added as Add Untriggerable. When selected, this field is now marked as triggerable in dataform admin.
40621   Hard error on change of generated item in child workflows

Removed the ability to change the Triggered Form configuration on a child workflow item after it has been saved because this could cause significant issues for downstream child workflow items that rely on this triggered form. 
40625  Removed Add Untriggered functionality from primary trigger

The Add Untriggered functionality was removed from the primary trigger on Email Templates and Workflow Channels. You can no longer add untriggered fields on the Primary trigger field of real-time workflow and email template configurations. The list now only displays fields previously marked as Triggerable in dataform administration. Add Untriggered functionality is still available from the Primary Triggered field for scheduled workflows and from the Conditions tab (Step 4) of workflow configuration.
40631   Activities icons

Resolved issues with icons that did not display correctly on the Workspace Landing page Activities widget.
40664   Calendar error

Resolved an exception when synching an external calendar with ClientSpace: Cannot insert the value NULL into column 'EndsOn', table 'ClientSpace.NetWise.dbo.tblExternalCalendarEventOccurrences'; column does not allow nulls. INSERT fails.
40809   COI not showing Signatures

Resolved an issue with the Reporting logo and Reporting signature branding files when signatures did not display on the Certificate of Insurance reports.
40868   XML Data type in Link and Widget Conditions

The XML datatype was removed as a usable datatype in Link display and Widget Condition configuration. The datatype did not add value and had the potential for causing unexpected behavior if improperly configured.
40879   Save and New in lookup Admin causes an exception on save

Resolved issues that caused exceptions when adding lookup configurations with Save and New functionality.
40897   Cannot edit text in Workflow Channel Conditions tab

Resolved issues with Workflow Channel and Email Template conditions with unresponsive fields when adding a condition.
41019   Email Template replace fields are not working

Resolved issues with Email Template field replacement when the email template was triggered from a multiform using replace fields from single forms.
41228   Scheduled Workflow Channel triggering bug

Resolved an exception when retrieving the triggering data while executing a scheduled workflow channel.
41334   Type (fkCaseTypeID) does not load as a value

Resolved populate issues with the Value field when using a parent form datatype as the primary trigger.
41339   Scheduled Workflow Channel triggering

Resolved an issue where, under certain conditions, real-time workflows triggered when a scheduled workflow on the same triggering form was triggered.
41353   Workflow Channel Replace Fields for SDP on Triggering Dataform

Replace fields from the Server Data Proc that is configured on the triggering dataform are available at all levels of a workflow channel.
41367   Users cannot save changes to their profile

Resolved issues with non-Global Admin users trying to update editable fields on their user profile.
41398   Payroll Batch and Billing Voucher issues

Payroll batch and billing voucher dataforms were added to the Client Template workspace. The Payroll Batch dataform was added to the Payroll Info Group. Date Paid and PayrollNumber were added as columns in the multiform list at 40 and 50.
41508   Editing Org Security from the Org

Resolved an issue that would check the view attribute of the first row in the security tab when clicking the white space of the tab.
41593   JavaScript application error in Workflow Channels

Resolved a JavaScript error when a unary Operator (IsDirty, Empty, Not Empty) was on the primary condition of an existing workflow item, causing an error when editing these types of workflow items.
