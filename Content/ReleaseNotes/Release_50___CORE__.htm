Release 50 ( CORE )

Product Version: 50 – Release CORE Notes - The following Tasks have been completed/fixed in this release.


Task 37583                                                                                                                                      

Accepted Date to OrgOtherInfo   

Resolution:                                                                                                    

The "_SetFieldOnFormFromForm" business rule has been enhanced to allow updates to the Organization (Other Info) forms,  denoted by a checkbox in dataform management.  This form may now be set as the target form with the following caveats: This rule will not generate a new form for an Organization which doesn't already have a saved Other Info form.


Task 37584                                                                                                                                       Click Here for the Video 

Attach a file from a dataform field to an Email Template   

Resolution:                                                                                                    

New functionality has been added to Triggered Email templates allowing the current pinned attachment from any single form in the triggering workspace to be attached to the triggered email that is generated.  Additionally any pinned attachment field on the triggering dataform can also be included in the generated email, whether the triggering form is single or multi-form.  A new Attachments section has been added below the trigger conditions on the Triggered email template configuration.  The Add dialog provides a list of Single dataforms with attachment fields available in the system, as well as the configured triggering dataform.  Selecting a dataform will allow you to choose a list of available attachment fields on the selected dataform.  For more information read Configuring Triggered Email Templates in ClientSpace. 


Task 37690                                                                                                                                      

Task Subject   

Resolution:                                                                                                    

The Automatically generated email notifications from the Task system has been updated, replacing "Incidents" references with "Task" in the subject line. 


Task 37687                                                                                                                                      

Email Integration, exception when retrieving attachment   

Resolution:                                                                                                    

Resolved an issue with email integration where processing of records would fail, marking the record as read if there was an exception when retrieving corrupt attachments from an email.  Process now ignores corrupt attachments, and removes such attachments from the total attachment count when creating the case.


Task 37693                                                                                                                                      

Email Integration, exception when retrieving attachment   

Resolution:                                                                                                    

Continuation of Task 37687 dealing with exception handling when the email integration process experiences intermittent message retrieval failures.


Task 37726                                                                                                                                      

Import "Required Column" functionality issues   

Resolution:                                                                                                    

Fixed a bug with Imports where the "Required Column" defaulting functionality was not working properly, causing every row after the first to error instead of defaulting a field value when "Required Column" was unchecked, but a default value provided.


Task 37734                                                                                                                                      

HE_RequireFieldOnFormOnTrigger Rule Not Working   

Resolution:                                                                                                    

Fixed a bug in the HE_RequireFieldOnFormOnTrigger business rule where the business rule would fail under certain circumstances. 


Task 37579                                                                                                                                      

Task Security Update   

Resolution:                                                                                                    

Task security has been updated to allow Notification users the ability to edit the Comments field by default without the need to implement a special security entity.   


Task 37694                                                                                                                                      

Error Message on duplicate ImportIDs   

Resolution:                                                                                                    

Resolved an issue where when entering an ImportID on an Org, if the Org already exists somewhere else, an "Unhandled Exception" error is thrown. The system has been updated  to a more descriptive error which lets the user know there is a duplicate ImportID. 


Task 37697                                                                                                                                      

File Extension Case Sensitive   

Resolution:                                                                                                    

Resolved an issue with Server side merge code where the file extension of the merge template file was case sensitive.  The system no longer requires specific casing for the file extension of a merge template.


Task 37777                                                                                                                                      

Default to Next   

Resolution:                                                                                                    

System has been enhanced so that if the user record has "Default to Next" checked, the user will be re-directed to Next on login, regardless of whether the URL they used to login contains the /next string.  Users who wish to sometimes use the classic version of the application can still remove the /next string and they will be re-directed to the Classic version of the application until they log out. Users who wish to default to Classic may uncheck this option and the system will default to Classic (only works with Internet Explorer version 10 and above).


Task 37787                                                                                                                                      

Field Audit Trail   

Resolution:                                                                                                    

The Field audit trail search dialog has been enhanced to account for large datasets which would cause issues when populating the list.  Search on Load has been disabled, and an Audit date range is now required to begin the search.


Task 37763                                                                                                                                      

Log datatype required issue   

Resolution:                                                                                                    

Resolved an issue where the required attribute on a log field would get ignored when you save the form. When field display was used to hiding/showing a required log field, the required-ness would be respected,  but the message would not contain the field name ie "• is required".  System now correctly handles the required attribute of Log Fields.
