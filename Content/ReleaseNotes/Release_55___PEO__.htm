Release 55 ( PEO )


Sprint Began: 5/8/2017     Sprint Ended: 6/2/2017


The following items have been completed in this release:


i3 Analytics Merge ENHANCEMENT

A new merge is available to display i3 Analytics data for a client.  This Excel merge retrieves i3 data from ClientSpace pricing comp codes and aggregates it with payroll data from the same pricing batch.  For more information about i3 analytics integration, or for help with configuring this merge, please contact your NetWise account manager.

Renewal Pricing Batch Header in both Classic and NEXT appears to display incorrect Total EE Count 

Fixed a bug in the header code for the Pricing Batch that would always pull the Employee Count and Gross Payroll amounts from the Activated Batch - system now appropriately displays the Employee Count and Gross Payroll for the current batch being viewed.

Email Reply to Case Discussion ENHANCEMENT

Email Integration can now be used to initiate CSC Discussions.  An external person can send an email to an Email Alias configured mailbox and a new Client Service Case and Discussion will be generated.  Subsequent Discussion Messages can be managed by using the ReplyTo link to open the discussion in a web browser or by replying to the email (either/or - not both).  External recipients have the option of clicking the standard ReplyTo link embedded in the email or they can 'Reply' to the email (either/or - not both) and the Email Integration process will attach it to the correct CSC/Discussion Thread.  For more information on configuring and using this feature, contact your NetWise account manager.

The RFP Pricing panel used on the Pricing Batch and Pricing State dataform header(s) has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

The Claim Info panel used on the EPLI Claim dataform header has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

The Claim Notes Info panel used on the Claim Notes dataform header has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

Validations on File Pricing Load (NEXT) ENHANCEMENT

The Quick Create preload process has been enhanced

Uploading a State/Code CSV file now performs Row Validation.  Any validation failures will result in the row being highlighted in RED  This validation performs the following:

  • Validates the State against the State Rate table (match on State or Formal Name); if not found, State field will not be populated
  • Validates the Comp Code against the Comp Code Master table; if not found, Comp Code field will not be populated
  • Pay Frequency (field in file not populated, does not verify that it is a valid Pay Frequency)
  • Duplicate State/Code records
    • each row from file will be checked against existing rows in the Quick Create grid
    • if a match on State and Comp Code is found, will REPLACE existing FTE, PTE, Gross Payroll, Pay Frequency, Manual Rate and SUTA with new data.  A new row will NOT be created

Uploading multiple files

  • Additional files can be added (one at a time) to existing records.  Validation and Duplication checks will be made as above
  • Generate Pricing button will alert an error if any rows have validation errors.  Generate Pricing is not allowed if errors exist.

Pricing Breakdown Updates (NEXT) ENHANCEMENT

The Pricing breakdown in NEXT has been secured - a new security entity has been created "biz_pricingconsole_breakdown" which allows the Pricing Breakdown Matrix at the bottom of the pricing console to be hidden.  At the same time, a link can be made available to display the breakdown information.

Update OSHA 300 - ENHANCEMENT

The OSHA 300 report in PEO has been enhanced -

  • A filter has been added to the filter page that allows the user to select whether they want to run the report for clients with claims, clients with and without claims, or clients without claims.
  • The system will now populate client name, city, state, and year on OSHA 300 regardless if there are claims or not.
  • If the report is run to only show clients "With Claims" on a client without or vice versa, the report will not populate the client information.

Pricing Module Error

Resolved an issue with Pricing module widgets where opening pricing directly from the Pricing module would cause the system to throw an error due to the lack of a pre-existing ProjectID. System now allows you to create pricing from the Pricing module directly.

Client Master Status & Org Status not Displayed in Classic (CLASSIC)

Resolved an issue where changes to the Client Master header Stored Procedure when it was widgetized caused the Org and Contract Status to not be displayed in ClientSpace Classic.  Both Classic and NEXT now appropriately display the correct status values.

ActOn Sync Status

The Contact Import was missing Act-On Status field, preventing us from setting up and turning on the API since we are not able to set the initial batch of Contacts to be pushed into Act-On.  This has been corrected by adding the field throughout the import configuration and custom import dataform.  For configuration help contact your NetWise account manager.

Divide By Zero Error on Payroll Info Panel Widget (NEXT) 

Resolved an issue where the Payroll Info widget on the Client Workspace Landing page would throw an "Divide by Zero" error when attempting to calculate annual wages for a workspace which had only one record in Payroll Statistics.

Update i3 Code from "RiskMD" to "i3 Analytics" ENHANCEMENT

The system has been updated to alter all areas which reference "RiskMD" to "i3Analytics" as this brand has changed their name.

Landing Page Email Link Not Working (NEXT)

Resolved an issue in NEXT where the email and phone links on the Workspace Landing pages would not open a default mail client or start a call.  System will now appropriately interact when Landing page links are clicked.   (Interactions on mobile devices may be browser and phone specific - iPhones for example require you to press and hold email and phone links before being prompted to start a call or email.  For questions on your device, contact your mobile provider).
