Release 55 ( CORE )


Sprint Began: 5/8/2017     Sprint Ended: 6/2/2017


The following items have been completed in this release:

Save and Create Appointment (NEXT) ENHANCEMENT

A link has been added to Activity records called "Save and Create Appointment". When clicked the system will save the Activity and open a new calendar event in add mode with connection to the organization related to the activity.  When the link is clicked, the workspace will default to the org's workspace so the workspace is defaulted on the calendar event (if the org has multiple workspaces, the workspace selector will allow you to choose one of the associated workspaces.

The activity work item will be saved and a an appointment work item will be created. The appointment org defaults to the activity's org, and the appointment contact defaults to the activity's selected contact (if there is one).

Any changes made to the activity prior to clicking the link are saved when the link is clicked. Validation on the activity, such as required fields, also happens when the link is clicked.

Disable the search button until the search is completed (NEXT) ENHANCEMENT

Resolved an issue where under certain conditions the Search buttons on list pages and dashboards would allow  multiple clicks that would lead to unexpected results.  Search buttons are now disabled while search results are being collected and enabled once search results are displayed in the list.

Jump button on search dashboards opening up new records (NEXT)

Fixed an issue where changes to the list manger caused the Jump to icon to jump to the selected record in the list instead of the row that was being clicked on.  System will now jump to the record based on which record is clicked.

Calendar Emails Not Getting Sent (NEXT)

Fixed a bug where calendar event related emails were not getting sent if a user had unchecked "Enable Email for this account" on their user profile.  Emails are now sent on calendar events regardless of this setting.

Sending Emails to Inactive Users (NEXT) ENHANCEMENT

A condition has been added before an email is sent to look to see if the user is Active or Inactive. If Inactive, the system does not send any email notifications.

the system does this check at the time the recipient row is inserted, and also at the time when the list of recipients is retrieved, in case they were inactivated between email creation and send.

Update Contact Field on Activities to Only Show Active Contacts (NEXT) 

Resolved an issue where when adding an Activity in a Client site the list of Contacts in the Contact field showed Active and Inactive Contacts. The Contact field on the Activity has been updated to only show Active Contacts in the drop down.

Time Tracker button not working for Stopping (NEXT)

Resolved an issue with the javascript on the time tracker detail record that was causing the "Stop Timer" button to not stop the time tracking record when used from the Time Tracker detail record after drilling into an open record.

Out of Office: IsPrimaryRecipient (NEXT) 

The IsPrimaryRecipient flag, on the Email Recipient that's inserted for the Out of Office backup user, now mirrors the Out of Office users "IsPrimaryRecipient" flag. So, if the main user is a Primary Recipient, the backup user is as well, on that email, so that it shows up in their "To" box, as opposed to the "CC" box, in the ClientSpace Inbox.

Badges on Dataform Subscribers Right Menu (NEXT) 

Resolved an issue in NEXT where the subscribers badge which shows notification counts for a dataform was not getting populated until the subscribers button was clicked.  Badge now displays subscription count when the dataform opens if subscriptions are present.

Convert Calendar Invite to Email Templates ENHANCEMENT

Calendar events have been updated to allow them to utilize email templates when formatting the calendar invite that is sent from ClientSpace.  This will allow you to configure a calendar event email with html formatting and branding.  For more information, review Configuring Calendar Event Notification Template or contact your NetWise Account manager

Task notifications are no longer coming to my email 

Recent changes to cut down on the amount of unnecessary email being produced by the system caused an issue where triggered template tasks were not notifying the triggering user if they should have been on the notification list (owner assigned to etc...)  Triggered template tasks will now notify the triggering user as they may not even be aware they are triggering a task.  Notifications are not sent however to the logged in user if they create a task manually through Task Manager and add themselves to the notifications.

DocuSign issues 

Resolved an issue where if there are more than one documents in the ClientSpace Queue that have overlapping Signers and Anchor Tags, the code that is sent to DocuSign duplicated the tags, which forces the users to have to click the DocuSign controls more than once. It appears to duplicate the number of times that the Signer and Anchors are in the ClientSpace Queue. The system no longer creates these duplicated tag/queue records.

Workspace Delete Failing due to FK relationships

Resolved an issue where foreign key relationships caused workspace deletes to fail and be rolled back in a transaction.  Workspaces may once again be deleted from the Classic Workspace search by global administrators.

App Settings page, BI tab

The Business Intelligence tab of the App settings page has been removed from App settings in ClientSpace.  This page contained only single adjustable setting and the install key, which if changed improperly causes Business Intelligence to stop functioning.  If you would like help adjusting the "Hide report filters if locked" setting, contact your NetWise Account manager.

Incident workflow exceptions

The decode field in the Incident configuration options has been lengthened from 50 to 500 characters to allow for longer descriptions in Task Category, Priority, Type and Workflow.

Email Template Error 

Resolved an issue where a NULL reference error would be logged if you changed the Table Name configuration of an existing triggered email template

Weird Dates in Audit Trail 

Fixed a bug in the Audit trail search that would truncate the year of the Begin and end date search parameters to two years, causing the system to interpret the dates incorrectly (06/01/17 or 1917 instead of 06/01/2017).  System now retains the appropriate 4 digit year format.
