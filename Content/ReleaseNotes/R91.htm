36156

PrismHR Event for Employee Garnishment data

The PrismHR API for Employee data was enhanced to include the HasGarnishments flag, mapping to the Yes / No value of the same name on the Employment record in ClientSpace. This enhancement applies to Employment and Benefits Ongoing (HRPAPI_employment*.csv and HRPAPI_emp_benefits*.csv) imports as well as the Employment Initials through the Import Sync Process (HRPAPI_initial_employment*.csv). This feature is only available on v1.20 or higher of the API (if v1.18 is configured, the value set in the Import file will always be No). See Employee identifying information.

41478

Risk Management BI Dashboard

A new Risk Management Dashboard (Business Intelligence > Risk > Risk Management Dashboard) and associated reports were added to ClientSpace. From the dashboard, you can track Workers’ Compensation Claims by Incurred Value, by Claim Specialist, by Frequency, and much more. The BI Dashboard should help provide visibility on Workers’ Compensation Risk for your PEO. See Risk Management Dashboard.

42644

Time Tracker

The timer feature of Time Tracking was enhanced to round up to the next minute. For example, 1 second = 1 minute; 1 minute 1 second = 2 minutes, and so on. When adding quick time tracking records, the default behavior is now 1 minute.

42705

 

Email Templates: From Friendly Name

Email Templates now allow an optional friendly name to display when the From Address uses AdHoc for Email Address Source. This makes it less likely that recipients will view the email as spam. The AdHoc option now allows the admin to include an alternate From Friendly Name, which then displays in the generated email along with the actual email address, such as Your friendly support staff <noreply@clientspace.net>. In this example, when configuring the From Address, the Email Address was set to "noreply@clientspace.net" and the From Friendly Name was set to "Your friendly support staff." See Specifying a friendly name as From Address.

42713

Email Templates: Current User Notification

Email template functionality was updated to exclude the current user (logged in user) who triggers an email template unless the template is explicitly configured to include the current user. For more information regarding email template recipients, review Email Templates: Step 3 Addresses.

42741

Custom Logic Rules

Custom Logic Rules now allow custom business rules on non-dataform objects, such as Organizations, Contacts, and Calendar Events. This enhancement includes:

  • The Custom Logic Rules list now has a column for Table Name.
  • More search was updated, replacing Dataform Name with Table Name.
  • Quick search now includes Table Name.
  • On the Rule Details configuration form, Dataform Name was replaced with Table Name.

See Configuring rules and business logic.

Certificate of Insurance cloning on policy renewal

The Certificate of Insurance cloning process, which occurs after a Workers' Comp Policy is renewed, now includes the following additional fields:

  • COI Version lookup
  • Additional Insured
  • Include Waiver of Subrogation

43028

Client Team: My Support Dashboard fields from PrismHR

The Client Team dataform has new fields that are populated by the My Support Dashboard in PrismHR. The fields, labeled PHRSupportDash1 – 4, are in a new hidden fieldset on the Client Team dataform. To use the fields, configure the fields in PrismHR with the appropriate users, and unhide the fields on the Client Team form. The scheduled process that handles inserting and updating client information also processes these new fields. For additional help with configuration, please log an Extranet case.

Benefits Batch link from Pricing Console

Users can now link to the Benefits Batch dataform from the Pricing Console, which opens the batch form directly. The Benefits Batch link displays if the user is assigned to the Benefits license, and on the Client Master, Providing Health Benefits? is set to Yes for the workspace. See Configuring the Benefits Batch link to the Pricing Console.

43211

 

Communication Method lookup group

The Communication Method lookup group is primarily used on forms that track client communications, such as the Client Service Case. The Communication Method lookup group was updated with the following values:

  • Employee Portal
  • Email Add In
  • Email Integration
  • Workflow Channel

43287

Delete Events from PrismHR

The PrismHR API Ongoing process was enhanced to improve handling DELETE events in PrismHR. This enhancement aids in eliminating excessive import queue failures caused by delete events. These failures would back up as the records would never be resolved and removed by the cleanup process. The enhancement now handles DELETE events in PrismHR in the following ways:

  • For Employee records, when a DELETE queue record is created, ClientSpace now inactivates the associated Employee record.
  • For Employment records, when a DELETE queue record is created, ClientSpace now sets the status of the associated Employment record to Deleted.
  • For Location records, when a DELETE queue record is created, ClientSpace now inactivates the associated Location record.

43288

 

Exception Log: Source field

Through a new Source field on the Exception Log, ClientSpace can capture the source of an error and provide more troubleshooting information. Application functions must be updated to pass source data. This development includes enhancing the import process to capture the source of an exception at point of failure. Source is also now a hidden column on the Exception search list

43292

Employee/Employment Initial Scheduled Process

The PrismHR Import system has a new PrismHR Initial Employee Sync scheduled process, which is scripted as inactive. This scheduled process uses the HRPAPI_initial_employee*.csv and HRPAPI_initial_employment*.csv import configurations (must be active) to retrieve a list of Active Client Master records where Client Number is not empty. These Client Numbers are then matched in PrismHR to create a list of Employee and Employment records for each client. See List of scheduled processes.

43554

Subscribing Client Team members to Cases

Adding subscribers to a Client Service case has never been easier. You can now subscribe Client Team members to a Client Service Case. The subscriber list has a column Client Team Role, with the associated role. When you add a user to a team role, it automatically makes them available for selection as a subscriber. You can also filter the subscriber list to show only team role members. See Subscribing Client Team members to cases.

43811

 

Classification on Employment records

The Employment record now includes Status Classification, which improves the alignment of employment statuses with values in PrismHR. The enhancement includes:

  • New lookup group Status Classification with the following values: Active, Terminated, On Leave.
  • New Status Classification lookup field on the Employment dataform.
  • New field on the HRPAPI_initial_employment and HRPAPI_employment for Status Classification. This is a TEXT Data Type.

Benefit Administration

The following enhancements are included in the ongoing Benefits Admin/Benefit Plan Manager project.

 

Benefits Tiers changed to Rate Groups (Case 43483)

Benefits and Offered Benefits plans now align with how benefits are managed in PrismHR. The plans use tiers as well as age banded plans. The enhancements include:

  • New lookup field for Rate Group Type on the Benefits Plan.
  • New lookup group called Rate Group Type with the values: Tiered and Banded
  • The Risk Tier labels throughout the application were updated to Rate Group
  • In all places, Risk Tier or Plan Tier was previously used:
    • When Rate Group Type = Tiered. No change to previous functionality. One Tier (Rate Group) is selected per Offered Plan. All cloning logic remains the same.
    • When rate Group Type = Banded, the Tier (Rate Group) field on the Offered Benefit Plan is hidden.

For the first phase of this project, the current logic for the header of the Offered Benefit Plan, which populates the Rate grid at the top of the Offered Plan, remains the same. This displays the base rates with no adjustments.

 

Apply Plan Group prevents duplicates (Case 43066)

When an Offered Benefit Plan is added to a Benefits batch (either manually or through the Apply Plan Group link), logic checks the fields Benefit Plan, Rate Group, and Classification on any existing Offered Benefits Plans for that Batch. If all three already exist on any one record, the system does not create another one.

 

Benefits Classifications changed to Benefits Groups (Case 43447)

The benefits system was updated to align more closely with PrismHR Benefits terminology. The following changes were applied:

  • The Benefit Classification form is now Benefits Groups.
  • The custom link to these Groups on the Benefits batch dataform is now Benefits Groups.
  • Fields on the Benefits related forms now refer to Groups instead of Classification.

43514

PrismHR API v1.20: New method for retrieving Events

The PrismHR API has a new NumberOfEvents parameter for the API configuration. The new parameter sets the max number of records to import per subscription record on each run of the HRPAPI_Ongoing import. Setting NumberOfEvents to 0 defaults the max records imported per subscription to 100. This parameter minimizes the risk of imports failing due to large recordsets. See Import Queue management and configuration.
