Release 60 ( CORE )


Sprint Began: 9/11/2017     Sprint Ended: 10/6/2017


The following items have been completed in this release:


Home page widgets on apple devices - NEXT

Resolved an issue where home page widgets didn't handle responsive design correctly on ipad/iPhone


Field Display Rules fix

Resolved an issue with creating style field display rules for dataform fields which had a "State" field display rule already applied.


Inactive Workspaces Not Showing Links to Child Records Correctly

Resolved an issue where records associated with Inactive Workspaces would not display properly throughout the system on lists and badge counters.  System now includes records for workspaces which are inactive.


Subscription Email Change

Resolved an issue where subscription emails were being sent which read "Changes are marked in bold" but then the changes were not marked in bold due to ClientSpace NEXT does not have access to the contents of Dirty Fields at the time the emails are generated.  Subscription email now contains a link to the dataform allowing the user to drill in and see the changes.


Incident Mass Updates executing candy Business Logic (Next) - NEXT

Resolved an issue where when mass updating Incidents, hard errors would fire on save causing the save to fail in the background, but not throw an error.  We now set the header action to 'massupdate' before submitting data to the web service save.  This allows custom business logic (HE, SE, BizLogic) to determine if the logic should execute during the mass update. 


Recurring Task Doesn't send notification

Resolved an issue where a task set as recurring on Save would send a notification, but subsequent tasks that are cloned from it would not.  Cloned tasks will now generate email responses to the users define in the Notification tab of the task.


Required Field issues - Field Display

Resolved an issue which occurred because of the way the field display code was relying on the field labels to find required fields, the recent change to add the help icon to the field labels caused this to break.  The field display code was changed to not rely on the labels, but to use an attribute, directly on the fields, instead.


Clone User, Default To Next  

Fixed an issue where a cloned user would not inherit the "Default to NEXT" setting of the user account from which it was cloned.  System will now also clone the "Default to NEXT" setting.


Scheduled Process List - Last Run -   ENHANCEMENT

Last Run (sec) has been added as a column on the Scheduled process list, displaying the last run duration in Seconds and allowing the admin to quickly assess long running scheduled processes.  This column is sortable


Merge Admin - Merge Type Filter - ENHANCEMENT

Merge Type has been added as a filter field to the Merge Admin dialog, allowing you to filter by type of PDF, Word or Excel.


Counter Badge on Activities

Corrected an issue where the Activities counter badge in the Action items list was not displaying the appropriate count for Organizations with more than 10 activities. 


Deleting Documents - dataform attachments and action item securtity - NEXT - ENHANCEMENT

Changes were made to standardize the way that Right Menu file security works across entities in the system. Previously the dataform file right Action item menu was wide open, so if you have view rights to the dataform you can add, edit and delete files, but, on orgs, contacts, activities and tasks, you needed at least edit rights to the entity in order to add, edit or delete files. All entities now work like this:
* First, you must have edit rights for the entity that you're on in order to do anything but view files.
* If you have edit rights, then you can optionally lock files down using an "_$Attachment" security entity.

  • Dataforms use {TableName}_$Attachment - set by checking the "Attachments Secured" box on Dataform properties
  • Orgs, contacts, activties will use CRM_$Attachment - must be added by a ClientSpace representative - if you would like to utilize CRM attachment security contact your Account manger.
  • Tasks will use Incident_$Attachment - must be added by a ClientSpace representative - if you would like to utilize Task attachment security contact your Account manger.


Database Performance Tuning - ENHANCEMENT

In our ongoing efforts to improve system performance and the user experience we have added several additional indexes to commonly used dataform tables to speed data access, including such dataforms as the Client Master and Policy Comp Codes.


Grid Column Resizing in Chrome 

Resolved an issue where grid columns would not resize properly in the Google Chrome browser.
