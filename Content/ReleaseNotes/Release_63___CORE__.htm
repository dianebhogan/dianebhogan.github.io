Release 63 ( CORE )

Sprint Began: 12/04/2017     Sprint Ended: 12/31/2017


The following items have been completed in this release:


Field display Hide enhancements -  ENHANCEMENT

The field display engine which hides or displays fields has been enhanced. Previously when both fields in the same row of a dataform were removed the white space where the fields live would persist, leaving holes in the page. The system will now collapse the whitespace if both fields are hidden.


External User Izenda Access -  ENHANCEMENT

Functionality has been added which can allow external users of the system access to Business Intelligence reporting.  This functionality is controlled through a hidden application variable.  Report security will need to be adjusted prior to enabling this functionality in order to limit external access to Client Data.  For more information contact your account manager.


Contact Detail: Remove Sync with Outlook from UI 

The "Sync with Outlook" functionality on the Contact Detail record has been deprecated and removed from the system.


Task Updates Email Bug - NEXT 

Resolved an issue where the date formatting on the email produced when the start date or due date of a task was changed was inconsistent. All dates are now formatted in the same way when the email is generated.


Organization form layout changes in Chrome - NEXT 

Resolved an issue in Google Chrome where the primary contact and telesales user fields would change position on the form under certain conditions.


Template Attachment on Tasks

The Action Panel label for Template attachments has been updated to read "Workflow Attachments"


Dataform Save and New Issue

When adding a child form from the parent form the parent form field is properly set. However, if the "Save and New" option is selected from the child form the parent form context is lost on the next record created. The form now properly remembers the parent form each time "Save and New" is selected.


Docusign, upload signed documents

The DocuSign API has been updated to correctly attach documents to the triggering dataform, including multiforms.


Add Branding to Sys Admin in Next -  ENHANCEMENT

There are 4 images that can be used for branding the application. A new branding section has been added to sys admin are allowing you to upload the files for branding. Information about appropriate image sizes can be found by clicking the field level info bubbles. Once the files are saved the changes are applied to the system immediately - simply refresh your browser to display the branded logos.


Field Display "Clear on Hide/Disable" issue

Fixed a bug where the "Clear field on Hide / Disable" was always being set to true when adding new field display. Save will now respect the flag setting.


Database Code changes

The underlying database code has been cleaned of Database Engine functions soon to be deprecated by Microsoft.


New View - Expose Auditing table to Business Intelligence -  ENHANCEMENT

A new CORE view has been created - nw_vwFieldChangeAudit which retrieves the last 30 days worth of field change data in the application. This can then be combined with other system tables to get a picture of changes being made across the application in the last 30 days.
