Release 38 ( PEO )

Product Version: PEO – Release 38 Notes - The following Tasks have been completed/fixed in this release.

Task 34289                                                                                                                                      

"Needs Review" error message   

Resolution:                                                                                                    

A Yes/No flag has been added to the Approval Definition form and the HE_CodesNeedReview business rule has been enhanced. 

Now if a Code is marked 'Needs Approval', the associated underwriting approval will check the flag on the underwriting definition prior to firing the Hard Error. 

Task 34410                                                                                                                                      

Manual Entered Cell Number

Resolution:                                                                                                    

The Contact import (PEO contact*.csv) has been updated and will only update or insert Contact related fields that are actually present in the import file columns so it  will not overwrite manually entered data with a blank if the field is disabled in the Import configuration.  If a column is included in the import map (import file header column) and the row data for that column is blank, the field on the form will be cleared.  Any Contact field that is not included in the map header and import file header row will not be modified during the import process. 

Task 34509         ENHANCEMENT                                                                                                                              

Different Status Error in Imports   

Resolution:                                                                                                    

The Client Import process has been enhanced to allow for terminations from the payroll system via imports.  If the "Allow Terminations from Payroll System" is checked under Admin Workspace | Default Settings and the Workspace status is "Client" and the status in the import file is "Terminated" the system will now create a Client termination record with Todays date, a Reason of "Terminated by Payroll System".  The Status of the Client will also be set to "Pending Termination".  The nightly scheduled process will then complete the process of terminating the client workspace.

Task 34373                                                                                                                                      

Externals Have Access to Cases They Shouldn't   

Resolution:                                                                                                    

A new rule has been configured "Set Default Created By User" on the Client Service Case.  This rule replaces the "default to current user" flag on the Created By User field of the client service case, allowing the system to automatically set the CreatedByUserID on the case, resolving an issue where this was not being set properly for External Client Service Case users.

Task 34392                                                                                                                                      

MCP Parent Policy - Activate   

Resolution:                                                                                                    

The command timeout for peo_policy_ui_renew and peo_policy_ui_activate to 600 (5 minutes) in the PEO system, allowing for more activation time and less timeout errors when activating or renewing a policy.

Task 34403                                                                                                                                    

Create Contact From Employee Issues   

Resolution:                                                                                                    

Resolved some issues with this feature, which  was recently released in R37.

  1. The Address on the Contact was defaulting to the home address.  It now defaults to the Org Address by checking the Sync with Organization Address.
  2. The Contact Business Phone was defaulting to the Employee's home phone.  It now defaults to the Organization Business Phone.

Task 34432                                                                                                                                      

Voucher error log missing employee IDs   

Resolution:                                                                                                    

The HRP_API voucher logic has been updated with the following changes:

A billing voucher without a corresponding payroll voucher will now be ignored (it will be logged in the voucher import log but it will not be included in the import file).

Also, when matching a billing voucher to a payroll voucher, the system will only search payroll vouchers for a specific client as VoucherID's may not be unique across clients.

Task 34545                                                                                                                                     

Surchages list only showing inactive surcharge types   

Resolution:                                                                                                    

Fixed an issue where if 'UsesPricing' is set to 'No' on the Client Master page, only inactive surcharges were displaying in the Surcharge List.  The system now does not display surcharges if 'UsesPricing' is set to 'No'. 

Task 34526    ENHANCEMENT                                                                                                                                   

Claim Notes - Comp Claim ID Custom Function   

Resolution:                                                                                                    

A custom function has been added for decoding the Comp Claim ID field on the Claim Notes dataform. This custom function displays the Carrier Claim Number (Date of Injury). If the Carrier Claim Number is blank/null, the system will display CS-{ClientSpaceClaimID} (Date of Injury).

Task 34465        ENHANCEMENT                                                                                                                               

Need to ensure correct Policies are being set in the Admin Workspace   

Resolution:                                                                                                    

Workers Comp Policy Parent form has been updated so that the decode function now displays Policy Title - Effective Date to make it easier to insure you are selecting the appropriate policy.  The following fields have been updated to use the new datatype:

  • Operation State  "Default Comp Policy"

In addition the State Master dataform fields have been updated as well:

  • Default Base Client Policy - now only shows Client policies

Task 34491      ENHANCEMENT                                                                                                                               

Claim Assignment   

Resolution:

If both are changed and they have different claim specialists specified the specialist on the Claim Type will be used.

Task 34299                                                                                                                                   

Pricing Policy vs Coverage Policy 

Resolution:                                                                                                  

A  business rule has been added to the Pricing State that will update the Coverage Policy field to match the Pricing Policy field. This rule has a parameter for a comma delimited list of valid Client Master Statuses for which the rule will trigger and will only trigger if the Pricing Policy field is dirty.

Task 34575                                                                                                                                   

Admin Pricing Not Defaulting    

Resolution:                                                                                                    

Corrected a bad configuration that was causing Admin fees to be calculated prior to some of the values required to calculate the fees being set.  Default pay frequency is now set on the Client Master page by the system prior to calculating Admin fees. 

Task 34546                                                                                                                                      

PrismHR API Voucher Import changes   

Resolution:                                                                                                    

The ImportID on the HRP_API voucher imports has been adusted - they will generate the following ImportID structure: <clientnumber>-V<PrismHR_voucherId>. 

A 'V' will also be appended to the Import ID if it is a void voucher. 

Task 34546                                                                                                                                      

WC Claim ImportID Not Set

Resolution:                                                                                                    

Two new rules have been created on the Comp Claim form:

SE_WarnImportIDChange will warn you that changing the Carrier Claim ID will modify the ImportID as well  

New business rule SetImportID sets (or clears) the ImportID if the Carrier Claim Number is changed
