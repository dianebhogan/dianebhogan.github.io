Task 36239   - ENHANCEMENT   

Secure Calendar Fields   

Resolution:                                                                                                    

Support for optional security entities has been added to Calendar events in ClientSpace, this is standard role security for the following fields with one exception - the organizer field may only be edited prior to save, once saved this field will remain read-only to avoid conflicting information with calendar ICS files created.  The fields (and security entities) in question are:

  • Organizer  -  Calendar_fkOrganizerUserID
  • Meeting Type  -  Calendar_luMeetingType
  • Meeting Category  -  Calendar_luCalendarEventCategory
  • Meeting Status  -  Calendar_luCalendarEventStatus

These new security entities are not included in packaging by default, if you would like to add any or all of these security entities to your installation, contact your NetWise Account manager or log a case in the Extranet.

Task 36259  - ENHANCEMENT                                                                                                                                     

Full File List: Add/Edit/Delete   

Resolution:                                                                                                    

The right menu File attach module has been enhanced to include user security - users can Add/Edit/Delete files if they have the following rights:

  • Dataforms: Wide open - no security applied.
  • Tasks: Current user is either:  
    • Global Admin  
    • Assigned To  
    • Owner  
    • Creator  
    • Or has View rights to the Incident_AllowTaskMaintenance entity.
  • Org, Contact, Activity: User has to have Edit rights to the Org, and the Org can't be locked.

Pricing Console Changes   

Resolution:                                                                                                    

The Pricing Console has been enhanced in NEXT - the following changes have been made:

-More fields have been added to the Company Info section. These fields include

  • Payroll Method (OOI)
  • Current PEO (OOI)
  • Primary Broker (CM)
  • Primary Agent (CM)
  • Health Benefits (CM)
  • SIC Code (CM) 

 -In the Pricing Breakdown section, a line has been added below the Admin Profit that displays the "Percent of Default". If Default Admin Fee is 3.25 and the proposed Admin Fee is 3.1, the Percent of Default will display 95.38%. 

  The formula used to calculate this uses the difference between Default Admin % and Admin %.

- A link has been edded to the pricing console header to display projected commissions for each Commission Record associated with that deal. This like is  a popup table containing a grid displaying the Name, Method and Calculated Total.

-The projected commissions function has been modified as follows:

  • Link must be configured on the Client Master using the custom next function 'pricingConsole.getProjectedCommissions'.  Any Custom Function can be set for classic as it will be ignored.  The link will not appear on the Client Master
  • Projected Commissions will honor the current Batch selected on the Pricing Console when performing calculations

- A new ClientMaster business rule SaveOrgOtherInfo() saves OrgOtherInfo fields from the Pricing Console when editing the ClientMaster  (rule does not apply to Adds)

-When the OrgOtherInfo Payroll Method field is not InHouse, Current PEO field will be visible

Task 36484

Quick Create Issue   

Resolution:                                                                                                    

Resolved an issue in NEXT where changing contract type before the Generate Pricing button had been clicked in a Quick Quote would remove states and codes without warning.  The system now prompts the user to let them know that the states and codes will be removed.  It is necessary to remove the states and codes as pricing factors can be tied to Legal Entities that may be set when Contract Type changes.

Two Factor Authentication   

Resolution:                                                                                                    

Added (optional) Two Factor Authentication to the login page in Next.  This configurable option allows the ClientSpace admin to configure and require 2 factor authentication for their users.  Authentication code may delivered via email or text message (if available and configured on the user) and codes expire after a configurable number of days.  For more information read Configuring Two Factor Authentication For ClientSpace NEXT.
