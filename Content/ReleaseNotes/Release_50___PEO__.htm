Pop Client Alerts   

Resolution:                                                                                                    

The PEO System in NEXT has been enhanced with a Client Alerts banner and related dataform field.  This banner appears at the top of key forms if CM.ClientAlerts field has text and displays the content of the Client Alerts field.  The banner can be closed via an 'X' in the upper right, but will reappear when page is reloaded.  The banner displays the same amount of visible text as CM.ClientAlerts field and allows scrolling if necessary.  Alert banners (and the related Client Master dataform field) are only visible to non-Global admins if the field is secured and the user account has at least "View" rights to the field.  Alert Banners have also been added to the following dataforms:

  • Client Master
  • Client Service Case
  • Client Visits
  • Prospect, Implementation and Client workspace landing pages  


Task 37729                                                                                                                                      

Last Payroll Date   

Resolution:                                                                                                    

Corrected an issue where the header of the Employees form was returning the oldest related pay date instead of the most recent. 


Task 37677                                                                                                                                      

OSHA 300A Proc 

Resolution:                                                                                                    

Removed the summing of Total Hours and Overtime Hours from OSHA 300A proc when research found this was causing hours to be sometimes over-reported.


Task 37735                                                                                                                                      

Update OSHA300A Stored Procedure to account for locations   

Resolution:                                                                                                    

Resolved an issue with the OSHA 300A report that would mis-report employee counts and hours, placing total employees and total hours on each location, instead of breaking these out by location.  Report now breaks this info out by location and appropriately summarizes by workspace when the "Summarize by Account" option is chosen.
