Change in Functionality

Case

Enhancement
42659

Payroll Batch data imported earlier

The PrismHR API now updates the payroll batch throughout the payroll creation process. Previously any Import Queue record created before the payroll was finalized in PrismHR would fail because of missing billing vouchers. This system now creates a new ClientSpace Payroll batch record as soon as the record is created in PrismHR. It then updates the batch status using a scheduled process, so if a batch is stopped at a certain point in the payroll system, we can create notification and workflow events from status changes. Additionally, new statuses and fields were added. See Payroll Batch information.

43044

Workflow Channel Full View Mode

The Full View option for Workflow Channels was deprecated and removed from the application.

43242

Number of fields on a Dataform

The maximum number of fields that can be added to a dataform is now limited to 150, which will improve application performance. At 150 fields, the admin will receive a message stating the maximum number of fields cannot be exceeded, and the Add button will be disabled. See Adding dataform fields.

43251

Reporting Profile maintenance

Business Intelligence reporting profile administration was moved from the Advanced Administration area to System Admin > Security > Reporting Profile, which makes this functionality available to all system administrators.

44396

Client Profitability Report

The client profitability merge was modified (peo_client_profitability_merge) to only show Completed or Finalized batches.

43423

Expired users receive Emails

ClientSpace now includes expired users in email recipient lists.

43620

WC Policy Agent field filtered by Carrier Org

The Agent field on the Workers' Compensation policy now filters the agent list by adding a related field that you can use for selecting another field on the form such as Carrier Organization. When properly configured, the list filters to the agents associated with the selected Organization. When related data is configured using an appropriate Organization field on the form, the list is filtered to the contacts for that Organization. If the field is left blank, all Organization contacts are displayed.

Case

Enhancement

41952

Workers' Comp Policy renewals

Risk Admins can now control whether to update the default policy configurations on the State Master, State Legal Entities, and Contract Type Metadata records when a WC Policy is renewed from the Admin Workspace. This enhancement controls automatic updating as it relates to pricing. For details, see WC Policy renewals.

43687

Re-sale of a terminated client

After a client is terminated, admins can now resell a client.

  • When the Client Master is in Terminated status, a Resale workflow link displays. When you click Resale, the termed batch type changes to Archived. The Client Master status changes to Prospect.
  • The associated Pricing Batch, Pricing States, and Pricing Codes are updated with a status of Terminated.
  • The new cloned batch has the type RFP and is marked as Current Batch. The Current Batch flag is set on the new RFP batch and cleared on the old one.
  • The Pricing Console does not show Archived (previous batches that are not the newly cloned batch) batches. Batches can still be searched through the workspace from the dataform tiles or links.
  • If pricing is set to No, then there is nothing to clone.

To implement this feature, please log an Extranet case.

43739

Benefits Batch: Apply Plan Group

When a Plan Group is applied to the Benefits Batch, using the Apply Plan Group link, the Benefits Batch form now updates to show the applied Offered Benefits Plans count on the Offered Benefits Plan link.

43837

Pricing Console fields

The following Pricing fields now adhere to the biz_pricing_submitted_batch security entity. They are editable as other pricing fields. The fields are now unlocked when Pricing is unlocked and read-only after submitted unless the user is in a role with the appropriate rights to the biz_pricing_submitted_batch security entity.

On the Pricing State (gen_PricingBatchState):

  • TDI (TDI)
  • Current SUTA (SUTA)

On Pricing Batch (gen_PricingBatch):

  • WC Mod Method (luWCModMethod)
  • WC Billing Modifier (BillingClientModifier)
  • Include Expense Constant (IncludeExpenseConstantInRate)

Advanced Administration

Case

Enhancement

40941

Salesforce API: defaultorgowner

The Salesforce API now accepts a new API parameter defaultorgowner. Previously, if an organization was created in ClientSpace from a Salesforce import and the Salesforce Owner did not exist as a ClientSpace user, the Assigned To on the organization would default to blank. The parameter defaultorgowner allows the admin to specify a LoginID (username) as the default. The configured user is set as the owner of organizations imported from Salesforce where the owner specified in Salesforce does not have a ClientSpace user account. See Additional parameters.

41756

Create Discussion: Business logic

The Client Service Case has a new rule Create Discussion that automatically creates a case discussion based on configured triggers. By default, the rule is inactive. To use, it requires configuration of the following parameters:

  • EmailTemplateCode: The template code (must start with CSC) of the system email template for replying to discussion threads.
  • InitialRecipients: A comma-separated list of User fields on the associated case or employee form for the discussion response. Field names should follow the tableName.fieldName format (gen_ClientServiceCase.BrokerContact, for example).

After you configure and activate, the rule creates a new discussion and triggers an automated email to the initial recipients based on the rule parameters. See Configuring Create Discussion business rules.

43289

PrismHR API: Import errors

When logging errors from imports, the Source field is now set to Imports. This enhancement enables reporting on the count of import errors per site.

44441

Prism API: ExcludeUserNames

The PrismHR API configuration has a new parameter ExcludeUserNames, to exclude certain users during the user security import. The new parameter is an exclude filter that looks for a username that starts with the ExcludeUserNames value, such as abc. For usernames that begin with the filter value, the queue record is stamped with the message "User '<userID>' has excluded name, this record will not be processed."

44472

Prism API: Benefits Events

The PrismHR API now suppresses a benefits event that does not contain a matching Client ID.

Case

Enhancement

42381

Case Discussion Replies

Recipients of a Case Discussion message in their email mailbox (for example, Gmail) can now reply above the line from any discussion reply. The Reply Above This Line logic uses Email Template codes CSCDISC and CSCDISCNEW. While the email templates (CSCDISC and CSCDISCNEW) themselves do not require Email Integration, the reply above the line functionality does. For Reply Above This Line to work, both CSCDISC and CSCDISCNEW must have a From Email Override address that matches an Active Email Integration mailbox. For details, see Configuring Case Discussion: Reply Above This Line.

43131

Employee Name in EPLI Module Quick Search

The EPLI Claims search was modified to more closely match the WC Claims dashboard. Employee Last and First Name were added to the Quick Search on the dashboard, as well as the ability to search by Carrier Claim number.

43132

Delete button on EPLI Module

The EPLI dashboard now has a delete button that is available for any user associated with a role containing the security entity biz_epliclaim_delete with View rights. This button allows designated users with appropriate security rights to delete EPLI claims dataform records.

43695

Workflow Channels and Email Templates: Date field Operator for Aging

Email Template and Workflow Channel triggering were enhanced with the new Date field operator for Aging. These new operators allow the system administrator to configure triggers based on a period before a specific date or a date on a field. For details, see Workflow Channels Step 4: Conditions and Email Templates Step 4 Conditions.

43807

Active Employee Count on Payroll Batch

The Payroll Batch dataform now includes more accurate definitions of the employee counts by batch. This enhancement includes:

  • A new fieldset for General Information that contains the existing Payroll Batch fields.
  • A new fieldset on the Payroll Batch dataform called Employee Counts.

The Employee Counts fieldset only displays for Batch Types of Regular with the following fields:

  • FT Active WSE Count
  • PT Active WSE Count
  • FT Terminated WSE Count
  • PT Terminated WSE Count
  • FT Leave WSE Count
  • PT Leave WSE Count

When a payroll batch record is inserted with the status of Completed, a business rule runs to stamp the counts from the ClientSpace Employment records (calculation of FT/PT and Status Classification). This rule only runs when Batch Type = Regular.

 

There is a new field on the Payroll Batch for Pay Frequency (lookup core, lookup Group = PayFrequency). This field displays when the Batch Type = Regular. The values for this field are calculated based on the difference between the start date and end date of the payroll batch.

See Payroll Batch information.

43901

Field Display Rules: Comma-separated values

Manage Field Display rules now use a comma-delimited list of values. Additionally, Multi Lookups in field display now recognize blank values, to include delimited lists (for example importcomplete,,onhold). For more information about managing field display in your ClientSpace installation, see Configuring field display with multiple values.

44293

Benefit Classification

Benefits dataforms were updated to more closely mirror the terminology used in PrismHR.

  • On the Apply Plan Group modal, the Classification field is now Benefit Group.
  • The rule title for HE_ClassificationCodeUnique on the Benefit Groups is Benefit Group Code Unique.
  • The description for HE_DuplicateOfferedPlan on the Offered Benefits Plan is Benefit Group rather than Classification.
  • The description for SetImportID on the Offered Benefits Plan uses Benefit Group anywhere it previously showed Classification.
  • The rule title and the description for HE_DuplicateCPCClassification on the Offered Benefits Plan uses Benefit Group anywhere it previously showed Classification.

44500

ClientSpace and Employee Portal integration enhancements

In the PrismHR June Quarterly Release, the Employee Portal Support Center will be delivered, which relies on data from ClientSpace. When a Client Service Case is created in ClientSpace from the Employee Portal, the Source is marked as Employee Portal, and a new, read-only field displays for the Reported By Employee. The new field captures the name of the worksite manager or worksite employee who submits the case. This enhancement requires additional configuration in . See Configuring ClientSpace to work with the Employee Support Center.

 

The More Search enhancement enables searching by the Reported By Employee on the Client Service Case dataform searches. Reported By (Employee) is also included in the More Search for Cases on a dataform search, which allows users to search for cases submitted by a specific employee. This enhancement ensures that the same case results display in ClientSpace and the Employee Portal.

Enhancement
43998

PrismHR API Export: Convert to REST

The Export methods from the PrismHR API for creating Clients and the related locations and employees were updated to use REST API technology, which does not rely on a shared WSDL system for updates. This enhancement means that as PrismHR adds new methods for accessing PrismHR data, the ClientSpace code can be updated to take advantage of these new methods without the need to download a new WSDL package containing the method names.

44281

TableName search for tblOrganization on Manage Rules

The TableName search in More Search now includes non-dataform objects where appropriate. Note: For future release.
44372

Remove Classic Code

In our continuing effort to sunset our classic application, more classic functionality was removed from the application to include:

  • Dataforms: the ability to access dataforms from the classic TOC
  • Search: the ability to search on multiforms from the classic TOC
  • Trigger
  • webusercontrols_net
  • Stylesheets

These functions were either removed or migrated into our cross-browser version of the application.

Prism API Initial Imports: Client Status

Initial Imports now only retrieve ClientSpace Clients in Client, Terminated, and PendingTermination status for the following:

  • Initial Location, scheduled process: HRPImportInitialLocations
