Import to Lookup Values

New functionality was added to allow imports to the lookup table for existing lookup groups. This includes the following:

A new Import_Lookup dataform used for processing the import.

A new Business rule on the Import_Lookup for CreateLookup, which handles the custom logic for inserting and validating the lookup entries during import.

This system allows you to insert and update lookup values with the following caveats:

The matching lookup group must already exist in the system.

System lookups do not allow updates.

Salesforce API converted to REST

The Salesforce API has been updated to v3 to use REST API technology, which does not rely on a shared WSDL system for updates. This enhancement means that as Salesforce adds new methods for accessing Salesforce data, the ClientSpace code can be updated to take advantage of these new methods without the need to download a new WSDL package containing the method names. The existing ClientSpace API configuration must be modified to take advantage of REST because the linking records that tie the Salesforce and ClientSpace records together are API Configuration specific, and creating a new API Configuration generates duplicate records. 42295

Prism Import enhancements

Resolved an issue with the PrismHR Imports. If PrismHR contains a record and the ClientSpace Client Master does not exist, an error was logged. The following changes were made to refactor this system.

Initial Process changes

Scheduled process PrismHR Import Initial is now deprecated and should NO LONGER be used (the scheduled process is now scripted out as inactive).

Scheduled process PrismHR Import Queue Initial Clients obtains a list of all clients available through the API, creates Import records for each, and sets the Sync with PrismHR flag on the Client Master for each Client imported (company initial import map HRPAPI_initial_company*.csv MUST be active). The company initial import map now contains the Sync with PrismHR column (scripted out active) to allow the setting of that flag on the Client Master.

Scheduled process PrismHR Import Initial Sync then processes the Clients to Sync and import remaining ACTIVE Import Map data (employee, employment, location).

The PrismHR Automation process is the preferred method to use when attempting to sync all client data. It handles the newer initial import process. It has logic to turn on/off respective scheduled processes to prevent conflicts (the Prism Events scheduled process is not included in this; it can execute while initials are running).

For more information, see Configuring the PrismHR API for Initial Imports .

Ongoing Process changes

Scheduled process PrismHR Import Ongoing was modified to prevent importing ongoing data for Clients that do not exist in ClientSpace.

If the ongoing event is a Company event (UPDATE or CREATE), that event is processed and an Import Queue record is created.

If the Event is a Security schema event, that Event is processed, and an Import Queue record is created.

All other Events test whether the associated Client exists; if not, the ongoing event is skipped (no Import Queue record created and a Business Intelligence report entry is logged "---Create Import Queue Events, ClientNumber not found in ClientSpace, this event will not be processed").

For more information, see Configuring the PrismHR API for Ongoing Imports .
