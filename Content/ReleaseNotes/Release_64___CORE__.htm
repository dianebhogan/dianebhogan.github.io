Release 64 ( CORE )

Sprint Began: 1/3/2018    Sprint Ended: 1/31/2018


The following items have been completed in this release:


Company Holiday NEXT conversion - ENHANCEMENT

The Company Holiday management system has been converted to NEXT. This system consists of a Search and Detail page.

  • Search Page:  
    • Quick Search:  
      • Holiday Name (starts with)
  • More Search  
    • Active/Archived/All radio button  
    • Holiday Date (date range)  
    • Holiday Year (integer)
  • Search Result Columns  
    • Holiday Date  
    • Holiday Name
  • Detail Page:  
    • Holiday Name (textbox, required) 
    • Holiday Date (date, required) 
    • Active (checkbox)


Document Search Category/Description Search parameter - ENHANCEMENT

"File Category" and "File Description" have been added as Search parameters to the "More" search area of the Document Search dashboard and removed from the Quicksearch area.

The "Status" search parameter in More search has been renamed to "Show". This is more consistent with other areas that work like this.

The "Entity Type" search parameter has been renamed to "Form" in the More Search, as well as in the column list that displays the data.

Quick Search now works with the Keyword parameter only.

Date Uploaded has been set as the First Column.
Ordering of forms in the "Form" dropdown in More Search has been updated to list forms alphabetically for ease of use.


Dataform Subscibers should be CC recipients - ENHANCEMENT

Resolved an issue where dataform subscribers are receiving notifications as "To" recipients. Dataform subscribers now are added as "CC" recipients to minimize the amount of clutter in the Notifications inbox


Field Display configuration - ENHANCEMENT

Field Display configuration has been enhanced to allow multiple field display triggering conditions against the same field / fieldset. So for example the system will now allow you to configure the following

When First Name == "Tiny" then the Some_Other Field will display.
When Last Name == "Tim" then the Some_Other Field will display.

You can also still configure multiple trigger conditions on a field using seperating commas so

When First Name == "Tiny" then the Some_Other Field will display.
When Last Name == "Tim,Bill,John" then the Some_Other Field will display.

These conditions are treated as "OR" conditions, so that when FirstName = 'Tiny" OR when LastName = "Tim" OR "Bill" OR "John" the field will display.

Due to the flexibility of this system it is possible to configure conflicting conditions, so use caution when setting up these multiple field display conditions.


Email Template Recipients (Email Element Type) - ENHANCEMENT

The Email template system has been enhanced to accept email type fields from the triggering dataform as a recipient record.


Followup in Home ENHANCEMENT

The following changes have been made to activities in the application:
The label on the Follow-up Home Page panel has been renamed to "Last Activity" instead of "Last Call".


On "Edit" of an Existing follow-up – the “Activity Date” is no longer set to today’s date, , all other Activity Type's result in the Activity Date being set to today's date.


Error when dataform record no longer exists

Resolved an issue where if there are references to a dataform which no longer exists, such as in the watch widget, attempting to navigate to the missing dataform would throw a generic system error.

The system will now alert the user that "The dataform you requested was not found or no longer exists. If you navigated here from Business Intelligence, please contact the report owner to have the link updated. "


Error when moving Cases to different Workspace 

Resolved an issue where under certain circumstances, while moving a dataform from one workspace to another the system would generate a foreign key exception when there were no foreign keys to the dataform and not allow the dataform to be moved.
