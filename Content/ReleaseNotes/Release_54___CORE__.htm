Release 54 ( CORE )

Sprint Began: 4/17/2017     Sprint Ended: 5/5/2017

The following items have been completed in this release:


Time Tracking on Activities   ENHANCEMENT

Time tracking has been added to Activities in NEXT, allowing you to now record time for any calls, emails, followups or other activities you record.  Time tracking is available from the Right "Action Items" menu in the activity detail.


Upgrade Kendo UI

Kendo UI, the third party add-in we use for grid controls, calendars and rich text editor controls has been upgraded to the latest version to resolve an issue where an error would be thrown during drag and drop of control items, such as columns in a list or calendar items. 


Contact search filters   ENHANCEMENT

The Organization Assigned To field has been added as a "More" filter parameter on the Contact Search dashboard in NEXT.  Contact search records may now be filtered by this field.

Quick search first name (NEXT)

Contact First Name has been added as a Quick Search parameter on the Contact Search dashboard in NEXT.

Convert the Document Search to Next   ENHANCEMENT

Document Search functionality has been added to ClientSpace NEXT, including the ability  to search by workspace, document type, keyword and Active/Inactive/All.

Min/Max validation message

The Min / Max validation of certain fields has been enhanced - now instead of displaying the upper and lower limits of the field the system now simply tells you which value you exceeded and what the max value is


Multi selects open as dirty (NEXT)

Resolved an issue where multi-lookup values were stored in the database in such a way that when the multi-lookup was loaded on opening the dataform the field would appear to be "dirty" as if changes had been made.  System now respects the true state of the field so the dataform so it no longer appears fields are "dirty" on the page when it is first opened


Org Activity Summary bug (NEXT)

Resolved an issue where when viewing Org Activity Summary for an Org with no Activities, if the user clicked the expand summary button then attempts to collapse the summary, a javascript error would be thrown.  Page will now display a message that there are no associated activities.


Possible Module Admin Bug

Fixed a bug in the Module Admin area where modules would disappear and not be available again if all of the module types were deselected.


Mobile login enhancements

Our login page has been enhanced to detect mobile devices and automatically re-direct the browser to ClientSpace NEXT.  Check to be sure that you are not navigating to one of our older Classic mobile apps for Contact or Organization search (the url should contain the word "mobile").   If you are still experiencing issues on a mobile device, clear your browsers history as old information may be cached in the browser.


Email mask issue

Fixed a bug in the email template functionality that was defaulting the "From" user to the default app setting email regardless of whether the "From Override" was set.   Application will now:

  • Attempt to retrieve a "From" user from a recipient stored procedure configured on the template
  • If none is found the system will use the "From Override" configured on the template.
  • If this is somehow blank the system will fall back to the App Admin Email configured in App settings.

Expose fields in Module Admin (NEXT) ENHANCEMENT

Both the "Target" field and the "ClientSpace Next" checkbox have been exposed to Global Admins in the UI, allowing Global administrators to control these settings.

Target - this determines if a configured module will open within the frame of the application or in a new tab. 

  • main - open the module within the application frame
  • new - open module in new tab

ClientSpace NEXT - this checkbox determines whether the module is available in NEXT

  • Checked - ClientSpace Next module
  • Unchecked - ClientSpace Classic module
