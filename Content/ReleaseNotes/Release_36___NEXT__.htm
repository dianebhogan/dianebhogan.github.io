Task 33735                                                                                                                                      

Dataform Move Change

Resolution:                                                                                                    

The system has been modified to enhance the transition when moving dataforms within the application.  When a dataform is 'moved' to a new workspace, it will still be visible to the user in the case of an 'Apply' or if edited from a multi-form list.  The record can still be opened, edited, saved, etc.  If the list is refreshed, however the moved record will no longer appear.

If however, when the record is moved, the current user does not have permission to access the new workspace, they will receive a pwDataformMoveNoPermissionException message, indicating that the requested data is no longer in this workspace and that they do not have permission to access the new workspace.

This change was implemented for COI Certificate holders, but applies to any dataform by changing the 'Workspace' field in the admin fieldset.

File attachments are also moved with the record.
