Release 47 ( PEO )

Product Version: PEO – Release 47 Notes - The following Tasks have been completed/fixed in this release.


Task 37264                                                                                                                                      

Client Dashboard

The code which selects records for the Client Dashboard in NEXT has been re-factored to improve performance.  This helps to resolve performance issues for installations with large numbers of Client Workspaces. 


Task 37265                                                                                                                                      

Refactor [peo_cs_prospect_s_landing_activities]   

Resolution:                                                                                                    

The code which selects activities when opening a workspace landing page has been optimized to improve performance when opening a Workspace with many activities in NEXT.


Task 37266    - ENHANCEMENT                                                                                                                                         

PrismHR API, upgrade to v1.6 and improve initial import process   

Resolution:                                                                                                    

The PrismHR API has been upgraded to version 1.6 and a new initial import process has been created which improves performance of large scale initial imports by batching and queueing all available import files, then allows you to configure how many records are processed at one time when the HRP Initial Import process is run.  For more information read the KnowledgeSpace article Configuring The PrismHR API For Initial Imports .

Task 37267

Task 37166    - ENHANCEMENT                                                                                              Click Here for the Video          

Default Client Teams   

Resolution:                                                                                                    

ClientSpace administration has been enhanced through the addition of the ability to default Client Team settings based on Sales Region.

A new Default Client Team dataform has been added to the Admin Workspace which mirrors the Client Team dataform fields and adds a Sales Region Selection criteria field.  This default client team form will be used to set the client team field on workspaces with a matching sales region in two ways.

  • If the sales region of the Client Master dataform is updated and a Default Client Team form exists in the client workspace for that region, the workspace client team will be set to match the default for the region except where a blank field exists on the Default team form and the field has been completed on the Workspace team.
  • Changes made at the default team level can also be pushed across all workspaces in the Sales Region of the default team dataform.  For example - lets say Bob Jones the Sales admin for a Region, such as Southeast leaves the company and is replaced by Andy Smith, the new admin for that region.  You can now make the change on the Default Team form and queue the change using the "Queue Changes" link on the Default team form, then a scheduled process will find any Workspaces where the Sales Region matches the Sales region of the Queued form and push the updated fields from the default form in the Admin Workspace to the Client Team form in the Client Workspaces.  The exception to this again being if the field is blank on the default and filled at the client level.


Task 37168  - ENHANCEMENT                                                                                                                                          

Update Client Team Member business rule   

Resolution:                                                                                                    

A new business rule  has been added to the  Client Master called UpdateClientTeamMember. This rule has  two required parameters, Client Team field, and LoginID. When this rule is executed, it will set the field on the Client Team to the correct user and add that user to the workspace via business logic.

Task 37249 (NEXT)  - ENHANCEMENT                                                                                                                                         

Quick Create Maximum   

Resolution:                                                                                                    

Resolved an issue where adding too many pricing states and codes at one time during the initial Quick Add process caused an exception which states : 'request is too large to be deserialized'.   Quick Add is now limited to 15 state and code combinations at which time you will be prompted to generate pricing with the message "Quick Add is limited to 15 Codes. Generate Pricing with these Codes and Add any additional Codes through the Pricing Console.".  Indvidual states and codes may be added after pricing is generated and upon save of each new state/code combination pricing will be recalculated.
