Enhancements

Enhancements are grouped as Change in Functionality, ClientSpace Premium (ClientSpace with additional modules), Advanced Administration, and General.

Change in Functionality

Case

Enhancement
39118

Case Discussion Reply Email Template: Attachments

Static attachments added to email templates with prefix CSCRPLY are now included in Case Discussion replies. Adding attachments to the template can save time and streamline processes for managing cases. For more information, see CSCRPLY: Case Discussion Reply Template.
38479

Widget Layouts: Published option

Widget layouts can now be published and unpublished using the Published option from the Widget Layouts list. See Adding widget layouts.

43527

DocuSign Email Template: Required tags

DocuSign Email Templates now offer a Required tag column. This option indicates if the tag is required during the signing process and helps to streamline the process if the field is not applicable. For details, see Configuring DocuSign Email Templates Step 3: Addresses.

43646

Importing into existing Lookups with no ImportIDs

When importing into existing Lookups with no ImportIDs, ClientSpace now matches records based on the fkLookupGroupID and the Code value. It then updates the correct records accordingly.

44002

Out of Office

Out of Office functionality now sends notifications to the Inbox with no additional configuration. This enhancement eliminates the requirement to enable Email on the profile of the user configuring out of office, or the selected backup user. See Enabling Out of Office option.

42461

Dataform admin: Related items

You can now view related items for dataforms from Dataform Administration, in the Action Center. This information can help you avoid deleting items that are essential, such as triggers and rules. See Viewing dataforms and their related items.

Advanced Administration

Case

Enhancement
44323

PrismHR API: Employee Sync With Prism

A new scheduled process PrismHR Employee Sync With Prism synchronizes Employees with PrismHR. It validates that Employees in ClientSpace (for Companies that have Client Numbers in PrismHR) are in PrismHR. If the Employee record is not in PrismHR, it sets the Employment status to Deleted and inactivates the Employee record. This process is scheduled to run once and then inactivates. See Scheduled processes.

43961

PrismHR API Import

The PrismHR API Import now sets the Parent Company field to the ClientSpace Org primary key value, if applicable. If the parent is not found in ClientSpace, an import error message is generated.

45747

PrismHR API: Payroll Batch

The HRPAPI_payroll_batch*.csv import map was enhanced with a new field mapping for Payroll Processor. The Ongoing process populates the field with a UserID from PrismHR. See Payroll Batch Import Mappings.

General Enhancements

Case

Enhancement
43213, 43214, 43215

Case Creation: Communication Method

Cases created from an Email, Email Integration, and Workflow Channels, now set the Communication Method to reflect Email, Integration, or Workflow Channel. The field is then set to read-only. See Case creation methods.
46026

PrismHR Ongoing Import: Active-only

For PrismHR API Ongoing events, ClientSpace ignores inactive Client Master records. When events are retrieved and converted into Import Queue records, for non-Company ongoing import maps, the Client associated with the event must be in ClientSpace. If not, the event is discarded, and an Import Queue record is not created for that event. See Importing to [%=MyVariables.ProductName%].

45833

PrismHR API Ongoing Import: No Employee

When an Employee record is not found in PrismHR, the associated Event Import Queue record is marked Completed with a note indicating the Employee was removed from PrismHR. See Importing to [%=MyVariables.ProductName%].

46334

PrismHR API: Deleted Client events

If a DELETE event is received for a Client, the Import Queue record is automatically Completed without attempting to retrieve the Client details. No import row is generated. The Queue record is marked with a comment indicating the Client has been deleted from PrismHR. See Importing to [%=MyVariables.ProductName%].

43819

Client Profitability Report: Tasks

The Client Profitability Report now includes the time associated with Tasks. The total time spent on Tasks is included in the Cost of Service total. See Generating the Client Profitability Report.

44456

Home Page: Widget Layouts

The home page now offers users the option to view any Home Widget Layout to which they have access and pin a layout as their default. Additionally, you can customize the header for Business Intelligence Widgets with new widget settings Url and Link Name. See Home Page widgets.

44657

Creation source

A new method was created to handle the creation of the rule to set the Communication Method field for Cases 43213, 43214, and 43215. This method identifies the source of a dataform creation.

43744

Email Templates: Immediate Supervisor

ImmediateSupervisor is now a recipient option when Source is Address Source Procedure. When ImmediateSupervisor is selected, the supervisor listed on the user record for the Assigned To user is notified. If there is no Immediate Supervisor on record for the Assigned To user, notifications are sent to all other specified users in the email template. See Configuring Email Templates Step 3 Addresses.

40241

Pricing Console

Pricing Console is now an available Target Type for widget layout configurations. This enhancement can improve navigation or the ability to present more data on the Pricing Console.

43250

Business Intelligence Dashboard

The Business Intelligence Dashboard now includes Last Run User and Last Design User. Additionally, Last Run By and Last Updated By are available in More Search.

43258

Time Zone for Log Date/Time Stamps

Log fields are now stamped with the user’s time zone. Existing log field dates remain as is and do not reflect the user’s time zone.

43256

Field Audit Trail

The Workspace and Dataform identifier now displays on the Field Audit Trail index, providing the admin user insight into who changed what dataform and in which workspace.

44671

External Source Linking

External Source Linking, used by some APIs to connect ClientSpace records to the adjoining records in external systems, is now accessible from System Admin > Imports > External Source Linking. It is also available from any dataform in Action Center > More options > External Source Linking. You can perform searches, Add, Edit, and Delete records. See External source linking.

Removed references to unused tables in PrimaryAgentChange business rule.

Resequenced the order of the PrismHR API Import Map processing, resolving an issue where imports could sometimes happen out of order (for example, Employment before Employee).

PrismHR API: Imports

For Employment ongoing, when retrieving location data for the Employee, if the Employee’s location code contains a single quote, the process now pulls in the location information. Before this change, the BI report would error, stating an invalid token in XPath.

39316

Admin security tests

Changes were applied to tighten up security in system admin areas of ClientSpace. This change only allows the specified users to access admin functions.

  • Benefit Plan Model
