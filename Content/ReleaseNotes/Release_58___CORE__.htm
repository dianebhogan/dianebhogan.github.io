Release 58 ( CORE )

Sprint Began: 07/18/2017     Sprint Ended: 08/11/2017


The following items have been completed in this release:


Subscriber Email Template -  ENHANCEMENT

Dataform properties have been enhanced with an added checkbox 'Disable Subscription Email' to dataform properties configuration page

  • When unchecked {default}, standard core dataform notifications will be sent to dataform Subscribers (current functionality)
  • When checked, standard core dataform notifications will not occur
  • Email Templates can be used in place of the dataform notifications

  - Select "Dataform Subscribers" as Recipients (may only be selected once)
  - If a Trigger Field is selected, notification will be sent only if that field has been changed prior to  dataform save
  - If a Trigger Field is not selected, notification will be sent on every save of the dataform
  - Dataform Notifications special attributes will NOT be available

For more information on configuring triggered email template notifications in ClientSpace, reference Configuring Email Template Triggering in ClientSpace


Merge Proc List -  ENHANCEMENT

The Merge Stored Procedure list has been enhanced to show anything configured on another row or any stored procedures that have a name like 'peo_merge' or like 'z_%merge' (% is a wildcard). This will allow all merge stored procedures to be displayed in the list whether or not they are currently in use by a merge.


Widget Issues 

Resolved a number of issues with Widgets in NEXT including:

  • Workspace field list is empty in field/value trigger condition dropdown
  • Workspace fields form template ID and group ID need to be eligible for conditions
  • BI widget "open" link in widget header is broken
  • Widget continuously reloads on an iPad


Remove iTextSharp from pdf merge code - ENHANCEMENT

The iTextSharp library that ClientSpace was using for PDF merges has been removed, and replaced with the Aspose library that we already use for Word merges. The iTextSharp library has been removed from all code that directly performs PDF merges as well as the  classic admin functionality that generates the SQL with the PDF field names, and the "Get Field Info" link.


Email to Case -  ENHANCEMENT

A new "IsDefaultAlias" checkbox has been added to the API configuration area.

When processing emails, if a matching Alias record cannot be located, the system will now use the API configuration to locate an Alias with "IsDefaultAlias" checked to define the workspace. 


Move ActiveReports into Separate Web - ENHANCEMENT

The ActiveReports engine which runs Classic - non-BI reporting has been moved into a Separate Web so that in the future we will be able to change the classic app pool to x64.


Google Maps Button - ENHANCEMENT

Google "Map It" functionality has been added to a button in the Org and Contact action center. This uses the Address fields from the Org/Contact record and searches maps.google.com in a new window.


Quick "Share Workspace" Button - ENHANCEMENT

A new button has been added to the Action Center on Workspace landing pages to "Copy Link to Clipboard". This copies the "linkto" url for that workspace directly to the clipboard of the local computer.


Secure "Save Anyways" button on Potential Dups - NEXT - ENHANCEMENT

Security has been added to the "Save Anyway" button on the Organization potential Dupe screen:  Security will be set by checking if the user has View rights to the "CRM_CanSaveDuplicates" security entity.  This gives the Global Admin the ability to secure whether duplicate organizations may be saved or not.  A script will need to be run on the installation to insert the custom Security entity - if you are interested in this functionality contact your account manager.


Upgrade Izenda BI dll to latest version - 6.10.0.20 - ENHANCEMENT

The business intelligence dll from Izenda has been updated to their latest release - for more detailed information about this version update, review the Izenda Release Notes.


Multiple Workspace Issue - Selecting Desired Workspace 

Fixed an issue where in the Chrome browser  when selecting items in a dropdown (a workspace selector for example), and you had two items in the dropdown that have the same Decode value, or the same Decode value but only differing by case, but these items have different Code values, when you attempted to select the second of the duplicate items in the list, it would end up reverting the original selection once you tabbed off of the field.


User Datatypes - ENHANCEMENT

All user datatypes have been enhanced to include Middle Initial in all User type ahead stored procedures.


External Calendar Syncing Errors 

Resolved an issue where external calendar events with a date outside of acceptable ranges would cause the entire calendar sync operation to fail.  System now will replace the bad values recurrence ids coming in the external calendar provided by the user.  Allowing the rest of the external calendar to sync normally.


Zip Code Datatype Error 

Resolved an issue where typing a plus sign (+) into a zipcode field would cause the system to generate an exception.


Add contact bug

Resolved an issue where the Contact Action Center was not properly handing the case of receiving an OrganizationID of zero. This can happen when attempting to add a Contact from the Contacts module, there is no Organization in this case.


Type Ahead Decode 

Fixed an issue where if the value stored in a type ahead lookup field does not have a valid decode, the system shows a blank field even though there is data stored in the field.  For that field the system will now display what is stored in the field as well as the Valid lookup values for that field.


Eternal Loading Screen Issue 

Resolved an issue where when attempting to add a new comp claim in NEXT and you click ADD and select a workspace  you could get a broken loading screen (unmoving swirly) that doesn't go away if there was a security rights issue on a tabbed dataform.


Make Email a Hyperlink on Contact Search Module - ENHANCEMENT 

The email column in the Contacts dashboard list has been enhanced as a clickable link.  Clicking this link will open the system default mail application, filling in the "To" field on the email with the email address that was clicked.


Default Active Checkbox on Import Map  - ENHANCEMENT 

The Import map detail configuration has been enhanced to make it easier to use:

  • The Active checkbox now defaults to 'checked' when adding a new column
  • The Project Type selector now defaults to the Import Map Header dataform selection when adding new column (add or from Save and New)
  • The Map Header Detail Project Type and Field selection have been Refactored to make them simpler dataform and field selectors instead of the annoying complicated dialog.
  • The Import Map Header Detail tab displays and sorts based on the Dataform Field Label (not Attribute Name)


Outlook Events in the past 

Previously calendar syncing deletes all events and then pulls the new external calendar file.  The system has been updated to only delete external calendar items more than 30 days old.


User Profile Permissions for External Users 

The following items have been hidden from External Users when accessing their User Profile in Next:

  • Default Workspace Search
  • All fields in the Settings Fieldset with the exception of the time zone ones. We might need to rearrange some thing.


Expose "IsFlattened" through UI

The "IsFlattened" setting for PDF merges has been added as an option in Merge Admin.  This defaults on new merges to unchecked, allowing for form fields to be completed or data overwritten.  Checking the "IsFlattened" box will merge produce a merged form that is read-only

Add "Save & New" option when adding dataform fields  - ENHANCEMENT

A 'Save & New' button has been added when adding dataform fields, making it easier than ever to add multiple new fields to a dataform.


New Next Exception Log  - ENHANCEMENT

The Exception log from Admin Settings has been converted to ClientSpace NEXT  this exception log can be found under System Admin in the user name drop down displayed in the upper right corner in NEXT.
