Release 56 ( CORE )

Sprint Began: 6/5/2017     Sprint Ended: 6/26/2017


The following items have been completed in this release:

CRM Search Optimization - ENHANCEMENT

The CRM search lists have been optimized so that they will not search on load, giving the user the opportunity to add search parameters - this is to avoid issues where large amounts of records cause the list population to perform poorly.  A message will instead be displayed, instructing the user to "Please enter search criteria and click the search button to see results."  in red text, if no results are returned from the search this will be replaced with the standard "No Records Found" message. 

Reporting Capabilities to Filter for Business Day/Week - ENHANCEMENT

Added the ability to filter in Business Intelligence Reporting for Business Days. 

The following values have been added as options to the dropdown when the 'In Time Period' operator is selected for filtering:
- Previous Business Day
- Previous Business Week
- Current Business Week

Business Days are configurable via a hidden tab in App settings but default  currently to Monday through Friday.  Holidays cannot be accounted for - if you would like to adjust the business week schedule for automated BI reports, contact your NetWise account manager.


Email Integration, custom business objects - ENHANCEMENT

Enhanced the email integration process to cache the assemblies it loads via reflection, instead of loading them over and over, which can cause memory issues.


Email Integration Error logging - ENHANCEMENT

Email integration error logging has been enhanced to include more useful information, such as the related API ID when exceptions are logged for processing failures.


Upgrade Izenda DLL - ENHANCEMENT

The Izenda Business Intelligence application has been upgraded to version 6.10.0.18 of the Izenda DLL.  Release notes for this version can be found here.
