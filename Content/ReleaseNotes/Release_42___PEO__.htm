Release 42 ( PEO )

Product Version: PEO – Release 42 Notes - The following Tasks have been completed/fixed in this release.

Task 35809  - ENHANCEMENT                                                                                               Click here for the Video

Prism API needs to be able to work with more than one prism installation.   

Resolution:                                                                                                    

The PrismHR API integration functionality in ClientSpace has been enhanced to allow one ClientSpace installation to receive data feeds from multiple PrismHR installations based on individual API configurations. 

Task 35904                                                                                                                                      

State field on ClientMaster reverts to Lookup Default   

Resolution:                                                                                                    

Resolved an issue where under certain circumstances the US States lookup on the Client Master was always defaulting to the lookup default on Save regardless of whether the field contained a value. 

Task 35912                                                                                                                                     

Pricing console required fields   

Resolution:                                                                                                    

Resolved an issue in NEXT where pricing fields on the Quick Create popup were not required, despite those same fields being marked as required on the associated dataform. 

Task 22428                                                                                                                                      

Sales_Rep Access   

Resolution:                                                                                                    

Resolved an issue where security entity mappings for the Quick Edit portion of the Pricing Console were not being respected.  All Portions of the Pricing Console now respect the View / Add / Edit functionality of Role security.

Task 36075      - ENHANCEMENT                                                                                                                                  

Client Dashboard Module with Workspace (DBA)   

Resolution:                                                                                                    

Many of the dashboards in NEXT have been enhanced - adding Workspace Name, Organization Name and DBA to the existing Quick Search parameters.  Organization Name is also now available to add as a list column The following dashboards have been enhanced:

  • Client Pricing
  • Employee Visits
  • UE (UI) Claim
  • Cases
  • Offered Benefit Plan
  • Certificate of Insurance
  • Comp Claim
  • EPLI
  • Client Rescue
  • Commission
  • WC Policy  

Task 35868     -ENHANCEMENT                                                                                                                                         

'Pay Frequency' Change after Client Status   

Resolution:                                                                                                    

A new business rule "SetPayFrequencyOnActivation"  has been created for both the Client Master and Pricing Batch dataforms which is triggered by the HdrAction of "ActivateNow" or via the Scheduled Process "ActivatePending".   This rule sets the Client Master and Pricing Batch PayFrequency fields to the Pay Frequency of the pricing comp code with the most total employees, and if all codes have the same employee count, then the code with the most Gross Payroll will be used. 

Task 35875       - ENHANCEMENT                                                                                                                                   

Employee Status Shows Not Set   

Resolution:                                                                                                    

A checkbox has been added to the Employment record which designates it as the current employment record.  Previously the current record would be the record where the IsActive checkbox was checked, but recent changes have provided the ability to configure the system to inactivate terminated records, even if they are the current record.
