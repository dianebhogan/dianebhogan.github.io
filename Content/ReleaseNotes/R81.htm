Enhancements

Hover on the icons to discover the level of product enhancements: .

Business Intelligence dashboard (26095)  

A Business Intelligence dashboard has been added to System Admin. From the dashboard, admin users can see at a glance their business intelligence report names and categories as well as the last edit and last run date of the report. This information can be useful when attempting to clean up a surplus of unused reports. This system also gives PrismHR developers the ability to script reports from one system to another.

Reactivate link for tasks (29507)  

A Reactivate link has been added to the Action Center for tasks. This link is only available to tasks with a status of Complete that have been Archived. The link works in much the same as the Reactivate link on the case, returning the status to New and setting the Active flag on the task to true. See Tasks: Creating and Managing.

Import process (30343)  

The import process has been enhanced to allow carriage returns or line feed characters into text area and log fields. Previously these types of characters would need to be stripped from the import data before the data was saved to the database.

Email administration (31467)

The Email Admin system has been enhanced for ease of use. A Date Sent column has been added to the email list, which displays the date and time an email was sent or UNSENT if the email cannot or has not yet been sent.

Business rule administration (35160)   

The business rule admin interface has been enhanced. The system now disables the secondary trigger field and secondary trigger value fields until after the Primary trigger fields have been completed. The system also now allows % Quick search on the Rule Title and Description fields.

Email Active Days before expiration (35553) 

The application has been enhanced to include a customizable parameter for Active Days before Expiration. Previously an external user of the system would have their account auto-expired if they had not actively logged in for more than 90 days. The system now makes this value customizable by installation, with a default value of 180 days, which is recommended. 0 Days = never expire.

HRPAPI Ongoing Import (37251) 

The HRPAPI Ongoing import has been enhanced to adhere to changes in the API user security parameter settings. Existing user accounts that have been excluded due to changes in user security parameters (UserType, UserName, or ExcludedEmail) are inactivated at the next user change in PrismHR. If the user does not exist in Pulse, their user records are removed from the import queue rather than the previous behavior of creating the user record in the system as Inactive. If the PrismHR user meets all the required parameters, their account is created/updated to match the Active status in PrismHR.

Business Rules (37987) 

New business rules have been added to the system:

  • Update Template From Sales Region uses metadata configured on the Sales Region lookup to set the template of the workspace when the sales region is changed on the Client Master dataform. If metadata for a region is not configured, no change is made. When you change templates, the action updates the users, security, and dataforms associated with the workspace to match that of the new template.
  • Warn Sales Region Template Change warns the user of these changes prior to saving. 

We recommend System Admins review each template carefully before implementing this functionality.

External Email dashboard (38022) 

The External Email dashboard has been Help enabled, to provide Help articles in Post to ClientSpace when using the email add-in for Outlook.

License system (38087) 

The license system released with our June R80 update is being implemented, tying specific features to licenses to determine which areas of Pulse are available by user.  Before installing the R81 update, you must complete the licensing configuration. For more information about managing user licenses, review Managing user Licenses in our Knowledge Base.

PrismHR Import Initial Sync (38127)

The PrismHR Import Initial Sync process has been updated to import company data. Previously the sync process excluded company-level data changes such as Contract type because these changes can affect other areas of the system. Exercise caution when including company data in the sync because changes to these data points can have far-reaching consequences.

A new API Config Parameter controls this functionality: ExcludeCompanyFromSync=0/1

0: The Company Initial import map WILL be included in the Sync if the map is active (i.e., will update Company information and all other active Initial maps).

1: The Company Initial import map will NOT be included in the Sync whether or not the map is active (i.e., all other active Initial maps will be processed).

Company information is imported first. Then all remaining active Import maps are processed (for each Company to sync). Note: If API Config param RunImport=0, import files are generated but not imported. The ExcludedContractTypes API Configuration parameter is IGNORED during the process (equates to the PrismHR 'serviceType' value).

Authentication method (38225)

The system has been enhanced to include the Authentication Type when cloning a user from a template. When the Authentication Type equal Domain, the template user domain is also cloned to the new user.

Email templates (38277) 

The email template system has been enhanced with the ability to Add Untriggered Form and Add Untriggered Field. Prior to this change, the only fields available for email template triggering were those marked as Triggerable in Dataform admin. This enhancement enables the system admin to update the triggerable flag for both dataforms and fields, without the need to leave the email template configuration. Selecting Add Untriggered Form or Add Untriggered Field presents a modal dialog window, where you can choose from a list of dataforms/fields that currently do not have the Triggerable flag set. When you save the email template, this flag is updated. This enhancement saves time for admins who would previously need to cancel the email template creation until the dataform or field could be made triggerable. For more information, review Configuring Email Templates in our Knowledge Base.

PrismHR API (38452)

The PrismHR API has been enhanced to support future employee changes. The import configuration HRPAPI_emp_futurechange*.csv has been created to capture Termination Date and Termination Reason from the employee record when that date occurs in the future. With this enhancement, you can synchronize the fields for future changes while leaving the employment status Active. For help configuring the import, log an Extranet case for one of our application specialists.

Email templates (38469)  

The From address selector in Email Templates has been updated to include the triggering dataform and all user fields on that form (even if the triggering form is a multiform). Previously the system would only display Single forms containing user fields.

Advanced System Administration (38483)

The system has been enhanced to secure Advanced System Administration using licensing. A new license has been added (Advanced System Admin), which now controls whether this set is visible in System Administration.  

Workflow Channels (38528) 

The workflow channel administration system has been enhanced with the ability to Add Untriggered Form and Add Untriggered Field. Prior to this change, the only fields available for workflow channel triggering were those marked as "Triggerable" in Dataform admin. This enhancement enables the system admin to update the triggerable flag for both dataforms and fields, without the need to leave the workflow channel configuration. Selecting Add Untriggered Form or Add Untriggered Field presents a modal dialog window, where you can choose from a list of dataforms/fields that currently do not have the Triggerable flag set. When you save the workflow channel, this flag is updated. This enhancement saves time for admins who would previously need to cancel the workflow creation until the dataform or field could be made triggerable. For more information, review Configuring your workflow in our Knowledge Base.

Core systems have been refactored to use a newer data access function. These systems include:

Database indexes (38817) 

Database tables have been optimized to improve performance for Benefits related modules such as the Offered Benefits Plan dashboard.

Database indexes (38820) 

The Users table has been optimized to improve performance for modules and searches that reference user type fields.

Workspace templates (38820) 

The system has been enhanced to allow Workspace defaults to be set based on the sales region. This enhancement includes the following changes:

  • New Dataform SalesRegionMetadata
  • LookupGroup SalesRegion now has the new dataform as its metadata form
  • Lookups in group SalesRegion can be configured to have a default workspace
  • Two new rules (inactive):
    • Soft Error rule titled Warn Sales Region Template Change, which provides an error if changing the Sales Region to one that will update the template.
    • BizLogic rule titled Update Template From Sales Region, which updates the template when changing the Sales Region to one that will update the template.
  • If a Sales Region is selected with metadata that points to a template different than the one used by the selected workspace, the system will update the workspace template.
  • If the template is not different, or no template is configured in the Sales Region metadata, template and workspace configuration will not change.
