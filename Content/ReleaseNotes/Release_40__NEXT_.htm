Task 34589                                                                                                                                      

Calendar View Module   

Resolution:                                                                                                    

There is a NEXT "Calendar" module that you can add to your User Profile, so that it shows up in the top nav, or you can get to it through the full Modules list - this is the analog to the "Calendar" module in classic.  It's meant to behave the same way as the classic calendar, with the exception of the following enhancements and known missing features:

  • Enhancements: * You can now drag events around to different time slots or days, depending on what view your in. And you can grab the top or bottom of the event and resize it to adjust the Start Time or End Time. * There are two additional views in the upper right of the calendar control: Agenda and Timeline
  • Known missing features: * You'll notice that the User field at the top is read only and defaults to the current user. We don't have the multi-select user datatype yet, so for the first release you'll only be able to see your own events when viewing outside of a workspace. When you view it from a workspace, it won't display the search fields at all, and display the name of the workspace in the Title bar. It will show all events in that workspace. Once we have the Multi-User selector you'll be able to filter to different users in that workspace. * Don't have the "Search" view, in the upper right of the calendar control, like classic does, yet.

Task 35192                                                                                                                                      

Calendar Detail - Pre-Save prompting   

Resolution:                                                                                                    

When saving a calendar event in NEXT, the pre-save prompting works as follows:

No prompting on adds, it will always send notifications. 

If any of the following fields are dirty:    

  • StartDate
  • StartTime
  • EndDate
  • EndTime
  • Location
  • IsRecurringEvent

The system will carry on with the save, without prompting, and always send a notification.

If something other than one of the above fields caused the form to be dirty, prompt with the following confirm:

"Changes have been made to this event. You can optionally send a notification to all previously invited attendees."

Click OK to send a notification. 

Click Cancel to continue saving without sending a notification.

NOTE: Newly invited attendees will always get notified.

Task 35233   - ENHANCEMENT                                                                                                                

Send Calendar Events to Contacts 

Resolution:                                                                                                    

The attendee selector for calendar events in NEXT has been enhanced to have the ability to select Contacts as attendees, in addition to Users. The selector page will continue to display Users for the Project to which the event is being added.

  • The "Type" column on the grid displays which type the attendee is, User or Contact.
  • An Organization Selector has been added to the attendee selector page, with the label "Include Contacts From".
  • Choosing an organization will include its Contacts in the search results.

The Organization that is selected on the calendar event detail is passed to the attendee selector page, and default the Organization Selector so that those Contacts are automatically included. It is editable so that you can choose Contacts from other Organizations.

If a Contact has a related User, this selector will favor the User record, and they will be added as an attendee using the User record's info rather than the info from the contact record.

Task 35048  - ENHANCEMENT

Resolution:
