Enhancements

Enhancements are grouped as Change in Functionality, ClientSpace Premium (ClientSpace with additional modules), Advanced Administration, and General.

Change in Functionality

Case

Enhancement
45781

Workflow Channels: Date fields

When a date field is used in dataform cloning and is not required, the configuration options now include None, Create Date, Specific Date, and From a Field on a Form. The None option is useful for dates that need to be filled by users after the workflow triggers. See Configuring your workflow, Step 2: Content.

44004

Workflow Channels: Date sources

You can now set date sources on a Workflow Channel task from connected tabbed forms. The option From a Field on a Form lists child tab forms when the Triggering Form (from Step 1) is a home form of a tabbed set. See Configuring your workflow, Step 2: Content.

43701

Visits Module: Mass Update

In the Visits Module, you can now use Mass Update to update Visit Status, Visit Date, and Visited By. Mass Update is a helpful time-saver, eliminating the need to open each Visit entry separately when the response is the same. See Updating visit status using Mass Update.

44621

Email Add-In: Attach emails to tasks

You can now attach an email message from your Gmail and Outlook email to a task. Attaching emails to tasks is beneficial to track relevant information when working on a task. See Attaching emails to Tasks.

46532

Custom Links: Linked Field for Workspace Dataform

When configuring a link to a Workspace Dataform, you can now select the field to which you want to link. This enhancement solves the dilemma when there is more than one parent field on a dataform to which you want to link. See Configuring links.

45967

Task Dashboard: Quick Search displays Active and Inactive records

When searching for Tasks by ID in Quick Search, the results now show both active and inactive records. You no longer have to search using All in More search. See Searching Cases and Tasks.

32107

Action Center: Tasks View Full Search

When you search using View Full Search for Tasks from the Action Center, by default, active and inactive tasks are displayed. This enhancement is available when searching for tasks from an Organization, Contacts, Workspace, and Dataforms. See Searching Tasks from an Organization, Viewing and adding Contact Tasks, Searching dataform tasks, and Workspace Action Center.

43134

Quick Search: Active and Inactive records

Quick Search now returns Active and Inactive records for WC Policies, WC Claims, EPLI Claims, and Unemployment Claims. See Quick Search.

41706

Business Intelligence Report: Case Type Definitions 

A new Business Intelligence report, Case Type Definitions, is available that lists your organization’s case types along with the category. You can run the report from BI or the Client Service Case Add page. This report is especially helpful when you want to view a case type list when on the Add page. See Finding your case types.

41798

Workflow Channels: Search

The Workflow Channels dashboard More search now provides more granular search criteria to include Channel Name, Triggering Form, Workspace Template, Workflow Item Name, Workflow Item #, Pipeline, and Type. See Searching Workflow Channels.

43192

ClientSpace Calendar: Recurring calendar appointments

You can now customize recurring monthly calendar appointments from the ClientSpace Calendar. Users can pick a day and week of the month for recurring meetings. Recurrence initiates through the Start Date field. See Recurring events.

46845

PrismHR API: Log exceptions

When the queue record fails to import into ClientSpace, we now stamp the error message onto the Queue record for Location, I9, Employee, and Employment.

ClientSpace Premium

Case

Enhancement
44677

Benefits Batch: Effective Date

The Activate link on a Benefits Batch now sets the effective date on its related Offered Benefits Plans to the Effective Date on the Batch.

47048

Payroll Batch: Get Pay Group

A new HRPAPI mapping was added to the HRPAPI_Employment*.csv and HRPAPI_initial_employment*.csv import configuration. The mapping populates the HRP Pay Group field on the Employment dataform for the associated employee.

47115

Payroll Batch: Pay Group dataform

A new multi-dataform, Pay Group, was added to associate the Payroll Batch and Billing Vouchers. Pay Group stores data about the pay groups related to the Payroll Batch. Payroll Batches with multiple pay groups will have a pay group dataform for each one associated with that batch. See Payroll Batch, Pay Group, and Billing Vouchers.

47155

47160

47198

PrismHR API: Payroll Batch structure

With the addition of Pay Group to the payroll batch dataforms, users can now track payroll batches with multiple pay groups more accurately. Payroll batches no longer choose between pay groups, now creates a record for each group. Billing Vouchers will still be viewed from the Payroll Batch, which adds an extra layer of reporting capabilities. See Payroll Batch, Pay Group, and Billing Vouchers. To manage the structure of the payroll batch, a pay group will be inserted under each payroll batch using data provided by the payroll batch.

Advanced Administration

Case

Enhancement
44563

PrismHR API: Secure HRPAPI mappings to IsDeveloper

The Import Map Header Detail has a new option, System. When the System option is enabled, only a developer user can configure the Import Map Header and Fields. This is set to True on all HRPAPI mappings. See Configuring the Import Map Header.

46654

PrismHR API: Onboarding Employees

When the Client Number exists and the Client is in a Status of Client, PendingTermination, Termination, or UnderContract, data flows from PrismHR as expected. See Importing to [%=MyVariables.ProductName%].

46333

DocuSign API: Retry threshold

The DocuSign API has a new parameter, retrythreshold, which limits the number of times a failed file in the Uploaded Files Queue can try to pass through. This enhancement prevents API downtime for the rest of the hour. Additionally, the Uploaded Files Queue has a new field, Retry Count, which reflects the number of times a connection failed. You can reset Queue Status and Retry Count in the Uploaded Files Queue. See Configuring the API record and Viewing document status in the DocuSign process in the DocuSign chapter.

44429

Salesforce: Date fields

When exporting to Salesforce, ClientSpace now provides a time zone offset for the date fields to adjust for Salesforce setting the date a day back. The new parameter, timezoneoffset, allows you to add or subtract hours. See Additional parameters.

General Enhancements

Case

Enhancement

46735

PrismHR API: Annual Pay out of range

The Employment record has a new option, Invalid Annual Pay. This option is enabled when the AnnualPay field from PrismHR is out of range. This logic applies to Employment imports that use the ClientSpace.PEO.Import.Employment Custom Import Class (in the Import Map Header Detail). See Importing to [%=MyVariables.ProductName%].
46999

PrismHR API: Payroll Batch Imports

Payroll Batch Import Queue records are now tested to determine if the Client Number in ClientSpace before processing. If the Client is not found, the queue record completes with a note. See Updates to Payroll Batch data.
44535

ClientSpace browser tab

ClientSpace browser tabs now reflect the active work item. For example, if Contact Search is the active work item, then your browser tab will reflect the same. This feature helps you to quickly identify where you want to go when you have multiple browser tabs open. See Work Center.

46064

Uploaded Files Queue

In the Uploaded Files Queue, you can now reset status back to New for documents in CS Failure status. This enhancement is for documents with a first-time error. The link only displays when the Status is CS Failure. See Viewing document status in the DocuSign process.

44307

Email Templates: Send Notification and Comments

When the Triggering Form is Task, you can now select Send Notification and Comments as triggering conditions. See Email Templates, Step 4: Conditions.

34520

Lookups: Metadata fields

Lookups now include metadata fields on the Lookup Details page, if metadata is configured for that Lookup Group. Users can configure metadata directly from the Lookup Details page; there is no need to open the metadata tab. Metadata fields are available on the edit of a lookup value.

46342

Email Admin: Search by date range

In the Email Admin dashboard, you can now filter on Date Sent. The filter is available in More search. See Email Admin dashboard.
