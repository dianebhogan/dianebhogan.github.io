Release 59 ( PEO )


Sprint Began: 8/14/2017     Sprint Ended: 9/8/2017

The following items have been completed in this release:


Callback URL for Case Discussion

The case discussion code that is used to locate discussion messages and discussion recipients have been modified to only allow the use of 8 character Row GUIDS when selecting the appropriate discussion.


Field Label Colors on Pricing Console - ENHANCEMENT

Label styles for Client Master and Pricing Batch fields exposed on the pricing console now reflect the styles as they would on the originating pages - triggering fields must also be exposed on the pricing console for this functionality to work.


Pricing Batch Minimum Admin Fields -  ENHANCEMENT

The "Minimum Admin Per Account" field and the "Minimum Admin Per Head" field on the Batch  are pricing-based fields and are now locked down after submit unless you have the biz_pricing_submitted_batch entity.  These fields now also respect the Override Pricing for this Batch so when checked they will be editable. 


Pricing Comp Code Breakdown - ENHANCEMENT

The Pricing Console has been enhanced with an information icon next to each WC Code on the pricing console. When clicked it produces a dialog similar to the Commission Detail dialog that presents detailed information about the pricing breakdown for the selected WC Code.

This breakdown displays Payroll, Admin, Taxes, Workers' Comp, Total Billing, and Oher information for the selected code.  

i3 Analytics data will additionally be displayed if your ClientSpace installation is configured to use i3 data.


Case Discussion Recipient Issues

Case discussion code has been updated to remove inactive Users, Employees and Contacts from the recipients list. 

If the current user is the only recipient selected for the message a confirmation box will be presented with the message "A recipient other than the current user has not been selected.  Do you wish to continue?"


Client Premium on OBP (Logic) - ENHANCEMENT

A new business rule has been added to set the Client Premium fields from the selected Benefits Plan's or the selected Plan Tier's Monthly Premium fieldset. This rule will trigger if the OBP Benefits Plan or Plan Tier is dirty.

The Offered Benefits Plan header has been updated so that the Total Premium column of the Rate Grid is pulled from the new fields on the form, associated formulas in the rate grid have also been updated.


Ben Recon: Client Billed Premium on OBP - ENHANCEMENT

The Convert EDV process has been updated so that the Premium Monthly Cost is pulled from the Offered Benefits Plan Client Billed Premium fieldset. 
The Import MBR import class has been updated so that the Premium Monthly Cost it is pulled from the Offered Benefits Plan Client Billed Premium fieldset.

Both these changes rely on the Benefits Plan associated with the Offered Benefits plan having the "Collection Record Premium Source" field set to "Plan Data".

Benefits Plan & Offered Benefits Plan data - ENHANCEMENT

New business rules have been added to clone data onto an Offered Plan from the associated Supplemental Benefits Plan. Fields to be cloned include:

  • Coverage
  • Cost
  • State Coordination Req
  • Employer Contribution
  • Notes 

Client Report Setting on State Legal Entity is getting Ignored

Fixed an issue where under some circumstances the Client Report Setting on State Legal Entity is getting Ignored, causing SUTA Profit, in certain situations, to not calculate correctly.


Add Logic on Case Mass Update to set Resolution Date/Time 

Resolved an issue in Mass Update for Cases where business logic would not run on Mass Update. If a Case was updated to a Status of "Complete" through Mass Update the Resolution Date/Time field was not being set. Logic was added to set the date/time field even if the Status is updated to Complete through the Mass Update functionality.


Case Type Security issue

The system has been updated to account for Case Type naming changes, adjusting the associated security entity of secured case types when the case type name changes.  This allows the entity to be created (if it does not exist) as it does today. 

If the Case Type title is changed, the business rule will locate the existing entity of the prior title and rename the entity to the new title.  This will allow for easy identification of entities for Case Types and existing security will remain intact as the title changes.

Salesforce API, user expiration

Resolved an issue where the Salesforce user (sfservice) that is used as the updated by user for dataforms expires on a regular basis, because that account typically is never logged in. 

Now, when creating a session for that user,  the User table DateLastLogin field is updated with the current date/time


PrismHR Ongoing Import enhancement - ENHANCEMENT

The system has been enhanced to catch exceptions caused by the API get methods, and log them in the Scheduled Process History table (not the exception log).  These files will be left in the Pending status (so that they will be reprocessed the next time the process executes), as it may be caused by a temporary outage of the API). 


Employee Search Dashboard Filtering

The Employee Search Dashboard was not filtering to "active" properly -  the code value being returned was incorrect and caused clients to have to remove the filter and search.  Default filtering now properly returns "Active" records.


Add Benefits Batch Dataform to PEO Workspaces - ENHANCEMENT

A Benefits Batch dataform has been added to PEO Workspaces - this dataform allows Offered benefits plans to be collected under an associated Batch.


WC Master Policy Cloning for MCP Comp Model

Resolved an issue where under certain circumstances the code which clones the policy to the workspace when choosing an MCP contract type could cause duplicates if contract types changed several times.


Pricing Comp Code - Exception Error

Resolved an issue in the Pricing calculations where an exception would be generated when total gross payroll exceeded 2 billion dollars. System is now able to handle extreme dollar amounts.


Create Business Rule to set Category on Offered Benefits Plan - ENHANCEMENT

"SetOfferedBenefitsCategory" business rule has been created that sets the Category field on the gen_BenefitBenefitsPlan dataform from the associated Plan (fkBenefitPlanID selected on the form).


Delete Comparison Not Working

Fixed an issue where surcharges and commission records associated with comparison batches in pricing were not getting deleted properly when the comparison batch was deleted causing orphaned records. 

Surcharge and commission records are now appropriately deleted with the batch.


Client Service Case Detail Report

Resolved an issue where the Case Detail report would over-run the margins of the page, this in turn would cut the text of the notes and detail short, dropping important detail.


Modify API Initial Code - ENHANCEMENT

A new API Configuration Parameter has been added to the PrismHR initial: IncludeTerminatedClients=x. 

  • If this parameter is not configured or it is set to 1, will import both Active and Terminated clients. 
  • If set to 0, will only import Active clients.

Application Error on Generate Pricing

Fixed bug when adding/editing quick create states and codes from a tablet (e.g. Safari/Chrome on iPad)

PEO Next widgets -   ENHANCEMENT

The PEO widgets in ClientSpace NEXT have all; been modified to make them more plug and play and allow them to work more easily with Widget layouts.


Field Label Colors on Pricing Console - ENHANCEMENT

'Style' type field display rules for Client Master and Pricing Batch can be used to manage Label styles on the Pricing Console

Multiple 'Style' rules can be configured for the same Controlling and Target fields (e.g. When Corp Type = LLC Pay Frequency will be RED, when C Corp then Pay Frequency will be GREEN

Only 'Style' rules are honored; 'State' rules are ignored

Multiple Workspaces under single Organization during import

During initial workspace import, do not allow a new workspace to be associated with an existing Organization if that Organization already contains a workspace.  Instead, throw an error.  The error will be formatted as follows:
A workspace was already found for the Organization associated with Client Number #####.


Application Error on Generate Pricing

Resolved an issue in Pricing Console javascript, to correct bug with jquery selector for the pricing grid that would cause an error to be thrown when adding states and codes on a mobile device.
