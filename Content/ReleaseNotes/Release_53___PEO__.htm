Release 53 ( PEO )

Sprint Began: 4/17/2017     Sprint Ended: 5/5/2017

The following items have been completed in this release:


Enhance the Home Page Case quadrant into a widget (NEXT)

The Homepage Cases quadrant in NEXT has been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

Employee Paperwork Merge  ENHANCEMENT

A filter page to the Employee Paperwork merge.

You are now able to configure the merge to run from the employee form with no filter, where it will run for that employee, and from the client master, where it will show the filter page.

Implementation Landing Widgets (NEXT)

Both the Project Milestones and Client Info panels on the Prospect Workspace Landing page have been re-engineered as a widget. This standardizes the modular design, allowing for more flexibility of future development.

Pricing Console Generate Pricing enhancements (NEXT) ENHANCEMENT  Video Coming soon...

ClientSpace pricing has been enhanced to display the underlying process behind the scenes when you click "Generate Pricing" while initially creating pricing in a workspace. The system now displays a running dialog as it Validates the codes and states, creates the codes in the workspace and finalizes pricing before finally reloading the pricing console. The maximum limit of 15 state-code combinations has also been removed.

Pricing Console enhancement/refactor (NEXT) ENHANCEMENT   Video Coming soon...

The pricing console in NEXT has been enhanced to allow you to pre-load data into the pricing grid via a pre-formatted CSV template. This template is available from the "Get Template" button under the Preload dialog (Sample template file must be uploaded under PEO Configuration in the Admin Workspace | CS PEO Configuration dataform).

Completing this template with state, code and payroll information and uploading via the dialog will pre-fill the Quick Add Grid, providing a quick method for adding large numbers of Codes to initial pricing and exposing the "Generate Pricing" button.

The maximum cap for adding codes and states to a pricing batch has also been removed as State-Code combinations are processed one at a time now, allowing for larger pricing batches to be created. 

External Landing Widgets (NEXT)

External Landing pages for both Brokers and External client service users have been "Widgetized" improving performance and allowing for more modular future development.

Service Agreement Hyperlink in WLP (NEXT)

Fixed an issue on the Workspace Landing Page Client Info widget that was allowing long file names to overrun the side of the panel. Widget now displays the words "Uploaded On ##/##/####" with the upload date of the file to the collateral form. This text is a link which opens the attached doc when clicked.

Widgetize Prospect Landing in PEO (NEXT)

Prospect Landing - (Client Info, Pricing, Activities) The Client Info, Pricing and Activities panels on the Prospect landing page in Next have been re-engineered as widgets to support future development.

Convert Shock Loss Indicator Field to Percent (Classic)

The Shock Loss Indicator field has been updated to a percent datatype to resolve issues that could arise when used in calculations.

Error converting money to data type numeric

The Full time and part time employee fields in the QuickEdit control of the pricing console have been capped with a maximum Employee count of 5000 each. Exceeding this value will present a message to the user that the Employee count is not to exceed 5000.  This is to avoid an error during pricing calculations when extremely large numbers were added for employee counts, causing the calculations to ovrerun the max field value for some pricing fields.

Pricing Console Error

Resolved an issue with Batch Repricing where the flag that prevents multiple people from attempting to change pricing was not being reset after uploading an Employee Census would lock pricing using the Batch Repricing checkbox.

Header Claim Info. Widget (NEXT)

Max Part/Full Time Employees

The pricing console Quick Add fields now respect the MAX value set on the fulltime and part-time employee count fields on the Pricing Code. If no MAX value is set on the Pricing Comp Code dataform a default max value of 5000 is set.

Slavic401k PlanHub Integration  ENHANCEMENT

The PEO System has been enhanced with an API to connect to the Slavic401K Planhub system - providing access to the first free online service that allows you to create instant, customized 401(k) cost comparison proposals for your clients.
