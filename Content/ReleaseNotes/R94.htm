Enhancements

Enhancements are grouped as Change in Functionality, ClientSpace Premium (ClientSpace with additional modules), Advanced Administration, and General.

Change in Functionality

Case

Enhancement
44446

Workflow Channels, Email Templates, Configuration Links: Clone trigger rows

You can now clone a trigger condition when creating Workflows, Email Templates, and Configuration Links. Cloning trigger conditions can save some time when you have similar triggers. See the following topics: Email Templates Step 4: Conditions, Workflow Channels Step 4: Conditions, and Configuring link display conditions.
39591

Action Center: Org Other Info and Contact Other Info

The Org Detail and Org Other Info tab now provide Action Center toolbar options for Map It, Audit Trail, Open Workspace, Refresh, and More Options: Copy Link to Clipboard and Show Org Lookups. See Portal to all actions: Action Center.

The Contact Action Center toolbar now provides options for Map It, View User / Add User, Audit Trail, Refresh, and More Options: Copy Link to Clipboard and Show Contact Lookups. See Adding contacts to the organization.

35157

Import Map Header and Detail

Configuring imports was enhanced with the following changes:

  • On the Import Map Detail form, when importing into a foreign key field, the Lookup Field now provides a selection list.
  • On the Import Map Header Detail form, when the Import Type is API, then on the Import Map Detail form, the API Path field is available. If Import Type is not API, the API Path field is hidden.
  • After the Import Map Header is created, you cannot change the Import Type to API.

See topics Configuring the Import Map Header and Configuring the Import Map Detail.
39592

External Calendar details

The ClientSpace calendar now provides basic details of external calendar events. This enhancement saves time, eliminating the need to open your external calendar. See Using the [%=MyVariables.ProductName%] Calendar.

43524

Dataform Search Modules

In Module administration, you can now create a module from a Dataform without the need to retrieve the Project Type Id from the Business Intelligence report. See Configuring a Dataform to open as a Module.

 

ClientSpace Help: Search Filters

Search Filters are now available in ClientSpace Help. You can search for a phrase or term and narrow the search using one of the available filters. See Searching ClientSpace Help.

Advanced Administration

Case

Enhancement
44665

Dataform fields: Cloneable validation

When clearing the Cloneable option for a dataform, a validation determines if the field is used by Workflow Channel dataform cloning. When used, a warning message displays along with a link to the referenced items. See Adding dataform fields.
43526

Dataform fields: Triggerable validation

When clearing the Triggerable option for a dataform, a validation determines if the field is used as a triggering condition by Workflow Channels or Email Templates. When used, a warning message displays along with a link to the referenced items. This option cannot be cleared until the dependencies are removed. See Adding dataform fields.

44666

Dataforms: Enable WFC Cloning validation

When clearing the Enable WFC Cloning option on a dataform, a validation determines if a Workflow Channel is triggering this form by cloning it. When used, a warning message is displayed with a link to the referenced items. This option cannot be cleared until the dependencies are removed. See Adding a dataform.

43802

Dataforms: Enable Triggering validation

When clearing the Enable Triggering option on a dataform, a validation determines if the dataform is used as a triggering condition in a Workflow Channel or Email Template. When used, a warning message is displayed with a link to the referenced items. This option cannot be cleared until the dependencies are removed. See Adding a dataform.

46810

Dataform fields: Limit change

The maximum number of fields allowed on a dataform was changed from 150 to 250. See Adding dataform fields.

46682

PrismHR API: Employment, Benefits, and Location Ongoing Imports

When importing Employment and Benefits data from PrismHR, errors occurred because the Employee was not found. We now create a stub Employee record so that the import runs successfully. Location imports with invalid values # or & caused errors and were not processed. Now the values will process correctly. See Importing to [%=MyVariables.ProductName%].

46821

PrismHR API: Ongoing Imports Location

During the PrismHR Ongoing process, if a Location cannot be located in PrismHR, the associated Import Queue record is now marked Completed with a note indicating the location could not be found.

Technical Enhancements

Case

Enhancement
43632

Dataform: Prevent bad characters

Bad characters are no longer saved to a dataform when receiving an email through Email Integration. The Case will be created but the invalid characters will be stripped from the text.
45860

Action Center: Task Security

From the Action Center, when adding a Task for generic Dataforms, Contacts, and the Workspace landing page, users must be a Global Admin or have a role with the Incident security entity with Add rights.

46485

Business Intelligence: Upgrade to v6_10_0_41

Business Intelligence was upgraded from v6_10_0_30 to v6_10_0_41.

46538

PrismHR Client Team Imports: Delete Event

For Ongoing Imports only, if a Client is deleted from PrismHR, the associated import queue record for the Client Team is now Completed without making a callback to PrismHR for data (the data might be gone from the system). Before this enhancement, the callback failed, and the queue record was marked Failed.

46603

Primary Key for core table tblEmailAction

A Primary Key constraint was added to the core table tblEmailAction to correct a performance concern.

46662

Zip Code Datatypes

Defensive code was added to Core Next for determining the ZipCode datatype field properties. This enhancement eliminates form errors on load.

46718

Pricing Console

Security was added so that users without rights to create tasks cannot add tasks to a Pricing console from the Action Center.

45936

Email attachments

An automated procedure now moves email attachments to the Files1 database. Previously, the attachments were stored in the standard database. Procedures were updated to include email attachments in the Files1 database.
