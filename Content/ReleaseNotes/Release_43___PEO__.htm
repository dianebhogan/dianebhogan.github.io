Release 43 ( PEO )

Product Version: PEO – Release 43 Notes - The following Tasks have been completed/fixed in this release.

Task 36232

PrismAPI Initial Import   

Resolution:                                                                                                    

The PrismHR API has been enhanced, allowing for an Automated initial Import of Client, Client Team, Location, Employee and Employment files, eliminating the need for Informer reports to seed data into a new ClientSpace system.  The scheduled process PrismHR Import Initial utilizes API configurations for logging in and processing the import.  Information on setup of API configurations for PrismHR can be found in Configuring The PrismHR API For Imports.

Task 36235                                                                                                                                      

Claim Deductible   

Resolution:                                                                                                    

Enhanced the Comp Claim dataform with a business rule which sets the Per Claim deductible to the Per Claim Deductible default set on the associated active pricing batch if this field Is not completed prior to save of the Comp Claim record and a default exists on the Pricing Batch. 

Task 36464                                                                                                                                      

convert EDV Health   

Resolution:                                                                                                    

The timeouts for the database calls during the convert EDV portion of benefits reconciliation have been increased to avoid errors during the scheduled process when running large files.

Task 36509                                                                                                                                      

Duplicating/Triplicating Pricing States   

Resolution:                                                                                                    

Resolved an issue in pricing where under certain circumstances cloning a batch could create duplicate cloned code records.  Clone function now creates a single clone record per code.
