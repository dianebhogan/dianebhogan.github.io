Release 37 ( CORE )

Product Version: CORE – Release 37 Notes - The following Tasks have been completed/fixed in this release.

Task 33781- ENHANCEMENT                                                                                                      Click here for the Video                                    

Need to handle recurring tasks   

Resolution:                                                                                                    

The task manager dashboard has been streamlined, moving some of the less used filter parameters have been relocated to the Advanced Search options.  "Recurring" has also been added to the Advanced Search parameters with options for Yes, No, or All.  If left blank this acts as "No". 

Task 33783  - ENHANCEMENT                                                                                                    Click here for the Video

ESIS Claim File Import   

Resolution:                                                                                                    

ESIS will transmit A CSV file with surrounding quotes that can be uploaded into ClientSpace. This file will include information that exists on multiple tabs within the Claim formset as well as the Claim Financial data. 

The import function has been enhanced to handle information and business logic to spread across multiple tabbed data forms and related forms, allowing the Claims import to update multiple records at once rather than creating separate import configurations.  The data includes updates to existing claims as well as inserts of new claims.

Task 33964                                                                                                                                      

"Schedule Business Intelligence..." Security not being respected from BI Dashboards   

Resolution:                                                                                                    

Fixed issue where Dashboard Scheduling was not respecting security.  We made a change where when the Business Intelligence Reports checkbox is unchecked, the scheduled process won't kick off the kickoff process. 

Task 33953                                                                                                                                      

 Index was outside the bounds of the array.   

Resolution:                                                                                                    

Fixed an issue where while importing multiple files via a scheduled process the column index being used was not being reset, causing the system to throw an "Index was outside the bounds of the array" error.

Task 34006    -ENHANCEMENT                                                                                                   

Add execution pipeline to Import Map Headers;dataform w/s move; ImportID column   

Resolution:                                                                                                   

The Import Map Header (IMH) has been enhanced with a pipeline setting that defaults to "Default", so as to not affect behavior for existing imports.

The IMH now allows you to set any column as the importid column.  Previously, the User Interface allowed you to do this, but it would clear the import ID's of affected records on subsequent imports.

Imports that attempt to move a record from one workspace to another (Client number for example has changed but importID has not) will now fail.  The error message generated will be formatted as follows:

"The Workspace (####) to which the record would be attached differs from the Workspace (####) in which the record was located.""

Task 34136    - ENHANCEMENT                                                                                                                                       

Import Log File Business Logic Errors   

Resolution:                                                                                                    

The import log process has been modified to produce an individual log entry row for each business rule error message, with date/time, imported, base message, business rule error message on the same line. Some examples of these errors include:

  • Business Logic Errors for data row ####:   - Employee Report Date cannot precede Date of Injury.
  • Business Logic Errors for data row 1 ####:   - Client Report Date cannot precede Date of Injury.
  • Business Logic Errors for data row ####:   - Carrier Report Date cannot precede Date of Injury.
  • Business Logic Errors for data row ####:   - A valid Employee record could not be located for SSN '12348765'

Task 34183  - ENHANCEMENT                                                                                                                                         

Support Multiple Lookup ImportIDs when decoding lookups during import   

Resolution:                                                                                                    

The import process has been enhanced to add the ability to pipe delimit ImportIDs in the lookup configuration.  The import will now search for lookup values that contain the data passed if it doesn't find a valid equals.

This will ensure no functionality changes.

The ImportID field must be bracketed in pipes if you need multiple values, ie: |value1|value2|value3|.

It is important to note that these values are case sensitive, so |Y| for example will be recognized as different than |y| by the system - when configuring ImportID options, plain accordingly.

Label comments have been added to the lookup ImportID field label explaining this.

A new field "Exclude from BI" has been added to the Dataform properties, allowing you to mark individual dataforms hidden from the business intelligence module and not available for reporting.
