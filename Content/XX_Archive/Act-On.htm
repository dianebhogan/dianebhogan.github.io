Act-On

Act-On is a powerful marketing tool to help you to automate Contact management and marketing. The following document describes the ClientSpace integration of this tool. If you have questions about implementing either of these products with ClientSpace, log an Extranet case.

Purpose

The purpose of this specification is to describe the integration with Act-On, and to provide a guide for configuring the system to properly implement the functionality.

Process Overview

Act-On is a system focused on collecting and managing industry marketing data. The ClientSpace integration process is designed to synchronize specific Contact information with this system.

This integration utilizes the following ClientSpace functionality:

Scheduled processes that perform the data synchronization

API Configuration forms that define the API connection and operational data

Contact, Contact Other Info forms that are used to store the Act-On data

HIGHLY configurable within ClientSpace. See the Configuration sections below for details

Act-On operates directly on the Contact form and the mapping of where data is stored cannot be configured. The process:

Adds ClientSpace Contacts to Act-On

Updates ClientSpace Contacts with Act-On Lead Score data

Triggers tasks within ClientSpace based on a Lead Score Threshold value; when a Lead Score ascends above the threshold and declines below the threshold appropriate custom task triggering will take place

These services charge for each request and limit the number of API calls allowed in a given time frame. Use caution when testing!

Act-On ClientSpace Configuration

The following form must be configured:

API Type: Web Service

App Code: Act-On (cannot deviate!)

Endpoint: this is the Act-On API server Url as in

https://restapi.Act-Onsoftware.com

Username (from Act-On)

Password (from Act-On)

SecondaryID (from Act-On). This is the ClientID value provided by Act-On.

Additional Parameters: this is a set of ‘name value’ pairs used to configure the process. Click "Add Parameter to begin adding these:

Required: clientsecret - the client secret value provided by Act-On

Required: listid - the Act-On list Id that will be used to manage Contacts. This is provided by the ClientSpace client implementing this process

Required: leadscorethreshold - the Act-On Lead Score threshold value that will be used to generate custom clone key tasks

Required: listreturnfields - a comma separated list of Contact fields to be returned when retrieving a list. For example, “listreturnfields=E-mail Address,First Name,Last Name,Company,Lead Score” will return the Email, First Name, Last Name, Company name and Lead Score for an Act-On Contact.

These fields are columns in Lists within Act-On. The field name in this configuration parameter MUST MATCH THE COLUMN NAME IN THE LIST ON Act-On!

Optional: addprocname - stored procedure used to retrieve Contacts that will be added to the Act-On list. If this field is not present the default core procedure ‘nw_contact_s_pendingAct-Onrecords’ will be used

Optional: leadscoreprocname -stored procedure used to retrieve Contacts that will be sync’d with Act-On Lead Score data. If this field is not present the default core procedure ‘nw_contact_s_pendingAct-Onleadscoresyncrecords’ will be used

Act-On Process

The Act-On integration process operates as follows:

Scheduled Process “Update Act-On Contacts” executes

Authenticate with Act-On:

Tests if an Act-On authentication has taken place in the past

If so, test the expiration time on that last authentication

If expired, create a new authentication request; store the expiration time of this new request to be used for the next authentication attempt

If not expired, use that last authentication token for this session

ADD CONTACTS: Retrieve a list of Contacts to add to Act-On based on the “addprocname” value specified in the API Configuration Additional Parameters field. The default stored procedure filters on Contact Data as follows:

luAct-OnSyncStatus = ‘Pending’

Act-OnID = empty

Active = true

for each Contact returned:

Add the Contact to Act-On with Email, First Name and Last Name

If successful, update the Contact:

Set Act-OnID

Set luAct-OnSyncStatus = Complete

Set DateLastAct-OnSend = now

Save the Contact (database save)

If adding the contact failed, update the Contact

Set luAct-OnSyncStatus = Failed

Set GeneralNotes = append “Act-On Add Contact Failed on ‘now’”

Save the Contact (database save)

Next Contact…

UPDATE LEAD SCORE: Retrieve a list of Contacts for Lead Score processing Act-On based on the “leadscoreprocname” value specified in the API Configuration Additional Parameters field. The default stored procedure filters on Contact Data as follows:

luAct-OnSyncStatus = ‘Complete’

Act-OnID = not empty

DateLastAct-OnSend = empty or > than TBD

Active = true

If there are Contacts to process

Act-On List Request: request from Act-On the List information specified in the Act-On API Configuration Additional Parameters, “listid”

For each Contact to process

If the List Request failed, update the Contact

luAct-OnSyncStatus = Failed

GeneralNotes: append “Act-On Retrieve Lead Score failed on ‘now’”

Save the Contact (database save)

If the List Request was successful

search for the Contact in the list by Act-OnID

If the contact is not found in the list, update the Contact:

luAct-OnSyncStatus = Failed

GeneralNotes: append “Act-On Retrieve Lead Score failed on ‘now’, unable to locate Contact in Act-On List”

If the contact is found in the list, update the Contact

LeadScore = Act-On Lead Score

luAct-OnSyncStatus = Complete

DateLastAct-OnSend = now

Save the Contact (database save)

If the contact is found in the list, evaluate the Lead Score

If the PRIOR Lead Score was BELOW the threshold AND the CURRENT Lead Score is ABOVE the threshold, trigger task “LEADSCORE_IMPROVED” in the CRM Workspace

If the CURRENT Lead Score is BELOW the threshold AND the PRIOR Lead Score was ABOVE the threshold, trigger task “LEADSCORE_DECLINED”

Next Contact…

Process Complete
