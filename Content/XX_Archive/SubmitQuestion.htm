Using the Employee Portal: Submit a Question

To use the Submit a Question functionality in the Employee Portal, you must have:

  • enabled for the account
  • configured with the PrismHR API, and
  • Submit a Question enabled in the Employee Portal

Submitting a case

This is what the user sees in the Employee Portal when they submit a question.

To submit a Case:

  1. From the Employee Portal dashboard, in the upper right, click (Support Center).
    The Support Center list displays.
  2. To submit a case, click Submit A Question.
    The Submit a Question form opens.

  1. Select a Case Type.
    Available case types are based on the Case Types that you configure in and must have the Expose Case Type in PrismHR option enabled.
  2. Enter a Subject and Description.
  3. Click Submit.
    A success message displays and the case is created in ClientSpace.

Viewing submitted cases in the Employee Portal

When you view cases submitted through the Employee Portal, you can see:

  1. Inclusion of Case Types when submitting a question. Managers and Employees indicate what type of question they are submitting (for example, Payroll, Benefits).
  2. A screen dedicated to showing details of submitted cases.
  3. The ability to see the details for Closed cases for easy reference.

The Employee view presents cases submitted by that employee. A Manager view shows all cases submitted BY that Manager, even if the Issue Regarding is an Employee. An Employee will not see case details if a Manager submitted the case. The power of functionality is driven through the ClientSpace API. Managers and employees can access the Support Dashboard to view the status of cases.

The status information includes:

  • Case Number
  • Subject
  • Assigned To
  • Status: Pending, Closed
  • Date Created and Date Closed
  • Description
  • Resolution
