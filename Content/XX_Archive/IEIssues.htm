ActiveX Controls need to be reinstalled

Application scope: CORE

Problem: You are a Global Admin and have installed the ActiveX Controls in IE that are used by some of the admin screens, but each time you access the admin screen you are prompted to re-install the ActiveX controls.

Cause: Protected Mode is enabled in Internet Explorer. Originally this was defaulted to unchecked, but at some point, Microsoft started pushing Windows updates which check this box by default.

Solution: Protected mode was introduced in Internet Explorer 7 and according to Microsoft:

Protected Mode is an important step forward in security for Internet Explorer (IE); it helps protect users from attack by running an IE process with greatly restricted privileges on Windows. While Protected Mode does not protect against all forms of attack, it significantly reduces the ability of an attack to write, alter, or destroy data on the user's machine or to install malicious code.

The Protected mode of Internet Explorer processes with greatly restricted privileges and also blocks the ability to install and run the ActiveX controls to your browser.

Security Warning! Protected mode is enabled for a reason. Malicious code is ubiquitous on the web today, so disabling a security measure in your browser should be thoroughly reviewed with your IT security group before proceeding. There has been some evidence that you may be able to disable this just to install the ActiveX controls, and then re-enable after, but discuss this with your internal IT group before proceeding.

To disable Protected mode in Internet explorer:

Open Internet Explorer.

Go to Tools > Internet Options > Security tab.

Clear the Enable Protected Mode option.

Click OK.
This saves your settings.

Close all Internet Explorer browser windows.

Re-open Internet Explorer and re-install the ActiveX controls. They should stay permanently installed.

Try reversing the Protected mode process when the controls are installed and see if Internet Explorer allows you to utilize the controls in Protected mode. It really is a good idea to have enhanced security.

 forces new login when switching workspaces

Application scope: CORE

Problem: When switching workspaces, the application drops the user to the login page.

Cause: A second browser has been opened to the login page, dropping the user session info.

Solution: When opening a second browser, do not click the Internet Explorer icon again. Instead, select File > New Session. This will prevent the session from restarting.

 IE 11 configuration

Problem: We have received reports of Windows updates installing Internet Explorer 11, which overwrites some of the browser configuration settings and causes unexpected behavior in the application. This topic provides step-by-step instructions on configuring the Internet Explorer 11 browser to work with .

Solution: When updating to Windows 8.1 or with recent Windows Updates, Internet Explorer 11 is also installed. There are a few steps to take to make IE 11 compatible with .

Compatibility Mode

When IE 11 is installed, it clears any current compatibility mode settings. The following error appears when attempting to login to : 

Error: pwInternetExplorerREquiredException - Internet Explorer Required. The page you are attempting to navigate to requires Internet Explorer.

To add ClientSpace to the Compatibility View Setting:

Click Tools > Compatibility View Settings.

Then add clientspace.net or your domain name to access .

Policy Editor

IE 11 also removes the Policy Editor registry settings that allow Word and Excel merges in .

To fix the settings for the Policy Editor:

Go to the Admin Workspace.

Open Active X Install.

Click the here link at the top of the page and it will ask to Repair or Remove. Leave the setting on repair and click next until it asks for the site name.

Add *.clientspace.net (or your appropriate domain) and click Create Policy.

The browser will now allow Word and Excel Merges in .

Clients who experience this issue on the Extranet may see their screen filled with gibberish, such as a repeating list of Contact Names or other scrambled data. The resolution is to enable compatibility mode for the site.

 Dialog windows are blank when opened

Application scope: CORE

Problem: Open a dataform such as a multi-form in a dialog or pop out window and when you attempt to open another dialog from that page, such as adding an attachment or using the calendar dialog the second window appears blank.

Cause: Windows Automatic update for Internet Explorer (Security Update for Microsoft Windows (KB3008923): December 9, 2014) has caused the dialog arguments to not be passed when a dialog is opened from within an existing modeless dialog. Microsoft pushed an automatic update to resolve a dialog box issue which caused issues in ClientSpace when launching a dialog box from within a dialog box. This issue has been reported to Microsoft.

Solution: Until Microsoft provides an updated security fix, the software update that causes the issue must be uninstalled to allow for multiple dialog function in .

To uninstall the Windows update:

Go to the Control panel and select Uninstall a Program.

Select View Installed Updates to display all installed Windows Updates.

In the Uninstall an Update window, find the article in question - Security Update for Microsoft Windows (KB3008923).

Right-click the article and choose Uninstall.

After this update has been uninstalled, you will need to re-boot your machine for the process to complete.

This security update is classed by Microsoft as a required Windows Update. Unless your system us specifically set to 'Hide this update' it will likely be re-downloaded and installed in the course of the Windows update process. Microsoft has provided instructions on how to hide updates here: http://technet.microsoft.com/en-us/magazine/ff382716.aspx - unfortunately to hide an update you need to catch it before it is installed, so if you have already installed the update you will need to uninstall it and re-download it prior to following these instructions. recommends communicating with your local IT department regarding the best method for insuring you will not need to repeat this process.

 Internet Explorer compatibility mode keeps resetting

Problem: Client reports that they are using Internet Explorer and have set the browser to compatibility mode using the URL button, but when they close the browser and re-open to log into ClientSpace, unexpected behavior results, and compatibility mode has been turned off.

Cause: Incompatible browser settings. It appears Microsoft has linked the "Clear History" setting to compatibility mode, so clearing the browser history will clear all the compatibility mode settings.

Solution: Correct the browser settings.

To correct the browser settings:

If client needs to have browser history cleared on close, try adding compatibility view:

Go to Tools > Compatibility View settings > Add instead of using the URL link. (Does not work in version 11 or higher)

Otherwise, simply go to Tools > Internet options and clear the Delete browsing history on exit option.

Alternatively, you can retain the Delete browsing history on exit setting but clear the option to clear browsing history under Delete settings. From Internet Options, click Delete... and clear History.

 Login values automatically cleared - System will not log in

Application scope: CORE

Problem: You attempt to log into , but when you click the login button the Username and Password fields are cleared, and you are returned to the login screen.

Cause: Cookies have been disabled in the browser.

Solution: Re-enable cookies in the browser.

To check cookie settings:

In IE 11 go to Tools > Internet Options > Security tab.

For Trusted Sites, check to make sure the Privacy slider is not all the way up. This blocks all cookies.

If this is not the case, click Advanced and check the cookie settings there. These should be set to accept both Third party and First party cookies.

Security policy editor issue

Application scope: CORE

Problem: The Security Policy Editor is returning an error or appearing as if it is not installed. When you check your programs menu, the Editor seems to be installed properly.

Cause: Internet Explorer has not been completely configured for use.

Solution: The first step to take is to be sure that the correct site has been added to the Policy Editor. Open the policy editor and when prompted insert the appropriate domain for your install. This domain should be your site name with the protocol (http://, https://) stripped off. For example, if your site name is http://knowledgespace.clientspace.net, you need to configure the policy editor for knowledgespace.clientspace.net OR *.clientspace.net as your Site Name. If it still does not work, follow these instructions to configure your browser for ClientSpace use.

In certain machine configurations, *.clientspace.net does not work, and you will be prompted to re-install the security policy editor. If this happens, do not re-install, but run the configuration tool and insure that you use the full domain name qualifier of the URL (e.g., knowledgespace.clientspace.net ).

Follow the steps in the topic Internet Explorer settings for [%=MyVariables.ProductName%] to make a trusted site. This normally indicates that other browser settings may be wrong, so take the time to complete all the steps.

 SSO Error: "Invalid Login" when using Extranet module

Application scope: CORE

Problem: Client attempts to log in to Extranet using Single Sign On (SSO) via the Extranet Module and receives a message stating they are authenticated, then immediately receives an "Invalid Login" message.

Cause: Extranet is not a trusted site.

Solution: Add the Extranet URL as a trusted site.

Go to Tools > Internet Options > Security tab

Select Trusted Sites.

Then click Sites.

Add the URL for Extranet: https://extranet.clientspace.net to the trusted sites list (this normally happens when the Require server verification option is enabled).

Save your settings.

 Stuck in loading when trying to add case or task

Application scope: CORE

Problem: When trying to add a case, task, or other item in , it will be stuck in Loading or it will show a Script Error.

Cause: There is a pop-up trying to open prompting you to select a Workspace. The Pop-up Blocker is preventing from opening Popups.

Solution: You can allow pop-ups in by following these steps:

In the Menu Bar, click Tools > Pop-up Blocker > Pop-up Blocker Settings. If the Menu Bar does not appear, right-click the header of Internet Explorer and click Menu Bar.

Add *.clientspace.net to the Pop-up Blocker Settings page (or your site URL).
