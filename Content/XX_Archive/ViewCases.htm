Viewing the submitted cases in

After a case is submitted in the Employee Portal, you can review the case details in within the client's workspace.

When the Workspace (Client) is unknown from cases created from the Employee Portal, (for example, a client number is undetermined or cannot be found), then the Case is created in the Client Services Department workspace. An administrator can change the name of the workspace for the system to redirect.

The following example shows the Category as Employee Portal with a Case Type of Manager Question.

The following fields populate from Submit a Question in the Employee Portal Support Center.

Case Type

Employee Question: Case submitted by an employee or manager for themselves.

General Question: Case submitted by a manager but not related to an employee.

Manager Question: Case submitted by a manager for an employee.
Reported By The reporter's username. If a manager submits the case, the manager's username displays. If worksite employee submits the case, the employee's username displays.
Employee The employee's name. If a manager manually enters an employee's name in the case, the information entered displays in the Issue field.
Subject The Subject defined in the case.
Issue The Description defined in the case.

When ClientSpace cannot communicate with PrismHR

When cannot communicate with PrismHR, an error message is displayed with error code: "Unfortunately, the service is currently unavailable. Please try again later. [Message ID: {Error Code}]" To view the error codes and their meaning, see Error messages: ClientSpace and the PrismHRAPI.
