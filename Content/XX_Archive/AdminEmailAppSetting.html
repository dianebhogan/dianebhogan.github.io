Admin Email application setting

Note - per product management, this content is not client facing, for internal use only.

In , you can control the email address for communications from the App Settings form. App settings are only available to Developer users and not Global Admins. For assistance, log an Extranet case.

To access App Settings:

Go to System Admin > Advanced > App Settings.
The App Settings form opens.

Let's focus on the Communications fieldset: Admin Email and Use Admin as Sender.

These two options combine to determine how the default admin email settings are used throughout :

Admin Email is the application administrator's email address.

Use Admin as Sender option (when selected) sends emails from the Admin Email address on behalf of the sender.

Using the Admin Email

In some areas of , the default Admin Email is truly defaulted and is always used (see the following table). Therefore, exercise caution with the Admin Email address. A best practice is to set the address to a noreply address so that the email address clearly indicates that recipients should not reply to these emails directly.

Area How Admin Email address used

Calendar events

Microsoft Exchange does not like the organizer to be the same as an attendee. The application will set the organizer to the Admin Email address when this happens.

Admin Email address is used when you select the Use Admin as Sender option.

Exception logging

Admin Email address is used for all exception logging for embedded email links.

File management issues

Admin Email address is used for all file management exception logging for embedded email links.

Two Factor Authentication

2FA Encryption key is sent as the application's configured Admin email address.

Email Template Triggering

If no From Address is configured and if the From Email Override is not specified, the system will look up the default Admin Email address.
