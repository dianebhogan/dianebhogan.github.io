Because of the amount of interactivity in , from dialog windows to merges, there are some basic browser settings that should be made on any machine to take full advantage of functionality. These browser changes range from security settings to add-ons and are comprised of the following.

Trusted site settings

The domain of your installation will need to be added to the browsers Trusted site list – for Clients who host with this domain is usually *.clientspace.net. As you can see from the following example, this may not be the case for your site.

You will know you have properly added the domain if you are in and access the security tab of Internet Options, the Trusted sites zone will be selected by default.

If you are unable to access the sites area, or unable to add your domain to the trusted sites list, please contact your local IT administrator, as these settings may be locked down in your environment.

Pop-up blocking

The blocking of pop-ups from can cause unexpected behavior in the application, so the domain for your install should also be added to the list of Allowed sites for pop-ups in Internet explorer.

To access the list, go to Tools> Internet Options.

Select the Privacy tab.

Click Settings.
The Pop-up Blocker Settings dialog box opens.

Ensure that your site is in the Allowed Sites list.

To add the domain of your installation to the list of allowed sites, type domain in the Address of website to allow box, prefacing it with an asterisk *.

Click Add.

Additional considerations for Internet Explorer 10+

With Internet Explorer 10 and higher, Microsoft made some major changes to the way the browser handles JavaScript and other web technologies, which can cause unexpected behavior in the application. To avoid these issues, set your browser to use compatibility mode for the application.

Compatibility mode can be set through the following steps:

Open Internet Explorer and go to your login page.

Some users have reported that the login page does not load under I.E. 10. That is OK, we just need to get the URL from the browsers address bar.

Select the domain portion of the URL.
In the example, this would be clientspace.net, your install may be different.

In Internet Explorer, go to the Tools menu.

If you do not have an Option for Tools, right-click the top of your browser and ensure the Menu Bar is selected.

Then go to Tools > Compatibility View Settings.

Add the domain portion of the URL that you just selected into the Add this website field (this will default in some cases) and click Add.
This will set the browser to use compatibility mode for any page where the URL ends in that domain (in our example, clientspace.net).

Save your settings, and you are done.

ActiveX filtering

In a few rare cases we have seen ActiveX filtering enabled in a client's browser, which again, can cause unexpected behavior in . ActiveX filtering can be disabled from the Tools menu in Internet Explorer. Review to ensure that ActiveX filtering is not selected.

Installing ActiveX Add-ins for

The ActiveX Addins can be installed from the Admin Workspace in a client's production site. In a few rare cases, the ActiveX Addin installer is not on the Admin Workspace TOC. In this situation, log an Extranet case.

From the Admin Workspace TOC, click the ActiveX Install link.

When this is first selected, It will say LOADED next to each ActiveX add-in but they will fail to load.

Although it shows the ActiveX Addins loaded, the install did not complete properly because the permission request at the bottom of the screen is needed to install properly.

Click Install and your browser will redirect off the page.

Click ActiveX Install again and this time the controls will not load manually. The application will ask to install before each ActiveX addin, totaling five times. After installed, the controls will show as LOADED.
