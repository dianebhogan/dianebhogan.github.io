Importing Policy Work Comp Code Rates

When you need to mass update several rates, use the following procedure to import the update rate file. To update a few rates, see Maintaining Policy Comp Codes. Before you can import mass updates, you must first configure the Import Map Header, review the business rules, create the Import file, edit the file, and upload.

Objective

Your primary objective is to create an import file that updates the WC Code Rates dataform. The source file (imported data) must map to the dataform fields as depicted in the following image: Comp Code, State, WC Policy, Status, Manual Rate, Funding Rate (Cost), and State Rate.

Step 1: Configure the Import Map Header and Fields

In this step, you configure the Import Map Header and Details (the fields) for the source file admin_wc_rates*.csv. First, determine if you already have the import mapping for the source file.

If the Import Map Header exists

If the import mapping exists, then you only need to activate it.

To activate the Import Map Header:

  1. Go to System Admin > Advanced > Configure Import.
    The Import Map Header Search dashboard opens.
  2. Locate the Source File admin_wc_rates*.csv.
  3. Click (Open).
    The Import Map Header Detail form opens.
  4. Select the option Active, to enable the import map header.
  5. Go to Step 2: Review Business Rules.

If the Import Map Header does not exist

If the Import Map Header does not exist, use following procedures to add the mapping.

This procedure does not include all form fields – only the ones applicable to the source file. For complete details on creating the import map header, see Configuring the Import Map Header.

Step 2: Review Business Rules

Ensure that the business rules run on the Imports pipeline. You can add to the list if it is not already there. Do this for Biz Logic or Hard Errors. Additionally, activate the rules if they are inactive.

To review business rules:

  1. Go to System Admin > Advanced > Manage Rules.
    The Custom Logic Rules dashboard opens.

  2. In Quick Search, type %WC Code Rates.
    The list of WC Code Rates rules display.

  1. Open each entry and ensure that in Execution Pipeline, Imports is present. If not, click the field and select Imports. Only update entries with the Rule Type of Biz Logic and Hard Error.

  2. Click Save.

Step 3: Create the Import File

Now you are ready to create the import file. The import file must contain the exact same column names as the mappings you established in the Import Map Detail forms (Step 1): CompCode, FundingRate, Manual Rate, PolicyNumber, State, StateRate, and Status. The import file needs two additional columns: ImportID and DefInfoImportID.

Your primary tool is a spreadsheet that provides the following:

  • Identifies your existing codes and rates for a workers' comp policy

  • Identifies the existing ImportIDs for these workers' comp code rates

  • Serves as the import template for importing rate updates

You can simply create a spreadsheet that contains the appropriate columns and then add the data. Or, you can use Business Intelligence to create the file. The spreadsheet must contain the list of existing WC Rate records and the corresponding ImportIDs. If you use a BI report, then you can download the data as a CSV file, and edit the file to update your WC Rates.

You must use existing ImportIDs to update existing records, else you risk creating duplicates.

Edit the file

A rule exists on the WC Rate table to set the ImportID. So if using an ImportID in this file that is different than the system generated one, the system generated one is saved. This is why it's important to use a BI Report to validate your existing ImportIDs. ImportIDs by the system are generated as follows: [WCPolicy.ImportID]-[StateAbbeviation]-[WCCodeMaster.CompCode]

Edit to reflect source file fields and values.

Comp Code *

Select a Comp Code.

This is a foreign key relationship to the WC Code Master table. Provide the exact value shown in the WC Code field. If a 0 or series of zeros precede the number, be sure your import file contains them too. This field should be treated as text not a number.

State *

Provide a two-letter state abbreviation.

WC Policy

(PolicyNumber * on source file)

This field points to the Policy Number. You can jump to the policy from this field. The policy has the field Policy Number.

 

Provide the ImportID of the related Policy. Changing the Policy Number or Effective Date fields on the Policy updates the ImportID on this WC Policy and related WC Rate records. Run your BI Report before importing to ensure you do not get duplicates.

Status *

Select one of the following:

  • Approved

  • Referral

  • Excluded

When you import the source file, the following values in the import file correspond to the dataform fields as:

  • Active: Approved code.

  • Restricted: Referral code.

  • Rejected: Excluded code.

Manual Rate *

The PEO manual rate. It is also used as the default manual billing rate for the SMB.

Funding Rate *

The PEO cost. The amount paid to the carrier. Sometimes this is referred to as the Net Rate or Effective Rate paid to the Carrier.

If the Use Cost Factors on the associated Policy is set to Yes and the Manual Rate is changed, this field is recalculated after imported.

State Rate *

Seldom used. Leave blank if not using. It is informational and can be used to store what a State Rate is, should it be different than the Manual Rate.

 

Step 4: Importing the File to update WC Code Rates

Now that you have an import file, the next step is to import the file so that you can update the WC Code Rates. More details about importing files is in the topic Managing import data.

To import codes:

  1. Go to System Admin > Imports > Manage Import.

  2. Click Add.
    The Upload File dialog box opens.

  3. Complete the fields and click Save.

  4. When the uploaded file is added, click Validate File.

  5. With the record in Validate Complete status, click Run Import to begin the import process.
    The action button reflects that the import process is Running. When complete, the action displays as Completed.
  6. To review the success of your import, open the record and use the action item Links on the right: View File, Log File, Log Data, and Reset Status. Log Data presents any errors during the import.
  7. To view the log file for import errors, click View Log.
